# 98. Validate Binary Search Tree

> [LeetCode 98](https://leetcode.com/problems/validate-binary-search-tree/)

给定一个二叉树，判断其是否是一个有效的二叉搜索树。

## BST 基本性质

- 节点的左子树只包含**小于**当前节点的数
- 节点的右子树只包含**大于**当前节点的数
- 所有左子树和右子树自身必须也是二叉搜索树
- BST **不需要是满二叉树**，孩子可以是 null，空树也是合法 BST
- **中序遍历下，BST 的节点值是严格递增序列**

## 我的犯错代码与纠正过程

### 第一版（错误）

```java
class Solution {
    public boolean isValidBST(TreeNode root) {
        if(root == null) return false; // ❌ 错误1：空树应该返回 true

        if(root.left != null && root.right != null){
            // ❌ 错误2：只比较了直接孩子，没有约束整棵子树
            if(root.left.val < root.val && root.right.val > root.val) return true; 
        }

        return isValidBST(root.left) && isValidBST(root.right);
    }
}
```

### 第二版（方向对了但细节有误）

```java
class Solution {
    public boolean isValidBST(TreeNode root) {
        if(root == null) return true;
    }

    private boolean helper(TreeNode root, int value){

        // ❌ 只传了一个边界值，需要同时传 lower 和 upper
```

### 纠正思路

我最初的想法是："从最基本的子树开始，只要每个节点比左孩子大、比右孩子小，递归下去就行。" 这个递归方向是对的，但问题是**只看直接孩子无法捕捉跨层违规**。

反例：

```
      5
     / \
    1   6
       / \
      3   7
```

每一层局部检查都通过：5 vs (1,6) ✓，6 vs (3,7) ✓。但 **3 在 5 的右子树里却比 5 小**，违反 BST。递归到节点 6 时，已经"忘记"了上面还有个 5，节点 3 作为 6 的左孩子确实 < 6，但它同时必须 > 5，这个信息丢失了。

**正确思路：不是检查"我和孩子的关系"，而是检查"我自己是否在祖先们允许的范围内"。** 通过参数传递上下界，每深入一层范围收紧一次：

```
         check(5, -∞, +∞)
        /                \
check(1, -∞, 5)     check(6, 5, +∞)
                    /            \
            check(3, 5, 6)   check(7, 6, +∞)
                 ↑
            3 不在 (5,6) 范围内 → false!
```

往左走 → 当前节点值成为新的**上界**；往右走 → 当前节点值成为新的**下界**。

## 踩坑记录

1. **只比较直接孩子是错的**：`root.left.val < root.val` 只验证了局部关系，无法捕捉跨层违规（如右子树深处出现比根小的值）
2. **空节点应返回 `true`**：`if (root == null) return false` 会导致所有递归最终返回 false
3. **用 `long` 而不是 `int`**：题目节点值可以是 `Integer.MIN_VALUE` 或 `Integer.MAX_VALUE`，用 `int` 做边界会出边界判断错误
4. **是开区间 `<=` / `>=`**：BST 不允许重复值，所以边界判断用严格不等
5. **初始化最小值的陷阱**：如果用 `long maxVal = Long.MIN_VALUE` 做前驱比较，当测试数据中有 `Long.MIN_VALUE` 时也会出错，更好的方式是用 `TreeNode pre` 记录前一个节点

## 代码随想录思路

**核心思路：中序遍历下，BST 输出的是有序序列。验证 BST 就变成了判断一个序列是否严格递增。**

两大陷阱：
- **陷阱1**：不能单纯比较左节点 < 中间节点、右节点 > 中间节点。要比较的是左子树**所有节点**小于中间节点，右子树**所有节点**大于中间节点
- **陷阱2**：样例中最小节点可能是 int 最小值，用 int 初始化比较值会出错。可以用 `long` 或者用 `TreeNode pre` 记录前驱来避免

## 解法一：递归 + 上下界收紧（简洁版）

```java
class Solution {
    public boolean isValidBST(TreeNode root) {
        return validBST(Long.MIN_VALUE, Long.MAX_VALUE, root);
    }
    boolean validBST(long lower, long upper, TreeNode root) {
        if (root == null) return true;
        if (root.val <= lower || root.val >= upper) return false;
        return validBST(lower, root.val, root.left) && validBST(root.val, upper, root.right);
    }
}
```

## 解法二：中序遍历递归（用 TreeNode 记录前驱）

利用 BST 中序遍历是严格递增序列的性质，遍历过程中直接判断是否有序。用 `TreeNode max` 记录前一个访问的节点，避免了 `Long.MIN_VALUE` 初始化的陷阱。

```java
class Solution {
    TreeNode max;
    public boolean isValidBST(TreeNode root) {
        if (root == null) {
            return true;
        }
        // 左
        boolean left = isValidBST(root.left);
        if (!left) {
            return false;
        }
        // 中
        if (max != null && root.val <= max.val) {
            return false;
        }
        max = root;
        // 右
        boolean right = isValidBST(root.right);
        return right;
    }
}
```

## 解法三：中序遍历递归（简洁版，用 long prev）

```java
class Solution {
    private long prev = Long.MIN_VALUE;
    public boolean isValidBST(TreeNode root) {
        if (root == null) {
            return true;
        }
        if (!isValidBST(root.left)) {
            return false;
        }
        if (root.val <= prev) {
            return false;
        }
        prev = root.val;
        return isValidBST(root.right);
    }
}
```

## 解法四：迭代法（栈模拟中序遍历）

```java
class Solution {
    public boolean isValidBST(TreeNode root) {
        if (root == null) {
            return true;
        }
        Stack<TreeNode> stack = new Stack<>();
        TreeNode pre = null;
        while (root != null || !stack.isEmpty()) {
            while (root != null) {
                stack.push(root);
                root = root.left; // 左
            }
            // 中，处理
            TreeNode pop = stack.pop();
            if (pre != null && pop.val <= pre.val) {
                return false;
            }
            pre = pop;

            root = pop.right; // 右
        }
        return true;
    }
}
```

## 解法五：统一迭代法

```java
class Solution {
    public boolean isValidBST(TreeNode root) {
        Stack<TreeNode> stack = new Stack<>();
        TreeNode pre = null;
        if (root != null)
            stack.add(root);        
        while (!stack.isEmpty()) {
            TreeNode curr = stack.peek();
            if (curr != null) {
                stack.pop();
                if (curr.right != null)
                    stack.add(curr.right);
                stack.add(curr);
                stack.add(null);
                if (curr.left != null)
                    stack.add(curr.left);
            } else {
                stack.pop();
                TreeNode temp = stack.pop();
                if (pre != null && pre.val >= temp.val)
                    return false;
                pre = temp;
            }
        }
        return true;
    }
}
```

## 复杂度

- **时间复杂度**：O(n)，每个节点访问一次
- **空间复杂度**：O(h)，递归栈深度 / 显式栈大小，h 为树高，最坏 O(n)

## 总结

1. BST 验证有两大思路：**上下界收紧**和**中序遍历判断递增**
2. 上下界法：每个节点都有一个合法范围 `(lower, upper)`，由根到当前节点的路径决定，每深入一层范围收紧一次
3. 中序遍历法：利用 BST 中序遍历是严格递增的性质，只需维护一个前驱节点做比较
4. 用 `TreeNode pre` 记录前驱比用 `long maxVal` 更安全，避免极端值的边界问题
5. 经典陷阱：只比较直接孩子 ≠ 验证整棵子树，一定要传递祖先的约束信息
