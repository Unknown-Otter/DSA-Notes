# 700. 二叉搜索树中的搜索

> [LeetCode 700. Search in a Binary Search Tree](https://leetcode.com/problems/search-in-a-binary-search-tree/)

## 思路

在二叉搜索树（BST）中查找值为 `val` 的节点并返回以该节点为根的子树。利用 BST 的性质：**左子树所有值 < 根 < 右子树所有值**，可以每次排除一半的搜索空间。

核心判断逻辑（穷举所有分支）：
1. `root == null` → 没找到，返回 `null`
2. `root.val == val` → 找到了，返回当前节点
3. `val < root.val` → 目标在左子树
4. `val > root.val` → 目标在右子树

## 解法一：递归，普通二叉树

不利用 BST 性质，左右子树都搜索。

```java
class Solution {
    public TreeNode searchBST(TreeNode root, int val) {
        if (root == null || root.val == val) {
            return root;
        }
        TreeNode left = searchBST(root.left, val);
        if (left != null) {
            return left;
        }
        return searchBST(root.right, val);
    }
}
```

- **时间复杂度**：O(n)，最坏情况遍历所有节点
- **空间复杂度**：O(n)，递归栈深度

## 解法二：递归，利用 BST 特点优化

根据 `val` 与 `root.val` 的大小关系，**每次只搜一边**。

```java
class Solution {
    public TreeNode searchBST(TreeNode root, int val) {
        if (root == null || root.val == val) {
            return root;
        }
        if (val < root.val) {
            return searchBST(root.left, val);
        } else {
            return searchBST(root.right, val);
        }
    }
}
```

- **时间复杂度**：O(h)，h 为树的高度
- **空间复杂度**：O(h)，递归栈深度

## 解法三：迭代，普通二叉树

用栈模拟 DFS 前序遍历，逐个检查节点。

```java
class Solution {
    public TreeNode searchBST(TreeNode root, int val) {
        if (root == null || root.val == val) {
            return root;
        }
        Stack<TreeNode> stack = new Stack<>();
        stack.push(root);
        while (!stack.isEmpty()) {
            TreeNode pop = stack.pop();
            if (pop.val == val) {
                return pop;
            }
            if (pop.right != null) {
                stack.push(pop.right);
            }
            if (pop.left != null) {
                stack.push(pop.left);
            }
        }
        return null;
    }
}
```

- **时间复杂度**：O(n)
- **空间复杂度**：O(n)

## 解法四：迭代，利用 BST 特点优化

利用 BST 性质，每次直接移动指针，**不需要栈**。

```java
class Solution {
    public TreeNode searchBST(TreeNode root, int val) {
        while (root != null)
            if (val < root.val) root = root.left;
            else if (val > root.val) root = root.right;
            else return root;
        return null;
    }
}
```

- **时间复杂度**：O(h)
- **空间复杂度**：O(1)，最优

## 踩坑记录

1. **`return` 的理解**：递归中每个 `return` 不是"返回最终结果"，而是**当前这一层对调用者的回答**——"以我为根的子树里，值为 val 的节点在哪？"
2. **写 `return` 卡住时**：先穷举所有分支（null、相等、小于、大于），每个分支自然对应一个 `return`，不会遗漏

## 总结

1. 普通二叉树的搜索需要遍历所有节点，时间 O(n)
2. BST 的搜索利用有序性，每次排除一半，时间 O(h)
3. 递归和迭代都能实现，BST 迭代版最优：O(h) 时间 + O(1) 空间
4. 递归的关键思维：每一层回答同一个问题，`return` 是"这一层的回答"
