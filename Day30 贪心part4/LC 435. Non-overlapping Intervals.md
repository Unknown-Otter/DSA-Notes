# LC 435. Non-overlapping Intervals

> [LeetCode 435 - Non-overlapping Intervals](https://leetcode.com/problems/non-overlapping-intervals/)

## 题目描述

给定一个区间数组 `intervals`，其中 `intervals[i] = [start_i, end_i]`，返回**需要移除的最少区间数**，使剩下的区间互不重叠。

注意：仅在端点相接的区间（如 `[1,2]` 和 `[2,3]`）**不算重叠**。

### 示例

```
Input:  [[1,2],[2,3],[3,4],[1,3]]
Output: 1   // 移除 [1,3]

Input:  [[1,2],[1,2],[1,2]]
Output: 2

Input:  [[1,2],[2,3]]
Output: 0
```

---

## 核心思路：贪心 + 按结束时间排序

### 等价转化

> "**最少删除几个**" 等价于 "**最多保留几个互不重叠的区间**"，再用总数减去它。
> 这就是经典的 **活动选择问题（Activity Selection）**。

### 贪心策略：为什么按 end 排序？

发生重叠时，应该删掉**结束时间更晚**的那个：

- 结束晚的区间"占用未来空间更多"，更容易和后续区间继续冲突。
- 保留**结束最早**的，给后面的区间留出最大的可用空间。

如果按 `end` 升序排序，扫描时只需要：
- 当前 `start >= prevEnd`：**保留**，更新 `prevEnd`。
- 当前 `start < prevEnd`：**删除**，`prevEnd` 不动（继续保护那个更早结束的）。

### 重叠判断

- **重叠条件**：`intervals[i][0] < prevEnd`（严格小于，因为端点相接不算重叠）
- **不重叠条件**：`intervals[i][0] >= prevEnd`

---

## 解法：贪心 + 按 end 排序

```java
class Solution {
    public int eraseOverlapIntervals(int[][] intervals) {
        if (intervals == null || intervals.length == 0) return 0;
        
        // 按结束时间升序排序
        Arrays.sort(intervals, (a, b) -> Integer.compare(a[1], b[1]));
        
        int count = 0;                    // 需要删除的数量
        int prevEnd = intervals[0][1];    // 上一个保留区间的结束时间
        
        for (int i = 1; i < intervals.length; i++) {
            if (intervals[i][0] < prevEnd) {
                // 重叠：删除当前区间（它的 end 更晚）
                count++;
            } else {
                // 不重叠：保留，更新 prevEnd
                prevEnd = intervals[i][1];
            }
        }
        return count;
    }
}
```

### 复杂度

- **时间**：`O(n log n)`，瓶颈在排序。
- **空间**：`O(log n)`，排序栈空间。

## 走一遍 Example 1

输入：`[[1,2],[2,3],[3,4],[1,3]]`

按 `end` 排序后：`[[1,2],[2,3],[1,3],[3,4]]`

| i | 当前区间 | prevEnd | `start < prevEnd`? | 操作 |
|---|----------|---------|--------------------|------|
| 0 | [1,2]    | -       | 初始化              | prevEnd = 2 |
| 1 | [2,3]    | 2       | 2 < 2? 否           | 保留，prevEnd = 3 |
| 2 | [1,3]    | 3       | 1 < 3? 是           | **删除**，count = 1 |
| 3 | [3,4]    | 3       | 3 < 3? 否           | 保留，prevEnd = 4 |

输出：`count = 1` ✅

## 备选解法：按 start 排序

也可以按起点排序，但重叠时的处理稍微绕一点：需要**主动选择"prevEnd 更小的那个"留下**。

```java
class Solution {
    public int eraseOverlapIntervals(int[][] intervals) {
        Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));
        
        int count = 0;
        int prevEnd = intervals[0][1];
        
        for (int i = 1; i < intervals.length; i++) {
            if (intervals[i][0] < prevEnd) {
                // 重叠：删一个，保留 end 更小的那个
                count++;
                prevEnd = Math.min(prevEnd, intervals[i][1]);
            } else {
                prevEnd = intervals[i][1];
            }
        }
        return count;
    }
}
```

> 两种排序方式都对，但**按 end 排序**的逻辑更干净——扫描到的每个保留区间天然就是"最早结束"的，不需要额外的 `Math.min`。

## 易错点 & Bug 历史

1. **重叠判断用 `<=` 还是 `<`？**
   题目明确说端点相接不算重叠，所以用**严格小于** `<`。
   - `[1,2]` 和 `[2,3]` 中：`2 < 2` 为假 → 不重叠 ✅

2. **删除区间时不要更新 `prevEnd`。**
   既然选择了保留前一个（更早结束的），`prevEnd` 就维持不变。如果错误地更新成当前区间的 end，等于把"应该删的"留下来了。

3. **想当然按 start 排序。**
   按 start 排序也能做，但要记得在重叠时取 `min(prevEnd, currEnd)`，否则会保留错误的区间。
   按 end 排序则没有这个心智负担。

4. **空数组检查。**
   虽然 LeetCode 一般不给空数组，但写代码时养成习惯。

---

## 一句话总结

1. **转化问题**：删最少 ⇔ 保留最多互不重叠的区间（活动选择问题）。
2. **排序**：按 `end` 升序排序，为后续区间留出最大空间。
3. **扫描**：用 `prevEnd` 跟踪上一个保留区间的结束点。
4. **决策**：`start < prevEnd` 则删（count++）；否则保留并更新 `prevEnd`。
5. **复杂度**：时间 `O(n log n)`，空间 `O(log n)`。
