# 452. Minimum Number of Arrows to Burst Balloons

> [LeetCode 452. Minimum Number of Arrows to Burst Balloons](https://leetcode.com/problems/minimum-number-of-arrows-to-burst-balloons/)

## 题目描述

气球以 2D 整数数组 `points` 表示，其中 `points[i] = [xstart, xend]` 表示一个气球的水平直径在 x 轴上从 `xstart` 到 `xend`。

弓箭从 x 轴上垂直向上射出，若 `xstart <= x <= xend`，气球被射爆。求**射爆所有气球所需的最少弓箭数**。

**示例**：

```
Input: points = [[10,16],[2,8],[1,6],[7,12]]
Output: 2
解释:
- 在 x = 6 射一箭，射爆 [2,8] 和 [1,6]
- 在 x = 11 射一箭，射爆 [10,16] 和 [7,12]
```

## 解题思路：贪心算法

### 直觉分析

- 直觉：射重叠最多的气球，用的弓箭一定最少
- 反例验证：是否存在"重叠 3 个气球，先射 2 个、留 1 个和后面一起射" 更优的情况？→ **不存在**
- 因此可以使用贪心策略

**贪心策略**：
- **局部最优**：当气球出现重叠，一起射，所用弓箭最少
- **全局最优**：把所有气球射爆所用弓箭最少

### 模拟方式选择

不需要真的从数组中 remove 元素，只需：
1. 排序后从前往后遍历
2. 用一个变量记录箭的数量
3. 被"射过"的气球跳过即可

### 排序策略

按 `xstart` 升序排序，从前向后遍历，让气球尽可能左侧重叠。

## 核心算法

### 关键判断条件

```java
if (points[i][0] > points[i-1][1]) {
    // 不重叠 → 必须新加一支箭
    arrows++;
} else {
    // 重叠 → 更新当前重叠组的右边界为最小值
    points[i][1] = Math.min(points[i][1], points[i-1][1]);
}
```

### 为什么右边界要取最小值？

**反例说明**：假设气球（已排序）`[1, 10]`、`[2, 4]`、`[5, 8]`

如果**不更新右边界**，一直用 `[1, 10]` 的 `10` 比较：
- `[2, 4]` 起点 2 ≤ 10，重叠 ✓
- `[5, 8]` 起点 5 ≤ 10，重叠 ✓
- 错误结论：1 支箭

但实际上 `[2, 4]` 和 `[5, 8]` 不重叠（4 < 5），一支箭无法同时穿透。

**正确逻辑**：右边界 = `min(前一个右边界, 当前右边界)`，代表**当前重叠组真正能被一支箭覆盖的区间**。

### 本质理解

`points[i][1] = Math.min(points[i][1], points[i-1][1])` 维护的是：

> 在已排序的当前重叠组中，**所有气球右边界的最小值** —— 也就是 **这一支箭最晚能射到哪里**。

- `min(右边界)` = 这组重叠气球的"共同右截止点"
- 一支箭只有在 `x ≤ min(右边界)` 时，才能同时穿透组内所有气球
- 一旦下一个气球的 `xstart > min(右边界)`，必须新加一支箭

**类比**：一群人约时间见面，碰面截止 = 所有人最晚时间里**最早**的那个。

## Java 完整代码

```java
class Solution {
    public int findMinArrowShots(int[][] points) {
        if (points.length == 0) return 0;

        // 按起始位置升序排序（用 Integer.compare 避免 int 溢出）
        Arrays.sort(points, (a, b) -> Integer.compare(a[0], b[0]));

        int arrows = 1;  // 至少需要一支箭
        for (int i = 1; i < points.length; i++) {
            if (points[i][0] > points[i - 1][1]) {
                // 不重叠 → 必须新加一支箭
                arrows++;
            } else {
                // 重叠 → 更新右边界为最小值
                points[i][1] = Math.min(points[i][1], points[i - 1][1]);
            }
        }
        return arrows;
    }
}
```

## Bug 历史 / 易错点

### 排序时不能用 `a[0] - b[0]`

```java
// ❌ 错误写法（会溢出）
Arrays.sort(points, (a, b) -> a[0] - b[0]);

// ✅ 正确写法
Arrays.sort(points, (a, b) -> Integer.compare(a[0], b[0]));
```

**原因**：当 `a[0] = Integer.MIN_VALUE`、`b[0] = 1` 时：
```
a[0] - b[0] = -2147483648 - 1 = 2147483647 (溢出！)
```
本应返回负数（a 排前面），溢出后变成正数，**排序结果完全错误**。

LeetCode 452 的数据范围包含 `Integer.MIN_VALUE`，必须用 `Integer.compare()` 或 `Long.compare()`。

## 知识点扩展：Java Comparator & Lambda

### Comparator 核心规则

比较器返回 `int`，含义是 **"a 应该排在 b 前面还是后面"**：

| 返回值 | 含义 |
|--------|------|
| `< 0` | a 排在 b 前面（a 更小） |
| `== 0` | a 和 b 相等 |
| `> 0` | a 排在 b 后面（a 更大） |

**升序记忆口诀**：`(a, b) -> a - b` → "a 减 b"，结果为负说明 a 小，a 排前面 → 升序

### Lambda 三种等价写法

```java
// 写法 1：完整匿名内部类（最直观）
Arrays.sort(points, new Comparator<int[]>() {
    @Override
    public int compare(int[] a, int[] b) {
        return Integer.compare(a[0], b[0]);
    }
});

// 写法 2：Lambda 表达式（最常用）
Arrays.sort(points, (a, b) -> Integer.compare(a[0], b[0]));

// 写法 3：Comparator.comparingInt（最简洁，推荐）
Arrays.sort(points, Comparator.comparingInt(a -> a[0]));
```

Lambda 本质：**"只有 compare 方法的类" 的简写**。

### 常见比较器模板

#### 1. `int[]` 升序（基础）

```java
int[] arr = {3, 1, 4, 1, 5};
Arrays.sort(arr);  // 直接升序，无需比较器
```

#### 2. `Integer[]` 降序

```java
Integer[] arr = {3, 1, 4, 1, 5};
Arrays.sort(arr, (a, b) -> Integer.compare(b, a));
// 注意：基础类型 int[] 不能直接降序，必须装箱成 Integer[]
```

#### 3. `int[][]` 按第 0 列升序（LC 452 用法）

```java
Arrays.sort(points, (a, b) -> Integer.compare(a[0], b[0]));
```

#### 4. `int[][]` 按第 1 列降序

```java
Arrays.sort(intervals, (a, b) -> Integer.compare(b[1], a[1]));
```

#### 5. 多关键字排序（先按 a[0] 升，相同时按 a[1] 降）

```java
Arrays.sort(people, (a, b) -> {
    if (a[0] != b[0]) return Integer.compare(a[0], b[0]);
    return Integer.compare(b[1], a[1]);
});
```

#### 6. `List<Integer>` 升序 / 降序

```java
Collections.sort(list);                              // 升序
list.sort((a, b) -> Integer.compare(b, a));          // 降序
Collections.sort(list, Collections.reverseOrder());  // 也是降序
```

#### 7. 字符串按长度升序，长度相同按字典序

```java
Arrays.sort(strs, (a, b) -> {
    if (a.length() != b.length()) return a.length() - b.length();
    return a.compareTo(b);
});
```

#### 8. `Comparator.comparingInt`（最优雅）

```java
Arrays.sort(points, Comparator.comparingInt(a -> a[0]));
// 按 a[0] 升序，自动用 Integer.compare，不会溢出
```

### 写比较器的 4 步思考流程

1. **元素是什么类型？**（`int[]`、`String`、`int`...）
2. **按哪个字段排序？升序还是降序？**
3. **选择安全的比较函数**：
   - `int` → `Integer.compare(a, b)`
   - `long` → `Long.compare(a, b)`
   - `String` → `a.compareTo(b)`
4. **升序写 `compare(a, b)`，降序写 `compare(b, a)`**（参数交换位置）

## 总结

1. **贪心策略**：按起点排序后，遇到重叠就一起射；不重叠则必须新加一支箭
2. **重叠判断**：`points[i][0] > points[i-1][1]` 即不重叠
3. **关键技巧**：用 `Math.min` 维护当前重叠组的"最严格右边界"——这是一支箭最晚能落点的位置
4. **排序陷阱**：永远用 `Integer.compare(a, b)` 替代 `a - b`，避免 `Integer.MIN_VALUE` 溢出
5. **复杂度**：O(n log n) 时间，O(log n) 空间
