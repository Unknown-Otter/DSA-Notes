# 105. Construct Binary Tree from Preorder and Inorder Traversal

> [LeetCode 105](https://leetcode.com/problems/construct-binary-tree-from-preorder-and-inorder-traversal/)

## 核心思路

给定 `preorder` 和 `inorder` 两个数组，还原一棵二叉树。

**两个数组的职责互补：**

- **preorder**：告诉你当前子树的 **root 是谁**（每层第一个就是 root）
- **inorder**：告诉你 **左右子树的范围**（root 左边是左子树，右边是右子树）

单独给一个数组无法唯一确定一棵树，必须两个配合。

## 递归逻辑

每一层递归做三件事：

1. 从 `preorder` 用全局指针 `preIdx++` 取当前 root
2. 在 `inorder` 中找到 root 的位置 `mid`，划分左右子树范围
3. **先递归左子树，再递归右子树**（顺序不能反，因为 preorder 是 root→左→右）

**关键设计决策：**

- 用 **HashMap** 存 inorder 的 `val → index` 映射，O(1) 查找 root 位置
- 用**全局 `preIdx`** 指针，每建一个节点自动前进，不需要手动计算下标跳转
- `inLeft > inRight` 时返回 null，保证 preIdx 不会多消耗

## 代码

```java
class Solution {
    int preIdx = 0;
    Map<Integer, Integer> inMap = new HashMap<>();

    public TreeNode buildTree(int[] preorder, int[] inorder) {
        for (int i = 0; i < inorder.length; i++) {
            inMap.put(inorder[i], i);
        }
        return build(preorder, 0, inorder.length - 1);
    }

    private TreeNode build(int[] preorder, int inLeft, int inRight) {
        if (inLeft > inRight) return null;

        int rootVal = preorder[preIdx++];
        TreeNode root = new TreeNode(rootVal);

        int mid = inMap.get(rootVal);

        root.left  = build(preorder, inLeft, mid - 1);  // 先左
        root.right = build(preorder, mid + 1, inRight);  // 再右

        return root;
    }
}
```

## 手动模拟（三层树）

```
        1
       / \
      2    3
     / \    \
    4   5    7

preorder = [1, 2, 4, 5, 3, 7]
inorder  = [4, 2, 5, 1, 3, 7]
inMap: {4:0, 2:1, 5:2, 1:3, 3:4, 7:5}
```

| 步骤 | preIdx | rootVal | mid | 左子树范围 | 右子树范围 | 说明 |
|------|--------|---------|-----|-----------|-----------|------|
| 1 | 0→1 | 1 | 3 | [0, 2] | [4, 5] | 整棵树的 root |
| 2 | 1→2 | 2 | 1 | [0, 0] | [2, 2] | 左子树的 root |
| 3 | 2→3 | 4 | 0 | [0,-1]空 | [1,0]空 | 叶子节点 |
| 4 | 3→4 | 5 | 2 | [2,1]空 | [3,2]空 | 叶子节点 |
| 5 | 4→5 | 3 | 4 | [4,3]空 | [5, 5] | 右子树的 root |
| 6 | 5→6 | 7 | 5 | [5,4]空 | [6,5]空 | 叶子节点 |

**preIdx 从 0 一路走到 6，每个节点恰好取走一次。**

## 复杂度

- **时间**：O(n)，每个节点建一次，HashMap 查找 O(1)
- **空间**：O(n)，HashMap 存 n 个映射 + 递归栈最坏 O(n)

# 106. Construct Binary Tree from Inorder and Postorder Traversal

> [LeetCode 106](https://leetcode.com/problems/construct-binary-tree-from-inorder-and-postorder-traversal/)

## 核心思路

和 105 思路完全一致，只有两个关键变化：

1. **从 postorder 末尾往前取 root**：`postIdx--` 代替 `preIdx++`
2. **先递归右子树，再递归左子树**：因为 postorder 是 `左→右→root`，倒着读就是 `root→右→左`

## 代码

```java
class Solution {
    int postIdx;
    Map<Integer, Integer> inMap = new HashMap<>();

    public TreeNode buildTree(int[] inorder, int[] postorder) {
        postIdx = postorder.length - 1; // 注意：不能在声明时赋值，此时 postorder 还不存在

        for (int i = 0; i < inorder.length; i++) {
            inMap.put(inorder[i], i);
        }
        return build(postorder, 0, inorder.length - 1);
    }

    private TreeNode build(int[] postorder, int inLeft, int inRight) {
        if (inLeft > inRight) return null;

        int rootVal = postorder[postIdx--];
        TreeNode root = new TreeNode(rootVal);

        int mid = inMap.get(rootVal);

        root.right = build(postorder, mid + 1, inRight);  // 先右！
        root.left  = build(postorder, inLeft, mid - 1);   // 再左！

        return root;
    }
}
```

## 105 vs 106 对比

| | 105 (Pre + In) | 106 (In + Post) |
|---|---|---|
| 取 root 的数组 | preorder | postorder |
| 指针方向 | `preIdx++`（从前往后） | `postIdx--`（从后往前） |
| 递归顺序 | 先左再右 | **先右再左** |
| inMap 作用 | 相同：查 root 在 inorder 中的位置，划分左右子树 | 相同 |

## 区间约定：左闭右闭 `[inLeft, inRight]`

本题的数组切割全程使用**左闭右闭**区间，这决定了三个地方的写法：

**1. Base case：`inLeft > inRight`**

```java
if (inLeft > inRight) return null;
```

左闭右闭下，`inLeft == inRight` 表示还有一个节点（叶子），仍需处理。只有 `inLeft > inRight` 才是真正为空。如果用左闭右开 `[inLeft, inRight)`，base case 就要改成 `inLeft >= inRight`。

**2. 切割时 root（mid）不属于左也不属于右**

```java
int mid = inMap.get(rootVal);
root.left  = build(preorder, inLeft, mid - 1);   // 左子树: [inLeft, mid-1]
root.right = build(preorder, mid + 1, inRight);   // 右子树: [mid+1, inRight]
```

`mid` 本身是当前 root，两边都跳过它。左边到 `mid - 1`，右边从 `mid + 1` 开始。

**3. 空子树的自然表达**

当 root 没有左子树时，`mid == inLeft`，左子树范围变成 `[inLeft, inLeft - 1]`，即 `inLeft > inRight`，自动返回 null。右子树同理。不需要额外判断。

**如果用左闭右开 `[inLeft, inRight)` 的写法：**

| | 左闭右闭 `[l, r]` | 左闭右开 `[l, r)` |
|---|---|---|
| base case | `l > r` → null | `l >= r` → null |
| 左子树 | `[inLeft, mid - 1]` | `[inLeft, mid)` |
| 右子树 | `[mid + 1, inRight]` | `[mid + 1, inRight)` |

两种都能 AC，保持统一即可。本题选择左闭右闭，因为语义更直观。

## 踩坑记录

1. **`postIdx` 初始化位置**：不能写 `int postIdx = postorder.length - 1;` 作为成员变量，因为声明时 `postorder` 参数还不存在，必须在 `buildTree` 方法内赋值
2. **递归顺序不能反**：105 先左再右，106 先右再左，和数组的遍历顺序严格对应。如果顺序写反，`postIdx` / `preIdx` 会消耗错误的节点
3. **preorder/postorder 中每个节点都是某棵子树的 root**（叶子是大小为 1 的子树的 root），所以全局指针简单 `++` 或 `--` 就够了，不需要手动计算跳转
4. **inorder 本质上就是一个查找表**，建 HashMap 让查找从 O(n) 优化到 O(1)

## 复杂度

- **时间**：O(n)
- **空间**：O(n)
