# 112. 路径总和 Path Sum

> [LeetCode 112. Path Sum](https://leetcode.com/problems/path-sum/)

给定一个二叉树和一个目标和 `targetSum`，判断该树中是否存在**根节点到叶子节点**的路径，这条路径上所有节点值相加等于目标和。叶子节点是指没有子节点的节点。

---

## 踩坑记录

### 我的错误代码

```java
class Solution {
    public boolean hasPathSum(TreeNode root, int targetSum) {
        if(root.val == targetSum && root.left == null && root.right == null) return true;

        int left = hasPathSum(root.left);
        int right = hasPathSum(root.right); 

        return left == targetSum || right == targetSum; 
    }
}
```

### 错误分析

1. **没有处理 `root == null` 的情况**：当 root 为空时直接访问 `root.val` 会 NPE
2. **递归签名错误**：`hasPathSum(root.left)` 少传了 `targetSum` 参数，且返回值类型应为 `boolean` 而不是 `int`
3. **无法体现路径累加**：只在 base case 里比较 `root.val == targetSum`，没有把路径上之前节点的值考虑进来
4. **核心思维误区**：试图"累加"路径和再比较，但其实应该反过来——**用递减的方式，每经过一个节点就减去它的值，到叶子节点时看剩余是否为 0**

---

## 关键思路：递减而非累加

不要累加然后判断是否等于目标和，而是让 `targetSum` 初始为目标值，然后**每经过一个节点就减去该节点的值**。如果到叶子节点时 `targetSum` 恰好减为 0，说明找到了目标路径。

### 递归函数何时需要返回值？

- **搜索某一条符合条件的路径** → 递归需要返回值（`boolean`），找到就立刻返回，**不需要遍历整棵树**（本题 112）
- **搜索整棵树，收集所有路径** → 递归不需要返回值（`void`），必须遍历完整棵树（113 路径总和 II）

## 递归法（展开版，体现回溯过程）

```java
class Solution {
    public boolean hasPathSum(TreeNode root, int targetSum) {
        if (root == null) return false;
        return traversal(root, targetSum - root.val);
    }

    private boolean traversal(TreeNode cur, int count) {
        // 叶子节点，且计数器为0 → 找到了
        if (cur.left == null && cur.right == null && count == 0) return true;
        // 叶子节点，计数器不为0 → 这条路不行
        if (cur.left == null && cur.right == null) return false;

        // 左子树
        if (cur.left != null) {
            count -= cur.left.val;                    // 处理
            if (traversal(cur.left, count)) return true; // 递归，找到就立刻返回
            count += cur.left.val;                    // 回溯，撤销处理
        }
        // 右子树
        if (cur.right != null) {
            count -= cur.right.val;                    // 处理
            if (traversal(cur.right, count)) return true;
            count += cur.right.val;                    // 回溯，撤销处理
        }
        return false;
    }
}
```

**回溯体现在哪里？** `count -= cur.left.val` 是处理，`count += cur.left.val` 是回溯撤销。走完左子树回来后，`count` 恢复原值，再去探索右子树。

## 递归法（精简版，隐式回溯）

```java
class Solution {
    public boolean hasPathSum(TreeNode root, int targetSum) {
        if (root == null) return false;

        // 到叶子节点，检查剩余值是否等于当前节点值
        if (root.left == null && root.right == null) return root.val == targetSum;

        // 减去当前值，传给左右子树
        return hasPathSum(root.left, targetSum - root.val) 
            || hasPathSum(root.right, targetSum - root.val);
    }
}
```

**隐式回溯**：`targetSum - root.val` 作为参数传递，每层递归有自己的局部值，函数返回后 `targetSum` 自然不变，不需要手动撤销。

### 举例：树 `5 -> 4 -> 3`，`targetSum = 12`

| 节点 | targetSum | 传给子节点 | 判断 |
|------|-----------|-----------|------|
| 5 | 12 | 12 - 5 = 7 | 非叶子，继续 |
| 4 | 7 | 7 - 4 = 3 | 非叶子，继续 |
| 3 | 3 | — | 叶子，3 == 3 ✅ |

## 迭代法（栈模拟 DFS）

栈中每个元素不仅存节点指针，还要存**从根到该节点的路径数值总和**。用两个栈分别存节点和累计值。

```java
class Solution {
    public boolean hasPathSum(TreeNode root, int targetSum) {
        if (root == null) return false;
        Stack<TreeNode> stack1 = new Stack<>();
        Stack<Integer> stack2 = new Stack<>();
        stack1.push(root);
        stack2.push(root.val);

        while (!stack1.isEmpty()) {
            TreeNode node = stack1.pop();
            int sum = stack2.pop();

            // 叶子节点且路径和等于目标 → 找到了
            if (node.left == null && node.right == null && sum == targetSum) {
                return true;
            }
            // 右子节点入栈，累加路径值
            if (node.right != null) {
                stack1.push(node.right);
                stack2.push(sum + node.right.val);
            }
            // 左子节点入栈，累加路径值
            if (node.left != null) {
                stack1.push(node.left);
                stack2.push(sum + node.left.val);
            }
        }
        return false;
    }
}
```

**迭代的回溯在哪里？** 每个栈元素自带独立的 `sum` 值，弹出一个节点后自然切换到另一条路径的累计值，天然完成了回溯。

## 延伸：113. 路径总和 II（收集所有路径）

> [LeetCode 113. Path Sum II](https://leetcode.com/problems/path-sum-ii/)

与 112 的区别：需要找到**所有**满足条件的路径，所以必须遍历整棵树，递归函数**不需要返回值**（`void`），且需要手动维护 `path` 列表做**显式回溯**。

```java
class Solution {
    List<List<Integer>> result = new LinkedList<>();
    LinkedList<Integer> path = new LinkedList<>();

    public List<List<Integer>> pathSum(TreeNode root, int targetSum) {
        if (root == null) return result;
        path.add(root.val);
        traversal(root, targetSum - root.val);
        return result;
    }

    private void traversal(TreeNode cur, int count) {
        // 叶子节点，且计数为0 → 收集路径
        if (cur.left == null && cur.right == null && count == 0) {
            result.add(new ArrayList<>(path)); // 注意要 new 一份拷贝
            return;
        }
        if (cur.left == null && cur.right == null) return;

        if (cur.left != null) {
            path.add(cur.left.val);               // 处理
            traversal(cur.left, count - cur.left.val); // 递归
            path.removeLast();                     // 回溯
        }
        if (cur.right != null) {
            path.add(cur.right.val);               // 处理
            traversal(cur.right, count - cur.right.val); // 递归
            path.removeLast();                     // 回溯
        }
    }
}
```

## 总结

1. **递减思路**：不累加路径和，而是用 `targetSum` 逐步减去节点值，到叶子时判断是否为 0
2. **回溯的两种形式**：显式（手动加减 count）和隐式（作为参数传递 `targetSum - root.val`），本质相同
3. **递归返回值的选择**：搜索一条路径用 `boolean` 返回值（112），收集所有路径用 `void`（113）
4. **迭代法**：栈中额外存储路径累计值，每个元素独立，天然实现回溯
5. **易错点**：别忘了 `root == null` 的判断；递归时参数要完整传递；叶子节点的定义是**左右子节点都为 null**
