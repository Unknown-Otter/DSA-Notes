# 513. 找树左下角的值

> [LeetCode 513. Find Bottom Left Tree Value](https://leetcode.com/problems/find-bottom-left-tree-value/)

给定一个二叉树的根节点 `root`，请找出该二叉树的 **最底层最左边** 节点的值。

## 思路

### 核心观察

要找"最后一层最左边的节点"，用 **BFS 层序遍历** 是最直观的：
- 每一层从左到右入队
- **每层的第一个节点**就是这一层最左边的值
- 遍历完所有层后，最后一层的第一个值就是答案

### 关键技巧：last-wins 模式

**不判断"是不是最后一层"，而是每层都记录一次最左值，不断覆盖**。循环自然结束时，最后一次记录的就是答案。

> 与其判断"终点"，不如每一步都假设自己是终点，让循环的自然结束替你做判断。

### 如何拿到"这一层的第一个节点"

在 `int size = q.size()` 执行的瞬间，队列里装的**正好是这一层的所有节点，从左到右排列**（因为上一层处理时按"先左后右"的顺序入队，FIFO 保证了顺序）。

所以：
- 队首（`peek()`）= 这一层最左节点
- 用 `peek()` 而不是 `poll()`，是因为**只想偷看不想移除**——内层循环还要靠 `poll()` 来依次处理该节点并入队它的孩子

### ⚠️ 易错点：peek 的位置

`q.peek()` 返回的是**"此刻"队首的元素**，不是"这一层"的第一个元素。

| peek 位置 | 结果 |
|---|---|
| 内层循环**之前** | 队首还没被 poll 过 → 是本层最左节点 ✅ |
| 内层循环**之中** | 每 poll 一次队首就变 → 记录的是"剩余节点中的第一个" ❌ |

**"这一层的第一个节点"这件事只成立一个瞬间——就是这一层刚开始处理、还没 poll 任何节点的那一刻。**

## 代码

```java
class Solution {
    public int findBottomLeftValue(TreeNode root) {
        Queue<TreeNode> q = new ArrayDeque<>();
        q.offer(root);
        int res = 0;

        while (!q.isEmpty()) {
            int size = q.size();
            res = q.peek().val;   // ★ 必须在内层循环之前

            for (int i = 0; i < size; i++) {
                TreeNode node = q.poll();
                if (node.left != null) q.offer(node.left);
                if (node.right != null) q.offer(node.right);
            }
        }

        return res;
    }
}
```

### DFS 递归解法

**关键思路**：维护当前最大深度和该深度下第一次访问到的节点值。**优先往左走**，并只在**深度第一次被更新时**记录值，记录下来的就是该深度最左边的节点。

```java
class Solution {
    private int maxDepth = -1;
    private int res;

    public int findBottomLeftValue(TreeNode root) {
        dfs(root, 0);
        return res;
    }

    private void dfs(TreeNode node, int depth) {
        if (node == null) return;
        if (depth > maxDepth) {   // 第一次到达这个深度
            maxDepth = depth;
            res = node.val;
        }
        dfs(node.left, depth + 1);    // 先左后右 → 保证最左优先
        dfs(node.right, depth + 1);
    }
}
```

## 复杂度

- 时间复杂度：**O(n)**，每个节点访问一次
- 空间复杂度：**O(n)**，BFS 队列最多存一层节点；DFS 递归栈深度为树高

## 踩坑记录

### 坑 1：把记录语句写在内层循环里

```java
while (size > 0) {
    TreeNode node = q.poll();
    res = node.val;   // ❌ 每 poll 一个就覆盖，最终记录的是本层最右值
    ...
}
```

假设当前层是 `[4, 5, 6]`：

| 迭代 | poll 出 | res |
|---|---|---|
| 1 | 4 | 4 ← 本想记录的 |
| 2 | 5 | 5 ← 被覆盖 |
| 3 | 6 | 6 ← 又被覆盖，循环结束 res=6 ❌ |

### 坑 2：把 peek 写在内层循环里

```java
while (size > 0) {
    res = q.peek().val;   // ❌ 每次 peek 到不同的节点
    TreeNode node = q.poll();
    ...
}
```

同一个 bug 换了个地方——每 poll 一次队首就变了，peek 到的是剩余节点中的第一个。

### 坑 3：if-else 冗余枚举

```java
if (node.left != null && node.right != null) {
    q.offer(node.left);
    q.offer(node.right);
} else if (node.left != null && node.right == null) {
    q.offer(node.left);
} else if (node.right != null && node.left == null) {
    q.offer(node.right);
}
```

左孩子存在就入队左，右孩子存在就入队右——**两件事独立**，不需要用 `else if` 组合判断：

```java
if (node.left != null) q.offer(node.left);
if (node.right != null) q.offer(node.right);
```

## 总结

1. **last-wins 模式**：不判断"是不是最后一层"，每层都记录一次最左值，让循环的自然结束替你做判断
2. **层级别 vs 节点级别的分界线**：`int size = q.size()` 到内层循环开始之间，是做"整层操作"的唯一窗口
3. **`peek()` 的时机**：必须在内层循环**开始之前**调用，此刻队首才是这一层最左节点
4. **`peek()` vs `poll()`**：想"偷看"队首信息但不想移除时用 `peek()`
5. **BFS 模板化写作**：

```java
while (!q.isEmpty()) {
    int size = q.size();
    // ★ 层级别操作（记录本层最左值、初始化本层统计量）

    for (int i = 0; i < size; i++) {
        TreeNode node = q.poll();
        // ★ 节点级别操作
        if (node.left != null) q.offer(node.left);
        if (node.right != null) q.offer(node.right);
    }

    // ★ 层级别收尾（本层统计结果加到答案里）
}
```

6. **Debug 技巧**：代码"怎么改都不对"时，在关键行加 `System.out.println` 打印中间状态，能快速定位"代码逻辑 vs 脑内意图"的偏差
