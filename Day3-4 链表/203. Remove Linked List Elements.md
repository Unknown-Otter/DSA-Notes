# 203. Remove Linked List Elements

> [LeetCode 203](https://leetcode.com/problems/remove-linked-list-elements/)

## 思路：Dummy Node + 遍历删除

- 用 **dummy node** 指向 head，统一处理头节点也可能被删除的情况
- `curr` 从 dummy 开始，每次检查 `curr.next`
- 如果 `curr.next.val == val`，跳过该节点；否则 `curr` 前进

## 我最初的错误版本

```java
class Solution {
    public ListNode removeElements(ListNode head, int val) {
        if (head == null) return null;

        ListNode dummy = new ListNode(-1);
        dummy.next = head;

        ListNode curr = head; // ❌ 错误1：从head开始，漏检查头节点

        while (curr.next != null) {
            if (curr.next.val == val) {
                curr.next = curr.next.next;
            }
            curr = curr.next; // ❌ 错误2：删除后也前进了，会跳过节点/空指针
        }

        return dummy; // ❌ 错误3：返回了dummy本身，而不是dummy.next
    }
}
```

### 三个错误

1. **`curr` 从 `head` 开始** → 第一次检查的是 `head.next`，如果 `head.val == val`，头节点永远不会被删除
2. **删除后没有 `else`，直接 `curr = curr.next`** → 删除节点后还往前走，可能跳过连续相同值的节点，也可能在删除尾节点后 `curr.next` 为 `null` 导致空指针
3. **返回 `dummy` 而非 `dummy.next`** → `dummy` 是哨兵节点，不属于链表本身

## 正确版本

```java
class Solution {
    public ListNode removeElements(ListNode head, int val) {
        if (head == null) return null;

        ListNode dummy = new ListNode(-1);
        dummy.next = head;

        ListNode curr = dummy; // ✅ 从dummy开始

        while (curr.next != null) {
            if (curr.next.val == val) {
                curr.next = curr.next.next; // 删除，curr不动
            } else {
                curr = curr.next; // 没删才前进
            }
        }

        return dummy.next; // ✅ 返回真正的头节点
    }
}
```

## 总结

1. **Dummy node 的 `curr` 要从 `dummy` 开始**，这样 `curr.next` 第一次就能检查到 `head`
2. **删除节点后 `curr` 不前进**，因为新的 `curr.next` 还没检查过，可能也要删
3. **返回 `dummy.next`**，dummy 只是辅助节点
