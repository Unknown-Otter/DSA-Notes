# 24. Swap Nodes in Pairs

> [LeetCode 24](https://leetcode.com/problems/swap-nodes-in-pairs/)

给定一个链表，两两交换其中相邻的节点，并返回交换后链表的头节点。**不能只是单纯地改变节点内部的值**，而是需要实际进行节点交换。

示例：`1 -> 2 -> 3 -> 4`  →  `2 -> 1 -> 4 -> 3`

---

## 一、初学常见误区：变量应该在哪里声明？

写这道题时，很容易写出这样的开头：

```java
ListNode dummy = new ListNode(-1);
ListNode curr = dummy;
ListNode first = head;
ListNode second = head.next;
ListNode temp = head.next.next;

while (curr.next != null && curr.next.next != null) {
    // ???
}
```

**问题在哪？** `first`、`second`、`temp` 这些指针只在**第一对节点**的位置上。但 swap pairs 需要处理**每一对**节点，所以这些指针必须在循环里"前进"到下一对的位置。如果只在循环外赋值一次，循环里就永远在原地打转。

### 关键理解

> **指针变量 ≠ 节点位置固定。** `first`、`second` 这种"角色"指针应该在循环里**每次重新指定**，因为每一轮循环它们指向的是不同的节点。

可以把 `cur`（或 `prev`）想象成一个"游标"，它每次循环都站在"我即将要交换的两个节点"的前面，循环结束时再往前走两步。

---

## 二、核心思路

链表两两交换的关键有两点：

1. **使用虚拟头节点（dummy）**：因为头节点本身也会被交换，使用 dummy 可以让所有节点的处理逻辑保持一致，不需要单独处理 head。
2. **指针操作的顺序**：每次交换涉及三个指针的修改，顺序错了会丢节点。判断标准是——**在覆盖某个 `.next` 之前，确保它原来指向的节点还能被找到**。

---

## 三、关键概念：指针赋值即断开

这是链表题里最容易让人困惑的地方。下面把它彻底讲清楚。

### `.next` 是"槽位"，不是"连接线"

很多初学者会把链表想象成节点之间有一根**线**连着，所以会担心："我加了一根新线，旧线还在吗？要不要剪掉？"

实际上，`node.next` 是节点对象里的**一个字段（一个槽位）**，这个槽位里只能放**一个**引用。给它赋新值，旧的指向**自动消失**。

```
Node 1 的内存长这样：
┌─────────┐
│ val: 1  │
│ next: ? │ ← 这是一个槽位，只能放一个引用
└─────────┘
```

### 类比：白板上的便利贴

想象 `node.next` 是一张贴在节点上的**便利贴**，上面写着"下一个节点的地址"。

- 撕下旧便利贴、贴上新便利贴 = 赋值
- 一个位置同时只能贴**一张**便利贴
- 旧的被撕下来就**直接扔了**，不会有两张同时存在

所以根本不存在"残留的旧指针"这个问题，**不需要手动断开**。

### 节点什么时候才算"丢"？

只有当一个节点**没有任何 `.next` 指向它，也没有变量名指向它**的时候，它才算"丢了"（在 Java 里会被垃圾回收）。

---

## 四、逐步走一遍：指针变化过程

以 `dummy -> 1 -> 2 -> 3 -> 4` 为例，假设 `cur = dummy`，`first = 1`，`second = 2`。

### 初始状态

```
Node 1: { val: 1, next: → Node 2 }
Node 2: { val: 2, next: → Node 3 }
```

注意：`second.next = 3`，**第三个节点 3 已经被 `second.next` 引用着，没丢**。

### 步骤一：`first.next = second.next;`

含义：让 1 的 next 指向 3。

```
Node 1: { val: 1, next: → Node 3 }   ← 旧的"指向 Node 2"被覆盖了，自动消失
Node 2: { val: 2, next: → Node 3 }   ← Node 2 没动
```

我们**修改的是 `first.next`**，并没有动 `second.next`。所以 2 还是指着 3 的，但这不影响——重要的是 1 现在指向 3 了，3 没丢。

### 步骤二：`second.next = first;`

让 2 的 next 指向 1：

```
Node 1: { val: 1, next: → Node 3 }
Node 2: { val: 2, next: → Node 1 }   ← Node 2 的 next 从 3 变成 1
```

现在结构是 `2 -> 1 -> 3`，1 和 2 之间不存在"双向连接"或"残留的旧连接"。

### 步骤三：`cur.next = second;`

让 dummy 指向 2：

```
dummy -> 2 -> 1 -> 3 -> 4
```

完成交换 ✓

### 顺序为什么不能乱？

如果先写 `second.next = first;`，那 `second.next` 就不再是 3 了，3 就**真的丢了**（因为没有任何指针指向它）。

所以这三行的顺序必须是：

1. `first.next = second.next;` ← **必须先做，趁 second.next 还指着 3，把 3 "救"到 first 后面**
2. `second.next = first;`
3. `cur.next = second;`

---

## 五、判断口诀

> **赋值即断开。你不需要"断"，你只需要"接对"。**

每次写 `a.next = b;` 之前，问自己一句：

> "我现在要覆盖的这个 `.next`，里面原来存的那个节点，还有没有别的方式能找到它？"

- 如果有 → 直接赋值，不用 temp
- 如果没有 → 必须先用 temp 存起来

### 什么时候才需要 temp？

当你**没办法用一句话同时完成"读取旧值"和"写入新值"**的时候。比如经典的反转链表：

```java
ListNode next = curr.next;  // 必须先存，因为下一句就要覆盖 curr.next
curr.next = prev;
prev = curr;
curr = next;
```

而在 swap pairs 里，我们交换的两个节点 `first` 和 `second` 本来就用变量存着了，第三个节点可以通过 `second.next` 临时访问到，所以**不一定需要额外的 temp**（取决于你的赋值顺序）。

---

## 六、方法一：迭代 + temp 暂存

最直观的写法，用 `temp` 提前保存"第二对的起点"，避免后续操作覆盖后丢失。

```java
class Solution {
    public ListNode swapPairs(ListNode head) {
        ListNode dummyhead = new ListNode(-1); // 虚拟头节点
        dummyhead.next = head;
        ListNode cur = dummyhead;
        ListNode temp;        // 保存两个节点后面的节点
        ListNode firstnode;   // 待交换的第一个节点
        ListNode secondnode;  // 待交换的第二个节点

        while (cur.next != null && cur.next.next != null) {
            temp = cur.next.next.next;
            firstnode = cur.next;
            secondnode = cur.next.next;

            cur.next = secondnode;        // 步骤一：cur 指向 second
            secondnode.next = firstnode;  // 步骤二：second 指向 first
            firstnode.next = temp;        // 步骤三：first 指向 temp（第三个节点）

            cur = firstnode; // cur 移动到下一对前面
        }
        return dummyhead.next;
    }
}
```

**为什么需要 temp？** 因为执行步骤二 `secondnode.next = firstnode` 之后，`secondnode.next` 就不再指向第三个节点了。如果不提前存住第三个节点，步骤三就找不到它了。

---

## 七、方法二：迭代 + 调整顺序（无需 temp）

把步骤二和步骤三的顺序调换一下，就可以省掉 `temp` 变量。

```java
public ListNode swapPairs(ListNode head) {
    ListNode dummy = new ListNode(0, head);
    ListNode cur = dummy;
    while (cur.next != null && cur.next.next != null) {
        ListNode node1 = cur.next;       // 第 1 个节点
        ListNode node2 = cur.next.next;  // 第 2 个节点

        cur.next = node2;        // 步骤 1：cur 指向 node2
        node1.next = node2.next; // 步骤 3：node1 接住第三个节点
        node2.next = node1;      // 步骤 2：node2 指向 node1

        cur = cur.next.next;     // cur 前进到下一对前面
    }
    return dummy.next;
}
```

**关键理解**：先执行 `node1.next = node2.next`，趁 `node2.next` 还指着第三个节点时把它"接"到 `node1` 后面，然后再让 `node2` 指向 `node1`。这样就不需要额外变量了。

### 走一遍 `1 -> 2 -> 3 -> 4`

- **第一轮**：`cur=dummy`，`node1=1`，`node2=2`
  - `cur.next = 2` → `dummy -> 2`
  - `node1.next = 3` → `1 -> 3`
  - `node2.next = 1` → `2 -> 1`
  - 整体：`dummy -> 2 -> 1 -> 3 -> 4`，`cur` 前进到 1
- **第二轮**：`cur=1`，`node1=3`，`node2=4`
  - 交换后：`dummy -> 2 -> 1 -> 4 -> 3`，`cur` 前进到 3
- **第三轮**：`cur.next == null`，退出

---

## 八、方法三：递归

递归版本最简洁，思路是：**只处理前两个节点的交换，剩下的交给递归函数。**

```java
class Solution {
    public ListNode swapPairs(ListNode head) {
        // base case：节点不足两个，直接返回
        if (head == null || head.next == null) return head;

        ListNode next = head.next;               // 第二个节点
        ListNode newNode = swapPairs(next.next); // 递归处理后面的部分

        // 交换前两个节点
        next.next = head;
        head.next = newNode;

        return next; // next 是交换后的新头节点
    }
}
```

**递归逻辑拆解**：

1. **base case**：当前剩余节点不足两个时，无需交换，直接返回。
2. **递归调用**：先处理第三个节点开始的子链表，得到处理完的子链表头 `newNode`。
3. **当前层交换**：让 `next`（原第二个节点）指向 `head`（原第一个节点），再让 `head` 指向递归返回的 `newNode`。
4. **返回新头**：`next` 现在是交换后的头节点。

---

## 九、易错点总结

1. **变量声明位置**：`first`、`second` 这类"角色指针"必须在 while 循环**内部**声明/赋值，因为每轮循环它们指向不同节点。在循环外赋值会导致永远只处理第一对。
2. **指针赋值即断开**：写 `a.next = b` 不需要"手动断开"旧连接，新值会自动覆盖旧值。`.next` 是一个槽位，同一时刻只能存一个引用。
3. **覆盖前先确认旧值是否还能找到**：每次写 `a.next = ...` 之前问自己：原来 `a.next` 指向的节点，还有没有别的方式访问到？如果没有，要么提前 temp 存起来，要么调整赋值顺序。
4. **while 条件顺序**：`cur.next != null && cur.next.next != null` 的顺序不能反，否则 `cur.next` 为 null 时访问 `.next.next` 会报空指针。Java 的短路求值保证了前面为 false 时后面不会执行。
5. **dummy 节点**：链表题中只要头节点可能被修改，就用 dummy 简化逻辑，避免单独处理头节点的特殊情况。

---

## 十、三种方法对比

1. **方法一（temp 暂存）**：最直观，适合初学者理解每一步在做什么。
2. **方法二（调整顺序）**：代码更精简，体现了"指针操作顺序"的核心理解。
3. **方法三（递归）**：代码最短，逻辑最优雅，但递归调用栈带来额外空间开销 O(n)。

| 方法 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 迭代 + temp | O(n) | O(1) |
| 迭代 + 顺序调整 | O(n) | O(1) |
| 递归 | O(n) | O(n) |
