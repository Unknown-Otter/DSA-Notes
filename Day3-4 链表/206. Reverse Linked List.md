# 206. Reverse Linked List

> [LeetCode 206](https://leetcode.com/problems/reverse-linked-list/)

## 解题思路

单链表只能从左往右走，**不能往左回退**。所以"先走到尾节点再往回反转"的思路在单链表里行不通——节点没有指向前驱的指针，强行实现需要每次从 head 重新遍历，时间复杂度退化为 O(n²)。

正确做法：**从 head 开始顺序遍历，每走到一个节点就把它的 next 指针反向指向前一个节点**。

## 三个指针的角色

需要三个指针配合完成"边走边翻转"：

- **prev**：当前节点的前一个节点，初始为 `null`（因为原头节点反转后要指向 null）
- **curr**：当前正在处理的节点，初始为 `head`
- **temp**：临时保存 `curr.next`，因为马上要修改 `curr.next`，不先存下来后面的链表就丢了

## 核心四步操作

```java
temp = curr.next;   // 1. 先存下一个节点(否则改 curr.next 后就丢了)
curr.next = prev;   // 2. 反转当前节点的箭头
prev = curr;        // 3. prev 前进一步
curr = temp;        // 4. curr 前进一步
```

**顺序至关重要**：第一步必须先保存 `curr.next`，因为第二步会覆盖它。

## 完整代码

```java
class Solution {
    public ListNode reverseList(ListNode head) {
        ListNode prev = null;
        ListNode curr = head;
        ListNode temp = null;

        while (curr != null) {
            temp = curr.next;
            curr.next = prev;
            prev = curr;
            curr = temp;
        }
        return prev;
    }
}
```

## 关键概念辨析

### 1. 移动指针 vs. 修改链接

这是链表题最容易混淆的点，本质区别在于**改的是变量本身，还是节点内部的字段**。

**`prev = curr`** —— **移动指针**（搬贴纸）

- 改变的是 `prev` 这个变量本身指向哪里
- 节点内部数据完全没变，链表结构也没变
- 只是"我现在看哪个节点"发生了变化

**`prev.next = curr`** —— **修改链接**（改节点内部）

- `prev` 这个变量没动，还指着原来的节点
- 但是钻进了那个节点内部，把它的 `next` 字段改成了 curr
- **链表的结构真的被修改了**

**判断口诀**：写每一行赋值时问自己——"我这一步是在搬贴纸，还是在改节点？"

回看反转链表的四步，会发现**真正修改链表结构的只有 `curr.next = prev` 这一行**，其他三行都只是在搬贴纸、更新观察位置。这也是为什么顺序不能乱——一旦改了 `curr.next`，原来的后续节点就丢了，所以必须先用 temp 存住。

### 2. 循环条件：`curr != null` vs. `curr.next != null`

这两个条件**跳出循环时 curr 的位置不一样**，不能随便互换。

| 循环条件 | 跳出时 curr 在哪 |
|---|---|
| `while (curr != null)` | curr 是 null（走过头了，掉到链表外） |
| `while (curr.next != null)` | curr 停在最后一个真实节点上 |

**怎么选？看循环体里最深的访问层级**：

| 循环体里最深的访问 | 需要的循环条件 |
|---|---|
| `curr` 或 `curr.val` 或 `curr.next`(只读) | `curr != null` |
| `curr.next.val` 或 `curr.next.next` | `curr != null && curr.next != null` |
| `curr.next.next.val` | 还要再多加一层保护 |

**规则**：循环条件要保证你"敢访问的最深一层"不是 null。每多穿一层 `.next`，条件里就要多加一道保护。

**注意**：读取一个 null 值是合法的（比如 `temp = curr.next`，curr.next 是 null 也没事，temp 就变成 null）。出错的是**对 null 解引用**——比如 `curr.next.val`，试图进入一个不存在的节点内部去拿东西，才会 NullPointerException。

### 3. 为什么 return prev 而不是 curr

追踪 `1→2→3` 的执行过程：

```
初始:        prev=null,  curr=1,     temp=null
第1轮结束:   prev=1,     curr=2,     temp=2
第2轮结束:   prev=2,     curr=3,     temp=3
第3轮结束:   prev=3,     curr=null,  temp=null  ← 跳出循环
```

跳出循环的瞬间：
- **curr = null**（条件 `curr != null` 决定的，跳出时 curr 必为 null）
- **prev = 3**（正好停在原来的尾节点，也就是反转后的新头）

每一轮循环，**prev 都指向"目前为止已经反转好的那段链表的头"**：

```
第1轮后: 已反转部分 [1→null],     prev → 1 ✓
第2轮后: 已反转部分 [2→1→null],   prev → 2 ✓
第3轮后: 已反转部分 [3→2→1→null], prev → 3 ✓
```

循环结束 = 所有节点都处理完 = prev 指向整个反转后链表的新头 → `return prev`

### 4. 这道题为什么不需要 dummy 节点

**dummy 节点的作用**：当头节点本身可能变化或难以处理时，加一个"假头"作为固定锚点，最后 `return dummy.next`。常用于：
- Remove Linked List Elements（删除节点，可能删到头）
- Merge Two Sorted Lists（新链表的头不确定）

**反转链表不需要 dummy**，因为新头节点就是原来的尾节点，而 prev 在循环过程中天然就会停在那个位置上。我们不需要固定锚点，prev 本身就是答案。

## 总结

1. **单链表只能从左往右走**，反转必须顺序遍历 + 边走边翻转
2. **prev、curr、temp 三指针配合**，四步操作的顺序不能乱：先存后继 → 反转箭头 → prev 前进 → curr 前进
3. **区分"移动指针"和"修改链接"**：`prev = curr` 是搬贴纸，`prev.next = curr` 是改节点内部
4. **循环条件根据循环体里最深的访问层级来定**：每多穿一层 `.next`，条件里就多一道保护
5. **return prev** 而不是 curr：循环结束时 curr 必为 null，prev 才指向新头
6. **无需 dummy**：prev 本身就是新头节点的天然指针
