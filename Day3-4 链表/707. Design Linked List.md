# 707. Design Linked List — 错误复盘与深入理解

> [LeetCode 707](https://leetcode.com/problems/design-linked-list/)

## 核心思维误区：`=` 赋值 vs `.next` 接线

链表操作只有两种动作，但我反复搞混：

- **等号左边没有 `.next`** → **换牌子**：变量指向别的节点，链表结构不变
- **等号左边有 `.next`** → **接线**：修改节点的指针，链表结构改变

```java
// 换牌子：curr 这个变量从 A 移到 B，链表没变
curr = curr.next;

// 接线：A 节点的 next 指针从 B 改指向 C，链表结构变了
curr.next = someNode;
```

**我犯的错：**

```java
// ❌ 我写的：想把 newNode 接入链表
newNode = curr.next;
// 实际效果：newNode 变量指向了 curr.next，new 出来的新节点丢了，链表没有任何变化

// ✅ 正确：修改 newNode 的 next 指针
newNode.next = curr.next;
// 实际效果：新节点的 next 指向了 curr 的下一个节点，新节点被接入链表
```

**我的误解：** 我以为 `newNode = curr.next` 是"让 newNode 指向 curr 的下一个"，逻辑上没错，但这只是让**变量**换了指向，**新节点本身没有被连进链表**。要改链表结构，必须改 `.next`。

## addAtTail：遍历结束后 curr 停在哪

**我写的错误代码：**

```java
public void addAtTail(int val) {
    ListNode newNode = new ListNode(val);
    while (curr.next != null) {
        curr = curr.next;
    }
    newNode = curr.next.next; // ❌ 两个问题
}
```

**问题1：** 循环条件 `curr.next != null`，结束时 `curr.next` 就是 `null`，访问 `curr.next.next` 会抛 `NullPointerException`。

**问题2：** 赋值方向反了，应该是把新节点接到尾部，不是给 newNode 重新赋值。

**修正：**

```java
public void addAtTail(int val) {
    ListNode newNode = new ListNode(val);
    ListNode curr = dummy;
    while (curr.next != null) {
        curr = curr.next;
    }
    curr.next = newNode; // 尾节点的 next 接上新节点
    size++;
}
```

**教训：** 写完循环后，先问自己"循环结束后 `curr` 停在哪个节点"，用具体例子（如 `1→2→3→null`）走一遍。

## addAtIndex：插入顺序——先接后面，再接前面

**我写的错误代码：**

```java
public void addAtIndex(int index, int val) {
    ListNode newNode = new ListNode(val);
    ListNode curr = dummy;
    for (int i = 0; i < index + 1; i++) { // ❌ 走了 index+1 步，走过头了
        curr = curr.next;
    }
    curr.next = newNode;       // ❌ 先接前面
    newNode.next = curr.next;  // ❌ 此时 curr.next 已经是 newNode，原来的后续节点丢了
}
```

**问题1：** 插入需要停在目标位置的**前驱**，从 dummy 出发走 `index` 步，不是 `index + 1` 步。

**问题2：** 顺序反了。先执行 `curr.next = newNode` 后，`curr.next` 已经变成 newNode，原来的后续节点没有任何变量引用，就丢了。

**为什么我会写反：** 我按人的直觉思考——"把新节点放到 curr 后面"，第一步自然是 `curr.next = newNode`。但这是**叙事顺序**，不是**安全顺序**。指针操作必须**先保存即将失去访问权的节点**。

**修正：**

```java
public void addAtIndex(int index, int val) {
    if (index < 0 || index > size) return;
    ListNode newNode = new ListNode(val);
    ListNode curr = dummy;
    for (int i = 0; i < index; i++) { // 走 index 步，停在前驱
        curr = curr.next;
    }
    newNode.next = curr.next; // 先接后面（保存即将断开的节点）
    curr.next = newNode;      // 再接前面
    size++;
}
```

**口诀：先接后面，再接前面。**

## deleteAtIndex：循环条件又写多了

**我写的错误代码：**

```java
for (int i = 0; i <= index + 1; i++) { // ❌ 走了 index+2 步
    curr = curr.next;
}
```

**修正：** 删除也需要停在前驱，走 `index` 步：

```java
public void deleteAtIndex(int index) {
    if (index < 0 || index >= size) return;
    ListNode curr = dummy;
    for (int i = 0; i < index; i++) {
        curr = curr.next;
    }
    curr.next = curr.next.next; // 跳过目标节点
    size--;
}
```

## 其他语法错误

- **`while` 写成了 `for` 的格式：** `while(int i=0; i<index; i++)` 是语法错误，应该用 `for`
- **多处漏了 `ListNode curr = dummy;` 声明**
- **多处漏了 `size++` / `size--`**
- **多处漏了边界检查**（index 越界）

## dummy 节点与 index 的关系

```
dummy → A → B → C → null
         0    1    2
```

- **dummy 不是 index 0**，它是哨兵节点，不参与编号
- **index 永远从 0 开始**，跟有没有 dummy 无关
- dummy 的意义：让 index 0 也有前驱，不需要特殊处理

**从 dummy 出发的步数规律：**

| 操作 | 需要停在 | 从 dummy 走的步数 |
|------|---------|-----------------|
| **get(index)** | index 本身 | index + 1 |
| **addAtIndex** | index 的前驱 | index |
| **deleteAtIndex** | index 的前驱 | index |

**不要背 +1 还是不 +1**，每次想"我要停在哪，从 dummy 走几步能到"。

## 深入理解：Java 中的 next 到底是什么

`ListNode next` 看起来类型是 Node，但它存的不是一个完整的节点对象，而是**下一个节点的内存地址（引用）**。

```java
class ListNode {
    int val;        // 存整数值
    ListNode next;  // 存下一个节点的地址
}
```

Java 所有对象变量天然就是引用（地址），不需要像 C 语言那样写 `*`：

```c
// C 语言：显式标记指针
ListNode *next;

// Java：对象变量自动就是引用
ListNode next;
```

**节点不存自己的地址**，因为你能访问到它，就说明你已经有它的地址了（比如通过前一个节点的 `next`）。每个节点只需要存下一个节点的地址，这就形成了链条。

## 总结：链表操作三件事

1. **遍历（换牌子）：** `curr = curr.next;`
2. **插入（接两根线）：** `newNode.next = curr.next;` → `curr.next = newNode;`
3. **删除（跳过节点）：** `curr.next = curr.next.next;`

**每次写之前问自己：**
1. 循环结束后 `curr` 停在哪？
2. 我要修改的是哪个节点的 `.next`？
3. 我即将断开的指针，对面的节点还有别的引用吗？没有就先保存。
