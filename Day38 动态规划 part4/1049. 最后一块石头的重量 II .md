# 1049. Last Stone Weight II

> [LeetCode 1049 - Last Stone Weight II](https://leetcode.com/problems/last-stone-weight-ii/)

## 题目转化

把每块石头分成两堆 A 和 B，让 `|sum(A) - sum(B)|` 最小。

设 `total = sum(stones)`，`S1 = sum(A)`，则 `S2 = total - S1`，差 = `|total - 2*S1|`。

**要让差最小，S1 必须尽可能接近 total/2。**

所以题目转化为 **01 背包问题**：在容量 `target = total // 2` 的背包里，挑选石头使总重量尽可能接近 target。

---

## 解法一：二维 DP（int + max，Java）

### 状态定义

`dp[i][j]` = 从前 `i+1` 块石头中挑选，**容量为 j 时能装下的最大重量**。

### 状态转移

对于每块石头 `stones[i]`，有两种选择：

1. **不放** `stones[i]`：`dp[i][j] = dp[i-1][j]`
2. **放** `stones[i]`：`dp[i][j] = dp[i-1][j - stones[i]] + stones[i]`（前提是 `j >= stones[i]`）

取两者最大值。

### 代码

```java
class Solution {
    public int lastStoneWeightII(int[] stones) {
        int sum = 0;
        for (int stone : stones) {
            sum += stone;
        }
        int target = sum / 2;

        // 初始化：第 0 行表示只考虑 stones[0]
        int[][] dp = new int[stones.length][target + 1];
        for (int j = stones[0]; j <= target; j++) {
            dp[0][j] = stones[0];
        }

        for (int i = 1; i < stones.length; i++) {
            for (int j = 1; j <= target; j++) {
                if (j >= stones[i]) {
                    // 不放 stones[i] vs 放 stones[i]
                    dp[i][j] = Math.max(dp[i-1][j], dp[i-1][j - stones[i]] + stones[i]);
                } else {
                    dp[i][j] = dp[i-1][j];
                }
            }
        }

        return sum - 2 * dp[stones.length - 1][target];
    }
}
```

### 复杂度

- 时间：`O(n * target)`
- 空间：`O(n * target)`

## 解法二：一维 DP（int + max，滚动数组优化）

### 核心观察

二维 DP 中，**第 i 行只依赖第 i-1 行**，更早的行用不上。所以可以用**滚动数组**压成一维。

### 二维 → 一维的两个关键

1. **`dp[i-1][j]` 和 `dp[i-1][j-w]` 都变成 `dp[j]` 和 `dp[j-w]`**——原地覆盖
2. **j 必须倒序遍历**——保证读到的 `dp[j-w]` 还是"上一行"的旧值

### 为什么倒序？

正序遍历时，`dp[j - stones[i]]` 在前面循环中已经被更新过，相当于 stones[i] **被使用多次**，退化成完全背包。

倒序遍历时，`dp[j - stones[i]]` 还没被这一轮触碰过，保留着 `dp[i-1][j - stones[i]]` 的语义——保证 **0/1 背包：每块石头只用一次**。

### 代码

```java
class Solution {
    public int lastStoneWeightII(int[] stones) {
        int sum = 0;
        for (int stone : stones) sum += stone;
        int target = sum / 2;

        int[] dp = new int[target + 1];

        for (int i = 0; i < stones.length; i++) {
            for (int j = target; j >= stones[i]; j--) {   // 倒序！
                dp[j] = Math.max(dp[j], dp[j - stones[i]] + stones[i]);
            }
        }

        return sum - 2 * dp[target];
    }
}
```

### Python 等价写法

```python
class Solution:
    def lastStoneWeightII(self, stones: List[int]) -> int:
        total = sum(stones)
        target = total // 2

        dp = [0] * (target + 1)

        for i in range(len(stones)):
            for j in range(target, stones[i] - 1, -1):   # 倒序
                dp[j] = max(dp[j], dp[j - stones[i]] + stones[i])

        return total - 2 * dp[target]
```

### 复杂度

- 时间：`O(n * target)`
- 空间：`O(target)`

## 解法三：bool + or 写法（Python 社区常见模板）

### 状态定义

`dp[i][j]` = 前 i 个石头能否**恰好凑出**和为 j（True/False）。

### 与 int + max 的本质区别

| 维度 | int + max | bool + or |
|---|---|---|
| dp 存什么 | "≤ j 的最大可达和" | "能否恰好凑出 j" |
| 合并算子 | `max` | `or` |
| dp[target] 含义 | 直接就是答案的最大重量 | 只表示 target 本身能否凑出 |
| 读答案方式 | 直接读 dp[target] | 需要**倒着扫**找最大的 True |

### 核心思路

**不算价值，填表时只判断"能不能凑成"。填完后循环 dp 表，找到第一个 True 就是答案。**

### 状态转移

```python
dp[i][j] = dp[i-1][j] or dp[i-1][j - stones[i-1]]
#          不放石头 i      或   放石头 i
```

**`or` 的语义**：只要有一种选法能凑成 j 就标 True，不关心怎么凑、用了哪些石头。

### 代码

```python
class Solution:
    def lastStoneWeightII(self, stones: List[int]) -> int:
        total_sum = sum(stones)
        target = total_sum // 2

        # dp[i][j] = 前 i 个石头能否恰好凑出和为 j
        dp = [[False] * (target + 1) for _ in range(len(stones) + 1)]

        # 初始化：要凑出和 0，只要一个石头都不选即可
        for i in range(len(stones) + 1):
            dp[i][0] = True

        for i in range(1, len(stones) + 1):
            for j in range(1, target + 1):
                if stones[i-1] > j:
                    dp[i][j] = dp[i-1][j]
                else:
                    dp[i][j] = dp[i-1][j] or dp[i-1][j - stones[i-1]]

        # 从 target 倒着找：第一个 True 的 i 就是"≤ target 的最大可凑和"
        for i in range(target, -1, -1):
            if dp[len(stones)][i]:
                return total_sum - 2 * i

        return 0   # 死代码，dp[i][0] = True 保证一定能找到
```

### 尾巴循环的逻辑

```python
for i in range(target, -1, -1):
    if dp[len(stones)][i]:
        return total_sum - 2 * i
```

**为什么从 target 倒着扫？**

- dp 表的最后一行 `dp[n]` 告诉我们：每个 j（从 0 到 target），能否恰好凑出
- 我们要找**最接近 target 的、能凑出的最大 j**
- 倒序扫描，第一个找到的 True 就是最优解

**举例**：`stones = [2,7,4,1,8,1]`，`total = 23`，`target = 11`

如果 `dp[6][11] = False`，`dp[6][10] = True`，则：
- 倒扫到 i=10 时找到 True
- 返回 `23 - 2*10 = 3`
- 两堆分别是 10 和 13，差 3

### 一维版本

```python
class Solution:
    def lastStoneWeightII(self, stones: List[int]) -> int:
        total = sum(stones)
        target = total // 2

        dp = [False] * (target + 1)
        dp[0] = True

        for stone in stones:
            for j in range(target, stone - 1, -1):   # 倒序
                dp[j] = dp[j] or dp[j - stone]

        for j in range(target, -1, -1):
            if dp[j]:
                return total - 2 * j
        return 0
```

## 三种写法对比总结

| 写法 | dp 类型 | 算子 | 代码长度 | 适用场景 |
|---|---|---|---|---|
| Java int + max（二维） | int | `max` | 较长 | 学习时理解状态转移最清晰 |
| Java/Python int + max（一维） | int | `max` | 最短 | 通用最优解 |
| Python bool + or | bool | `or` | 中等 | 复用 416 模板，思维朴素 |

---

## 核心 Takeaway

1. **题目转化**：差最小 ⟺ 找最接近 `total/2` 的子集和 ⟺ 01 背包
2. **二维 DP**：`dp[i][j]` 依赖 `dp[i-1][j]` 和 `dp[i-1][j-w]`，**两种选择取 max**
3. **二维 → 一维**：滚动数组，**j 必须倒序遍历**才能保持 0/1 背包语义
4. **bool 写法**：把"最优化"拆成"对每个 j 做判定" + "倒扫找最大 True"
5. **判断框架**（01 背包通用）：
    - 求最大/最小值 → `int + max/min`
    - 求能否凑出 → `bool + or`
    - 求方案数 → `int + 加法`
6. **bool 写法的限制**：依赖"重量是整数"的离散特性，重量为小数时不适用
