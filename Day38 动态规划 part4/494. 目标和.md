# 494. 目标和

> [LeetCode 链接](https://leetcode.cn/problems/target-sum/)

给定一个非负整数数组 `nums` 和一个目标整数 `target`。对数组中每个整数前面添加 `+` 或 `-`，构造一个表达式。返回可以构造出表达式等于 `target` 的不同方法数量。

**示例**：`nums = [1,1,1,1,1], target = 3` → 输出 `5`

---

## 一、问题转化（核心思路）

把添加 `+` 的数字归为 **left 集合**，添加 `-` 的归为 **right 集合**：

```
left - right = target
left + right = sum   (固定值)
```

两式相加消去 right：

```
left = (target + sum) / 2
```

由于 `sum` 和 `target` 都固定，`left` 也是固定的。问题转化为：

> **在 nums 中选若干个数，使其和恰好为 `left`，有几种方法？**

这就是一个标准的 **01 背包求方法数** 问题，背包容量 = `left`。

---

## 二、边界判断

转化前必须处理两种无解情况：

```java
if (sum < Math.abs(target)) return 0;       // target 绝对值超过 sum
if ((sum + target) % 2 != 0) return 0;       // 除以 2 得不到整数
```

⚠️ **注意**：是 `sum < Math.abs(target)`，不是 `<=`。当 `sum == |target|` 时仍有 1 种方法（全部取同号）。

---

## 三、二维 DP 推导

### 1. dp 数组定义

`dp[i][j]`：使用下标 `[0, i]` 的物品，**装满**容量为 `j` 的背包，有 `dp[i][j]` 种方法。

注意是「**方法数**」，不是「最大价值」。

### 2. 递推公式

对物品 `i` 做「**用 / 不用**」二选一：

- **不用物品 i**：方法数 = `dp[i-1][j]`
- **用物品 i**：占掉 `nums[i]` 的容量，剩下用前面物品装满 → `dp[i-1][j - nums[i]]`

两种情况互不重叠，相加：

```java
if (nums[i] > j) {
    dp[i][j] = dp[i-1][j];                          // 装不下，只能不放
} else {
    dp[i][j] = dp[i-1][j] + dp[i-1][j - nums[i]];   // 不放 + 放
}
```

**关键理解**：`dp[i-1][j - nums[i]]` 不是"物品 i 自己的解法"，而是"用了物品 i 之后，剩下空间交给前面物品有几种凑法"。

### 3. 初始化

**最左列 `dp[i][0]`**：背包容量为 0，"什么都不放"是 1 种合法方案。

**特殊处理 0**：若 `nums` 中有 0，每个 0 可选可不选，让方法数翻倍：

```java
int numZeros = 0;
for (int i = 0; i < nums.length; i++) {
    if (nums[i] == 0) numZeros++;
    dp[i][0] = (int) Math.pow(2, numZeros);
}
```

**最上行 `dp[0][j]`**：只用物品 0，唯一能装满的容量是 `nums[0]`：

```java
if (nums[0] != 0 && nums[0] <= left) {
    dp[0][nums[0]] = 1;
}
```

### 4. 遍历顺序

**从上到下，从左到右**。因为 `dp[i][j]` 依赖正上方和左上方的格子。

```java
for (int i = 1; i < n; i++) {
    for (int j = 1; j <= left; j++) {   // ← 必须 <=，包含 left 本身
        // 递推
    }
}
```

⚠️ **常见 bug**：内层写成 `j < left`，会导致最后一列没被填，返回 0。**答案存在 `dp[n-1][left]`，边界必须取到**。

### 5. 完整代码（二维）

```java
class Solution {
    public int findTargetSumWays(int[] nums, int target) {
        int sum = 0;
        for (int num : nums) sum += num;

        if (sum < Math.abs(target)) return 0;
        if ((sum + target) % 2 != 0) return 0;

        int left = (sum + target) / 2;
        int n = nums.length;
        int[][] dp = new int[n][left + 1];

        int numZeros = 0;
        for (int i = 0; i < n; i++) {
            if (nums[i] == 0) numZeros++;
            dp[i][0] = (int) Math.pow(2, numZeros);
        }

        if (nums[0] != 0 && nums[0] <= left) {
            dp[0][nums[0]] = 1;
        }

        for (int i = 1; i < n; i++) {
            for (int j = 1; j <= left; j++) {
                if (nums[i] > j) {
                    dp[i][j] = dp[i - 1][j];
                } else {
                    dp[i][j] = dp[i - 1][j] + dp[i - 1][j - nums[i]];
                }
            }
        }

        return dp[n - 1][left];
    }
}
```

## 四、一维 DP（滚动数组优化）

### 1. dp 数组定义

去掉行维度：`dp[j]` 表示装满容量 `j` 的背包有几种方法。

### 2. 递推公式

```java
dp[j] += dp[j - nums[i]];
```

📌 **这是「背包求方法数」的通用公式**，后面 518、518、377 都会用到。

### 3. 初始化

```java
dp[0] = 1;   // 装满容量 0 有 1 种方法：什么都不放
```

### 4. 遍历顺序（关键）

**外层物品，内层背包，且内层必须倒序**：

```java
for (int i = 0; i < nums.length; i++) {
    for (int j = left; j >= nums[i]; j--) {   // ← 倒序
        dp[j] += dp[j - nums[i]];
    }
}
```

**为什么倒序？** 倒序保证每个物品只被使用一次。如果正序，`dp[j - nums[i]]` 已经被本轮更新过（包含了物品 i），再用它更新 `dp[j]` 就会让物品 i 被用两次，变成完全背包。

### 5. 完整代码（一维）

```java
class Solution {
    public int findTargetSumWays(int[] nums, int target) {
        int sum = 0;
        for (int num : nums) sum += num;

        if (sum < Math.abs(target)) return 0;
        if ((sum + target) % 2 != 0) return 0;

        int left = (sum + target) / 2;
        int[] dp = new int[left + 1];
        dp[0] = 1;

        for (int i = 0; i < nums.length; i++) {
            for (int j = left; j >= nums[i]; j--) {
                dp[j] += dp[j - nums[i]];
            }
        }

        return dp[left];
    }
}
```

**复杂度**：时间 `O(n × left)`，空间 `O(left)`。

---

## 五、关键总结

1. **问题转化**：`left = (target + sum) / 2`，把"加减号组合"转成"01 背包装满方案数"。

2. **递推公式**（方法数类背包的通用形式）：
   ```
   dp[i][j] = dp[i-1][j] + dp[i-1][j - nums[i]]
              └─不放物品i─┘   └────放物品i────┘
   ```

3. **初始化两个要点**：
   - 最左列 `dp[i][0] = 1`（"空选"也算 1 种合法方案）
   - 数组含 0 时要做 `2^numZeros` 处理

4. **一维 DP 必须倒序遍历背包**，否则物品被重复使用，变成完全背包。

5. **答案存在 `dp[n-1][left]` 或 `dp[left]`**，循环边界必须包含 `left`，**别忘了 `<=`**。

6. **与回溯的关系**：39. 组合总和那种「列出所有方案」必须回溯；本题只要「数量」，DP 更高效。
