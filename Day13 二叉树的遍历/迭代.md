# 二叉树的迭代遍历（非递归）

> 用栈模拟递归，实现二叉树的前序、中序、后序遍历

## 为什么可以用栈来迭代遍历

递归的本质就是**函数调用栈**：每次递归调用都会把局部变量、参数值、返回地址压入调用栈，返回时从栈顶弹出上一层。既然递归靠的是栈，那我们自己手动维护一个栈，也一样能实现二叉树的前中后序遍历。

---

## 一、前序遍历（迭代法）

**前序顺序：中 → 左 → 右**

### 核心思路

把根节点先入栈，然后循环：每次 **pop 一个节点就处理它**，再把它的**右孩子、左孩子**依次压栈。

### 为什么入栈顺序是"右 → 左"？

栈是**后进先出（LIFO）**。我们希望下一次 pop 出的是左孩子（先处理左），所以左孩子要**后**入栈，处于栈顶。

### 代码

```java
class Solution {
    public List<Integer> preorderTraversal(TreeNode root) {
        List<Integer> result = new ArrayList<>();
        if (root == null) return result;

        Stack<TreeNode> stack = new Stack<>();
        stack.push(root);
        while (!stack.isEmpty()) {
            TreeNode node = stack.pop();        // 中
            result.add(node.val);
            if (node.right != null) stack.push(node.right);  // 右先入栈
            if (node.left != null) stack.push(node.left);    // 左后入栈（先出）
        }
        return result;
    }
}
```

### 关键点

- **访问节点**和**处理节点**顺序一致，都是"中"在最前，所以 pop 的瞬间就能记录结果
- 空节点不入栈

---

## 二、中序遍历（迭代法）

**中序顺序：左 → 中 → 右**

### 为什么不能照搬前序写法

在迭代过程中其实有两个动作：

1. **访问节点**：遍历到这个节点（指针走到它）
2. **处理节点**：把它的值放进 `result`

前序遍历时，**访问顺序 = 处理顺序 = 中左右**，所以一 pop 就能处理。

但中序遍历里，**第一个访问的是根节点（从顶往下走）**，但**第一个要处理的是最左下的节点**。访问顺序和处理顺序**不一致**，所以不能 pop 出来就处理。

### 解决办法：用指针 `cur` 辅助访问，用栈暂存节点

- **指针 `cur`**：负责**向下访问**，一路走到最左
- **栈**：暂存那些**已访问但还没处理**的节点，等左子树处理完再回头处理它们

### 代码

```java
class Solution {
    public List<Integer> inorderTraversal(TreeNode root) {
        List<Integer> result = new ArrayList<>();
        if (root == null) return result;

        Stack<TreeNode> stack = new Stack<>();
        TreeNode cur = root;
        while (cur != null || !stack.isEmpty()) {
            if (cur != null) {          // 一路向左访问
                stack.push(cur);
                cur = cur.left;         // 左
            } else {                    // 左到头了，回头处理
                cur = stack.pop();
                result.add(cur.val);    // 中
                cur = cur.right;        // 右
            }
        }
        return result;
    }
}
```

### 关键点

- `cur != null` 时：**还在往下探**，不断压栈
- `cur == null` 时：**左到底了，开始回溯**，pop 栈顶处理，然后转向右子树
- 循环条件 `cur != null || !stack.isEmpty()`：指针还有路走，或栈里还有欠账，就继续

## 三、后序遍历（迭代法）

**后序顺序：左 → 右 → 中**

### 核心思路：前序的"反向技巧"

- 前序是 **中 → 左 → 右**
- 如果把前序里左右入栈顺序交换，就变成 **中 → 右 → 左**
- 把这个结果**反转**，就得到 **左 → 右 → 中** = 后序

### 代码

```java
class Solution {
    public List<Integer> postorderTraversal(TreeNode root) {
        List<Integer> result = new ArrayList<>();
        if (root == null) return result;

        Stack<TreeNode> stack = new Stack<>();
        stack.push(root);
        while (!stack.isEmpty()) {
            TreeNode node = stack.pop();        // 中
            result.add(node.val);
            if (node.left != null) stack.push(node.left);   // 左先入栈
            if (node.right != null) stack.push(node.right); // 右后入栈（先出）
        }
        Collections.reverse(result);            // 反转得到 左右中
        return result;
    }
}
```

### 关键点

- 相比前序，**仅仅交换了左右入栈顺序**，最后加一个 reverse
- 本质上还是"访问 = 处理"同步的风格

---

## 四、三者对比

| | 前序 | 中序 | 后序 |
|---|---|---|---|
| 目标顺序 | 中左右 | 左中右 | 左右中 |
| 栈里存什么 | 待访问的节点 | **已访问但未处理**的节点 | 待访问的节点 |
| 是否需要 `cur` 指针 | 不需要 | **需要** | 不需要 |
| 处理时机 | pop 瞬间 | pop 之后（从左子树回来时） | pop 瞬间，最后反转 |
| 代码风格 | 统一 | 特殊 | 与前序几乎一样 |

**核心区别**：前序和后序里，**访问顺序和处理顺序一致**，所以 pop 出来就能直接处理。而中序里两者**不一致**，必须借助指针走到最左下，再回头处理，所以栈的语义和代码结构都不同。

## 五、统一风格的迭代法（标记法）

### 为什么需要统一写法

上面三份代码里，前序和后序风格一致，中序却"另起炉灶"（用了 `cur` 指针）。能不能让三者**代码结构完全一致，只调整几行顺序**？可以，用**标记法**。

### 核心思想

中序之所以特殊，是因为"**访问**"和"**处理**"时机不一致。那我们干脆**把两件事都用栈来表达**：

- **要访问的节点**：正常入栈
- **要处理的节点**：入栈后紧跟一个 **`null` 标记**，表示"这个节点该被收割了"

pop 出一个节点时：
- 如果它**不是 null**：说明是"访问"动作，把它和它的孩子按**目标顺序的逆序**重新入栈，并在自己后面压一个 `null` 做标记
- 如果它**是 null**：说明下一个 pop 的节点是"该处理的"，把它加入 result

### 为什么要按"目标顺序的逆序"入栈

栈是后进先出。想要 pop 出的顺序是 `左 → 中 → 右`，就必须按 `右 → 中 → 左` 入栈。

### 统一写法——中序遍历

```java
class Solution {
    public List<Integer> inorderTraversal(TreeNode root) {
        List<Integer> result = new LinkedList<>();
        Stack<TreeNode> stack = new Stack<>();
        if (root != null) stack.push(root);

        while (!stack.isEmpty()) {
            TreeNode node = stack.peek();
            if (node != null) {
                stack.pop();  // 先弹出，避免重复处理
                // 中序：左中右 → 入栈顺序：右、中、左
                if (node.right != null) stack.push(node.right);  // 右
                stack.push(node);                                // 中
                stack.push(null);                                // 标记：中已访问未处理
                if (node.left != null) stack.push(node.left);    // 左
            } else {
                // 遇到 null，说明下一个节点该被处理
                stack.pop();                // 弹出 null
                node = stack.pop();         // 弹出真正要处理的节点
                result.add(node.val);
            }
        }
        return result;
    }
}
```

### 统一写法——前序遍历

相比中序**只调整了两行的顺序**（把"中"的位置放到最后入栈）：

```java
class Solution {
    public List<Integer> preorderTraversal(TreeNode root) {
        List<Integer> result = new LinkedList<>();
        Stack<TreeNode> stack = new Stack<>();
        if (root != null) stack.push(root);

        while (!stack.isEmpty()) {
            TreeNode node = stack.peek();
            if (node != null) {
                stack.pop();
                // 前序：中左右 → 入栈顺序：右、左、中
                if (node.right != null) stack.push(node.right);  // 右
                if (node.left != null) stack.push(node.left);    // 左
                stack.push(node);                                // 中
                stack.push(null);                                // 标记
            } else {
                stack.pop();
                node = stack.pop();
                result.add(node.val);
            }
        }
        return result;
    }
}
```

### 统一写法——后序遍历

同样**只调整几行顺序**：

```java
class Solution {
    public List<Integer> postorderTraversal(TreeNode root) {
        List<Integer> result = new LinkedList<>();
        Stack<TreeNode> stack = new Stack<>();
        if (root != null) stack.push(root);

        while (!stack.isEmpty()) {
            TreeNode node = stack.peek();
            if (node != null) {
                stack.pop();
                // 后序：左右中 → 入栈顺序：中、右、左
                stack.push(node);                                // 中
                stack.push(null);                                // 标记
                if (node.right != null) stack.push(node.right);  // 右
                if (node.left != null) stack.push(node.left);    // 左
            } else {
                stack.pop();
                node = stack.pop();
                result.add(node.val);
            }
        }
        return result;
    }
}
```

### 统一写法的代价

每个节点会**进出栈两次**（一次访问，一次处理），常数比普通迭代法大，但**三种遍历代码结构完全一致**，只需改几行入栈顺序，不用为中序单独写一套逻辑。

---

## 六、总结

1. **前序和后序代码结构相似**：访问和处理顺序一致，pop 即处理；后序只需交换前序的左右入栈顺序，最后 reverse 结果。

2. **中序是特殊的**：访问顺序和处理顺序**不一致**，必须用 `cur` 指针向下探到最左，再用栈回溯处理。栈里存的是"**已访问但未处理**"的节点。

3. **统一写法用 `null` 标记法**：把"访问"和"处理"两个动作都交给栈管理，节点**后面跟一个 `null`** 表示该被处理了。三种遍历**只差几行入栈顺序**。

4. **选择建议**：面试时普通迭代法（前序/后序一套，中序一套）**更直观、更容易写对**；统一写法理解难度高，但能体现对访问/处理分离的深刻理解，可作为加分点提及。

5. **入栈顺序的万能口诀**：**想让谁先被处理，就让谁最后入栈**（因为栈是 LIFO）。
