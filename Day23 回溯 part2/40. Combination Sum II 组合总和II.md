# 40. Combination Sum II 组合总和II

> [LeetCode 40. Combination Sum II](https://leetcode.com/problems/combination-sum-ii/)

## 一、题目核心

给定一个**有重复元素**的数组 `candidates` 和目标数 `target`，找出所有**和为 target 的唯一组合**。

- 每个数字在**每个组合中只能使用一次**
- **解集不能包含重复的组合**

**难点**：数组有重复元素 → 但组合不能重复 → 需要"同层去重"。

---

## 二、我的原始代码（含 bugs)

```java
class Solution {
    List<List<Integer>> res = new ArrayList<>();
    LinkedList<Integer> path = new LinkedList<>(); 
    int sum = 0; 

    public List<List<Integer>> combinationSum2(int[] candidates, int target) {
        Arrays.sort(candidates);
        backtracking(candidates, target, 0);
        return res; 
    }

    public void backtracking(int[] candidates, int target, int startIndex){
        if(sum == target){
            res.add(new ArrayList<>(path));
            return;
        }

        for(int i = startIndex; i < candidates.length(); i++ ){  // ❌ length()
            if(i+1 == i) continue;                                // ❌ 永远不成立

            if(sum + candidates[i] > target) continue; 
            sum += candidates[i]; 
            path.add(candidates[i]);
            backtracking(candidates, sum, target, i);             // ❌ 参数错
            path.removeLast();
            sum -= candidates[i];
        }
    }
}
```

### 🐛 三个 bug 回顾

| # | 错误 | 正确 | 原因 |
|---|------|------|------|
| 1 | `candidates.length()` | `candidates.length` | 数组的 `length` 是**属性**，不是方法 |
| 2 | `if(i+1 == i) continue;` | `if (i > startIndex && candidates[i] == candidates[i-1]) continue;` | 去重条件：同层遇到重复值才跳过 |
| 3 | `backtracking(candidates, sum, target, i);` | `backtracking(candidates, target, i + 1);` | 参数顺序/数量错；每个元素只用一次，应 `i+1` |

## 三、解法一：不使用 `used[]`（推荐，最简洁）

```java
class Solution {
    List<List<Integer>> res = new ArrayList<>();
    LinkedList<Integer> path = new LinkedList<>();
    int sum = 0;

    public List<List<Integer>> combinationSum2(int[] candidates, int target) {
        Arrays.sort(candidates);  // 必须排序，让重复元素相邻
        backtracking(candidates, target, 0);
        return res;
    }

    private void backtracking(int[] candidates, int target, int startIndex) {
        if (sum == target) {
            res.add(new ArrayList<>(path));
            return;
        }

        for (int i = startIndex; i < candidates.length && sum + candidates[i] <= target; i++) {
            // 同层去重：i > startIndex 说明不是本层第一个元素
            if (i > startIndex && candidates[i] == candidates[i - 1]) {
                continue;
            }

            sum += candidates[i];
            path.add(candidates[i]);
            backtracking(candidates, target, i + 1);  // i + 1：每个元素只用一次
            sum -= candidates[i];
            path.removeLast();
        }
    }
}
```

**关键点**：
- `sum + candidates[i] <= target` 写在 for 循环条件里，相当于 `break` 剪枝（排序后，一旦超了后面更超）
- `i > startIndex` 是「同层去重」的精髓

## 四、解法二：使用 `used[]` 标记数组

```java
class Solution {
    LinkedList<Integer> path = new LinkedList<>();
    List<List<Integer>> ans = new ArrayList<>();
    boolean[] used;
    int sum = 0;

    public List<List<Integer>> combinationSum2(int[] candidates, int target) {
        used = new boolean[candidates.length];
        Arrays.fill(used, false);
        Arrays.sort(candidates);  // 必须排序
        backTracking(candidates, target, 0);
        return ans;
    }

    private void backTracking(int[] candidates, int target, int startIndex) {
        if (sum == target) {
            ans.add(new ArrayList<>(path));
            return;
        }
        for (int i = startIndex; i < candidates.length; i++) {
            if (sum + candidates[i] > target) break;

            // 同层去重：candidates[i-1] 在同层用过（used[i-1] == false）
            if (i > 0 && candidates[i] == candidates[i - 1] && !used[i - 1]) {
                continue;
            }

            used[i] = true;
            sum += candidates[i];
            path.add(candidates[i]);
            backTracking(candidates, target, i + 1);
            used[i] = false;
            sum -= candidates[i];
            path.removeLast();
        }
    }
}
```

## 五、⭐ 两种去重条件的本质

这是 LC 40 的**核心难点**，必须彻底理解：

> **树层去重** vs **树枝去重**
>
> - **树层（horizontal)**：同一个 for 循环里，已经选过 `candidates[i-1]`，现在又遇到相同值 → **跳过**（会产生重复组合）
> - **树枝（vertical)**：从根到叶的一条路径上，选了 `candidates[i-1]` 后又选 `candidates[i]`（值相同）→ **保留**（如 `[1,1]` 是合法组合）

### 两种写法的等价性

| 判断方式 | 条件 | 含义 |
|----------|------|------|
| **不用 used** | `i > startIndex && candidates[i] == candidates[i-1]` | `i == startIndex` 是本层第一个，`i > startIndex` 说明本层已经选过更小索引的相同值 |
| **用 used** | `i > 0 && candidates[i] == candidates[i-1] && !used[i-1]` | `used[i-1] == false` 说明 `candidates[i-1]` 不在当前树枝上 → 一定是被同层回溯过的 |

### 📊 以 `[1, 1, 2]`, target = 3 为例

```
                    [ ]  startIndex=0
                   / | \
            选1(i=0) 选1(i=1) 选2(i=2)
              /       ✂同层去重   \
            [1]                   [2]
           / \
      选1(i=1) 选2(i=2)
        ✅             
        [1,1]      [1,2]
         |           ✓ sum=3
        选2(i=2)
        ✂ sum+2>3
```

- 第 0 层：`i=1` 时 `i > startIndex(0)` 且 `candidates[1]==candidates[0]` → 剪掉第二个 1
- 第 1 层：`i=1` 等于 `startIndex(1)`，不触发跳过 → `[1,1]` 得以保留 ✅

## 六、🔗 回溯去重三连对比（LC 39 / 40 / 47)

这三道题去重条件非常相似但**意义完全不同**，放在一起整理印象最深：

| 题目 | 数组特点 | 元素可复用？ | 起点 | 去重条件 |
|------|----------|--------------|------|----------|
| **LC 39** Combination Sum | 无重复 | ✅ 可重复使用 | `i`（下层仍从 i 开始) | **不需要去重** |
| **LC 40** Combination Sum II | 有重复 | ❌ 每个只用一次 | `i + 1` | `i > startIndex && nums[i]==nums[i-1]` |
| **LC 47** Permutations II | 有重复 | ❌ 每个只用一次 | 每次从 0 开始 + `used[]` | `i > 0 && nums[i]==nums[i-1] && !used[i-1]` |

### 代码骨架对比

**LC 39 — Combination Sum**（可复用，无去重)

```java
for (int i = startIndex; i < candidates.length && sum + candidates[i] <= target; i++) {
    sum += candidates[i];
    path.add(candidates[i]);
    backtracking(candidates, target, i);  // ⭐ i（可重复使用）
    sum -= candidates[i];
    path.removeLast();
}
```

**LC 40 — Combination Sum II**（有重复，同层去重)

```java
for (int i = startIndex; i < candidates.length && sum + candidates[i] <= target; i++) {
    if (i > startIndex && candidates[i] == candidates[i - 1]) continue;  // ⭐ 同层去重
    sum += candidates[i];
    path.add(candidates[i]);
    backtracking(candidates, target, i + 1);  // ⭐ i + 1（只用一次）
    sum -= candidates[i];
    path.removeLast();
}
```

**LC 47 — Permutations II**（排列问题，用 used)

```java
for (int i = 0; i < nums.length; i++) {           // ⭐ 每次从 0 开始
    if (used[i]) continue;                        // 本路径已用过
    if (i > 0 && nums[i] == nums[i-1] && !used[i-1]) continue;  // ⭐ 同层去重
    used[i] = true;
    path.add(nums[i]);
    backtracking(nums);
    used[i] = false;
    path.removeLast();
}
```

### 🧠 三道题的本质差异

1. **组合 vs 排列**
   - 组合用 `startIndex` 控制顺序（选过的不回头)
   - 排列每次从 0 开始，用 `used[]` 记录哪些被占用了

2. **元素可否重复使用**
   - 可复用 → 下层 `i`
   - 不可复用 → 下层 `i + 1`（组合）或 `used[i] = true`（排列)

3. **数组本身有无重复**
   - 无重复 → 不需要去重代码
   - 有重复 → 必须排序 + 同层去重

---

## 七、📝 最终总结

1. **排序是前提**：让相同元素相邻，才能通过比较相邻元素判断"同层重复"
2. **组合问题用 `startIndex`**：保证选取的元素有序，避免 `[1,2]` 和 `[2,1]` 同时出现
3. **"每个元素只用一次" → 下层传 `i + 1`**
4. **"同层去重" 两种等价写法**：
   - 不用辅助数组：`i > startIndex && candidates[i] == candidates[i-1]`
   - 用辅助数组：`i > 0 && candidates[i] == candidates[i-1] && !used[i-1]`
5. **剪枝优化**：排序后 `sum + candidates[i] > target` 可以直接 `break`（写在 for 条件里更优雅）
6. **时间复杂度 O(n · 2ⁿ)，空间复杂度 O(n)**
