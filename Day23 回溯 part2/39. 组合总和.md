# 39. 组合总和 (Combination Sum)

> [LeetCode 39. Combination Sum](https://leetcode.com/problems/combination-sum/)

给定一个**无重复元素**的数组 `candidates` 和一个目标数 `target`,找出 `candidates` 中所有可以使数字和为 `target` 的组合。`candidates` 中的数字可以**无限制重复**被选取。

**示例:**

- `candidates = [2,3,6,7], target = 7` → `[[7], [2,2,3]]`
- `candidates = [2,3,5], target = 8` → `[[2,2,2,2], [2,3,3], [3,5]]`

---

## 一、本题与 77.组合 / 216.组合总和III 的区别

| 题目 | 数量要求 | 能否重复 | 递归终止 |
|------|---------|---------|---------|
| 77.组合 | 取 k 个 | 不能 | 层数 == k |
| 216.组合总和III | 取 k 个 | 不能 | 层数 == k 且 sum == target |
| **39.组合总和** | **无** | **可以** | **sum == target 或 sum > target** |

**关键:** 本题没有数量要求,但有总和限制,所以靠 `sum > target` 来剪枝。

## 二、我的调试过程(错误 → 修正)

### 🔴 第一版:最初的错误代码

```java
class Solution {
    List<List<Integer>> res = new ArrayList<>(); 
    LinkedList<Integer> path = new LinkedList<>(); 
    int sum = 0; 

    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        backtracking(candidates, sum, target, 0);
        return res; 
    }

    public void backtracking(int[] candidates, int sum, int target, int startIndex){
        // stop condition
        if(sum == target){
            res.add(path);           // ❌ Bug 2: 直接加引用
        }                            // ❌ Bug 3: 没写 return

        // backtracking for loop 
        for(int i = startIndex; i < candidates.length; i++){
            sum += i;                // ❌ Bug 1: 加的是下标,不是值
            if(sum > target) break; 
                                     // ❌ Bug 4: 没有 path.add(...)
            backtracking(candidates, sum, target, i+1);  // ❌ Bug 5: i+1 不允许重复用
            path.removeLast();
            sum -= i;                // ❌ Bug 6: 减的也是下标
        }
    }
}
```

**发现的 bug 清单:**

| Bug | 问题 | 说明 |
|-----|------|------|
| 1 | `sum += i` | `i` 是下标,应该加 `candidates[i]`(值) |
| 2 | `res.add(path)` | 存的是引用,回溯时会被修改,最终都变空 |
| 3 | 找到答案后没 `return` | 应该停止当前路径的搜索 |
| 4 | 没有 `path.add(candidates[i])` | 只 `removeLast` 没 `add`,逻辑不对称 |
| 5 | `i+1` | 题目允许重复使用,应该传 `i` |
| 6 | `sum -= i` | 和 `sum += i` 一样,应该是值不是下标 |

### 🟡 第二版:修掉 Bug 2、3、4、5、6,但 `sum -= i` 还错着

```java
public void backtracking(int[] candidates, int sum, int target, int startIndex){
    if(sum == target){
        res.add(new ArrayList<>(path));   // ✅ 独立拷贝
        return;                            // ✅ 加了 return
    }

    for(int i = startIndex; i < candidates.length; i++){
        sum += candidates[i];              // ✅ 改成值
        if(sum > target) break; 
        path.add(candidates[i]);           // ✅ 补上了 add
        backtracking(candidates, sum, target, i);   // ✅ 传 i 允许重复
        path.removeLast();
        sum -= i;                          // ❌ 这里还是下标!
    }
}
```

**教训:** 做选择和撤销选择必须**对称一致**。加了 `candidates[i]` 就要减 `candidates[i]`。

---

### 🟠 第三版:修掉 `sum -= i`,但 `break` 在未排序数组上错杀

```java
for(int i = startIndex; i < candidates.length; i++){
    sum += candidates[i]; 
    if(sum > target) break;            // ⚠️ 数组没排序,break 会错杀
    path.add(candidates[i]);
    backtracking(candidates, sum, target, i);
    path.removeLast();
    sum -= candidates[i];              // ✅ 改对了
}
```

**测试用例翻车:** `candidates = [8,7,4,3], target = 11` → 输出 `[]`,期望 `[[8,3],[7,4],[4,4,3]]`

**原因:** `break` 直接退出整个 for 循环,只有在数组**已排序**时才安全(因为后面的更大,肯定也超)。`[8,7,4,3]` 没排序,选 8 之后 `8+8=16` 超了就 `break`,后面的 3 根本没机会被试到 → `[8,3]` 丢了。

---

### 🟡 第四版:改成 `continue`,但忘了恢复 `sum`

```java
for(int i = startIndex; i < candidates.length; i++){
    sum += candidates[i]; 
    if(sum > target) continue;         // ❌ sum 没减回来就 continue 了!
    path.add(candidates[i]);
    backtracking(candidates, sum, target, i);
    path.removeLast();
    sum -= candidates[i];              // 这一行在 continue 路径上不会执行
}
```

**测试用例依然翻车:** `[8,7,4,3], target = 11` → `[]`

**Trace 分析(path = [4,4] 时):**

```
ENTER: sum=8, start=2, path=[4,4]
  i=2, candidates[i]=4, sum becomes 12
    ⚠️ sum > target, CONTINUE (sum 保持在 12,没减!)
  i=3, candidates[i]=3, sum becomes 15   ← 应该是 8+3=11,但因为 sum 卡在 12,变成 12+3=15
    ⚠️ sum > target, CONTINUE
```

因为 `continue` 前没有 `sum -= candidates[i]`,下一次循环 `sum` 带着错误的状态继续累加,`[4,4,3]` 这种合法解就被错过了。

**核心原则:** 
> **"选择"和"撤销选择"必须成对出现**。无论通过哪个路径(return / continue / break)退出当前迭代,都必须确保已做的修改被撤销。

### ✅ 最终版:排序 + 提前判断 + break

```java
class Solution {
    List<List<Integer>> res = new ArrayList<>(); 
    LinkedList<Integer> path = new LinkedList<>(); 

    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        Arrays.sort(candidates);                     // ⭐ 关键:先排序
        backtracking(candidates, 0, target, 0);
        return res; 
    }

    public void backtracking(int[] candidates, int sum, int target, int startIndex){
        if(sum == target){
            res.add(new ArrayList<>(path));
            return;
        }

        for(int i = startIndex; i < candidates.length; i++){
            if(sum + candidates[i] > target) break;  // ⭐ 先判断,不用撤销
            sum += candidates[i]; 
            path.add(candidates[i]);
            backtracking(candidates, sum, target, i); // ⭐ 传 i 允许重复使用
            path.removeLast();
            sum -= candidates[i];
        }
    }
}
```

**两个关键优化:**
1. **`Arrays.sort(candidates)`** — 排序后,一旦当前元素加上就超,后面更大的必然也超,`break` 安全有效
2. **`if(sum + candidates[i] > target) break`** — 提前判断,不真的修改 `sum`,就不需要为 `break` 手动回滚

## 三、老师的讲解:回溯三部曲

### 1️⃣ 递归函数参数

```java
void backtracking(int[] candidates, int target, int sum, int startIndex)
```

- `sum`:累加路径和(也可以用 `target` 做减法代替)
- `startIndex`:控制 for 循环起点,**避免重复组合**

**什么时候需要 `startIndex`?**

| 场景 | 是否需要 startIndex |
|------|---------------------|
| 一个集合求组合(如本题、77、216) | ✅ 需要 |
| 多个集合取组合(如 17.电话号码) | ❌ 不需要 |
| 排列问题 | ❌ 不需要(用 used 数组) |

### 2️⃣ 递归终止条件

```java
if (sum > target) return;      // 剪枝后可省略
if (sum == target) {
    res.add(new ArrayList<>(path));
    return;
}
```

### 3️⃣ 单层搜索逻辑

```java
for (int i = startIndex; i < candidates.length; i++) {
    sum += candidates[i];
    path.add(candidates[i]);
    backtracking(candidates, target, sum, i);  // ⭐ 传 i 而非 i+1,允许重复
    sum -= candidates[i];
    path.removeLast();
}
```

**关键点:** `backtracking(..., i)` — 不是 `i+1`,表示下一层可以继续用当前这个数。

## 四、剪枝优化(排序 + break)

对 `candidates` 排序后,可以在 for 循环条件里直接剪枝:

```java
for (int i = startIndex; 
     i < candidates.length && sum + candidates[i] <= target; 
     i++) {
    ...
}
```

或者在循环体内提前判断:

```java
for (int i = startIndex; i < candidates.length; i++) {
    if (sum + candidates[i] > target) break;
    ...
}
```

**效果:** 一旦 `sum + candidates[i] > target`,后面更大的元素也必然超,直接 `break` 跳出整层循环。

## 五、Java 标准解法(代码随想录版本)

```java
class Solution {
    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        List<List<Integer>> res = new ArrayList<>();
        Arrays.sort(candidates);                            // 先进行排序
        backtracking(res, new ArrayList<>(), candidates, target, 0, 0);
        return res;
    }

    public void backtracking(List<List<Integer>> res, List<Integer> path, 
                             int[] candidates, int target, int sum, int idx) {
        // 找到了数字和为 target 的组合
        if (sum == target) {
            res.add(new ArrayList<>(path));
            return;
        }

        for (int i = idx; i < candidates.length; i++) {
            // 如果 sum + candidates[i] > target 就终止遍历
            if (sum + candidates[i] > target) break;
            path.add(candidates[i]);
            backtracking(res, path, candidates, target, sum + candidates[i], i);
            path.remove(path.size() - 1);                    // 回溯
        }
    }
}
```

**复杂度分析:**
- 时间复杂度:`O(n × 2ⁿ)` — 上界,剪枝后实际远小于此
- 空间复杂度:`O(target)` — 递归栈深度

## 六、踩坑总结 🪤

1. **`sum += i` vs `sum += candidates[i]`** — 下标和值别搞混,永远记住 `i` 是**位置**,`candidates[i]` 才是**值**
2. **`res.add(path)` vs `res.add(new ArrayList<>(path))`** — 必须**深拷贝**,否则所有结果都指向同一个对象
3. **找到答案后必须 `return`** — 不然会继续往下搜,浪费时间(有剪枝时未必出错,但逻辑上必须)
4. **`i` vs `i+1`** — 能重复用传 `i`,不能重复用传 `i+1`
5. **做选择和撤销选择要**对称**(单位一致)** — 加 `candidates[i]` 就减 `candidates[i]`,加值不能减下标
6. **`break` 只在**排序后**才安全** — 未排序数组用 `break` 会错杀后面的更小元素
7. **`continue` 前必须手动撤销修改** — 否则状态污染后续迭代
8. **最优雅的写法:"先判断,再修改"** — 把 `if (sum + candidates[i] > target) break` 放在 `sum += candidates[i]` 之前,就不需要为 `break/continue` 手动回滚

## 七、回溯模板(组合问题)🎯

```java
void backtracking(参数, int startIndex) {
    // 1. 终止条件(收集结果)
    if (满足条件) {
        res.add(new ArrayList<>(path));   // ⚠️ 一定要深拷贝
        return;                            // ⚠️ 一定要 return
    }
    
    // 2. for 循环横向遍历(带剪枝)
    for (int i = startIndex; i < n; i++) {
        if (剪枝条件) break;               // 排序后的大小剪枝
        
        path.add(...);                     // ⭐ 做选择
        backtracking(..., i 或 i+1);       // ⭐ 递归(能重复传 i,不能重复传 i+1)
        path.removeLast();                 // ⭐ 撤销选择
        // 其他状态(如 sum)也要对称撤销
    }
}
```

**最重要的三条原则:**

1. **深拷贝** — `res.add(new ArrayList<>(path))`
2. **成对对称** — 做选择必撤销,加什么减什么,单位一致
3. **先判断再修改** — 剪枝条件放在状态修改之前,省去回滚麻烦
