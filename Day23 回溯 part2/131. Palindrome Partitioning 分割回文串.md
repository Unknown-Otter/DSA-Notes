# 131. Palindrome Partitioning 分割回文串

> [LeetCode 131](https://leetcode.com/problems/palindrome-partitioning/) | [代码随想录讲解](https://programmercarl.com/0131.%E5%88%86%E5%89%B2%E5%9B%9E%E6%96%87%E4%B8%B2.html)

## 题目描述

给定一个字符串 `s`，将 `s` 分割成一些子串，使每个子串都是回文串。返回 `s` 所有可能的分割方案。

**示例**：
```
输入: "aab"
输出: [["aa","b"], ["a","a","b"]]
```

---

## 一、核心思路

本题涉及两个关键问题：

1. **切割问题**：有不同的切割方式
2. **回文判断**：判断子串是否是回文

### 切割问题 ≈ 组合问题

对于字符串 `abcdef`：

- **组合问题**：选取一个 `a` 之后，在 `bcdef` 中再去选取第二个，选取 `b` 之后在 `cdef` 中再选取第三个......
- **切割问题**：切割一个 `a` 之后，在 `bcdef` 中再去切割第二段，切割 `b` 之后在 `cdef` 中再切割第三段......

**本质一样**——都是在「选切割点 / 选元素」，可以用同样的回溯树形结构建模。

递归纵向遍历，for 循环横向遍历，**切割线切到字符串结尾**就说明找到了一种切割方法。

---

## 二、回溯三部曲

### 1. 递归函数参数

- `path`：存放已经切割出来的回文子串（全局变量）
- `res`：存放最终结果集（全局变量）
- `startIndex`：下一轮递归遍历的起始位置（**切割线**），切割过的地方不能重复切割

### 2. 终止条件

`startIndex >= s.length()`：切割线切到字符串末尾，说明找到了一种切割方案，将 `path` 加入 `res`。

### 3. 单层逻辑

在 `for (int i = startIndex; i < s.length(); i++)` 循环中：

- `[startIndex, i]` 就是当前要截取的子串
- 如果是回文 → 加入 `path` → 递归 `i+1` → 回溯弹出
- 如果不是回文 → `continue` 跳过

```java
class Solution {
    List<List<String>> res  = new ArrayList<>();
    LinkedList<String> path = new LinkedList<>();

    public List<List<String>> partition(String s) {
```

```java
        return res;

        return true;
    }

        if(startIndex >= s.length()){
            res.add(new ArrayList<>(path));
            return;
        }

        for(int i = startIndex; i< s.length(); i++){
            // 🐛 隐蔽 bug：检查的子串和加入的子串不一致！
            StringBuilder temp = new StringBuilder(s.substring(startIndex, i));
            if(check(temp) == true){
                String sub = s.substring(startIndex, i + 1);  // ← [startIndex, i+1)
                path.add(sub);
                backtracking(s, i + 1);
                path.removeLast();
            }else{
                continue;
            }
        }
    }
}
```

#### 🐛 Bug 分析：边界不一致

```java
StringBuilder temp = new StringBuilder(s.substring(startIndex, i));   // 检查 [startIndex, i)
String sub = s.substring(startIndex, i + 1);                          // 使用 [startIndex, i+1)
```

**反例验证** `s = "aba"`，走到 `startIndex=0, i=1`：

- `temp = s.substring(0, 1) = "a"` → 是回文 ✅
- `sub = s.substring(0, 2) = "ab"` → **但 `"ab"` 根本不是回文！**

于是 `["ab", "a"]` 会被错误地加入结果 ❌

#### 🪞 复盘：为什么这个 bug 隐蔽？

1. `"aab"` 这种短测试用例会蒙混过关（因为 `"a"`、`"aa"` 恰好都是回文）
2. 代码逻辑「看起来」对——都是 `substring`，都是 `startIndex` 到 `i`
3. 只有当 `[startIndex, i)` 是回文但 `[startIndex, i+1)` 不是的时候才暴露

**调试心法**：当循环里写两次 `substring`，**立刻警觉**——边界是否一致？能不能合并成一个变量？

## 四、✅ 正确代码（推荐版本）

```java
class Solution {
    List<List<String>> res = new ArrayList<>();
    LinkedList<String> path = new LinkedList<>();

    public List<List<String>> partition(String s) {
        backtracking(s, 0);
        return res;
    }

    private void backtracking(String s, int startIndex) {
        // 终止条件：切到末尾，path 就是一种合法方案
        if (startIndex >= s.length()) {
            res.add(new ArrayList<>(path));
            return;
        }

        // 单层逻辑：尝试在 [startIndex, i] 这个区间切一刀
        for (int i = startIndex; i < s.length(); i++) {
            if (isPalindrome(s, startIndex, i)) {
                path.add(s.substring(startIndex, i + 1));
                backtracking(s, i + 1);
                path.removeLast();  // 回溯：撤销刚加入的子串
            }
            // 不是回文就跳过这个切割点（continue 可省，for 自动进入下一轮）
        }
    }

    // 双指针判断 s[left..right] 是否是回文
    private boolean isPalindrome(String s, int left, int right) {
        while (left < right) {
            if (s.charAt(left) != s.charAt(right)) return false;
            left++;
            right--;
        }
        return true;
    }
}
```

### 相比我的第二版改进

- 🚀 **性能**：`isPalindrome` 用索引判断，不再 `new StringBuilder`，省了每次 O(n) 的内存分配
- 🛡️ **安全**：`substring` 只调用一次，杜绝边界不一致
- 🧼 **简洁**：去掉冗余的 `else continue`

## 六、优化：DP 预处理回文表

### 优化动机

双指针判断 `isPalindrome` 存在**重复计算**：例如已知 `"bcd"` 不是回文时，判断 `"abcde"` 又要从头双指针走一遍。

### 回文的递推关系

给定字符串 `s`，它是回文字串的**充分必要条件**：
- `s[0] == s[n-1]` **且** `s[1:n-1]` 是回文

### DP 状态定义

`dp[i][j]` 表示 `s[i..j]`（双边包括）是否是回文字串。

**递推公式**：
```
if s[i] == s[j]:
    if j - i <= 1:              // 长度 1 或 2
        dp[i][j] = true
    else:
        dp[i][j] = dp[i+1][j-1]  // 依赖内部子串
```

**注意**：`dp[i][j]` 依赖 `dp[i+1][j-1]`，所以 `i` 要**倒序**遍历，`j` 要**正序**遍历（保证计算 `i` 行时 `i+1` 行已算好）。

### DP 优化版完整代码

```java
class Solution {
    List<List<String>> result = new ArrayList<>();
    LinkedList<String> path = new LinkedList<>();
    boolean[][] dp;

    public List<List<String>> partition(String s) {
        dp = new boolean[s.length()][s.length()];
        computePalindrome(s);
        backtracking(s, 0);
        return result;
    }

    private void backtracking(String s, int startIndex) {
        if (startIndex >= s.length()) {
            result.add(new ArrayList<>(path));
            return;
        }
        for (int i = startIndex; i < s.length(); i++) {
            if (dp[startIndex][i]) {         // O(1) 查表
                path.add(s.substring(startIndex, i + 1));
                backtracking(s, i + 1);
                path.removeLast();
            }
        }
    }

    // 预处理：dp[i][j] 表示 s[i..j] 是否回文
    private void computePalindrome(String s) {
        for (int i = s.length() - 1; i >= 0; i--) {      // i 倒序
            for (int j = i; j < s.length(); j++) {       // j 正序
                if (j == i) {
                    dp[i][j] = true;                     // 单字符
                } else if (j - i == 1) {
                    dp[i][j] = (s.charAt(i) == s.charAt(j));  // 两字符
                } else {
                    dp[i][j] = (s.charAt(i) == s.charAt(j) && dp[i + 1][j - 1]);
                }
            }
        }
    }
}
```

## 七、复杂度分析

| 版本 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 双指针判断 | O(n · 2ⁿ) | O(n) 递归栈 |
| DP 预处理 | O(n · 2ⁿ) | O(n²) dp 表 |

- **2ⁿ**：每个位置「切或不切」，最多 2^(n-1) 种方案
- **n**：每种方案做拷贝 / 回文检查

## 八、总结

### 本题的难点

1. ✅ **切割问题可以抽象为组合问题** ← 最关键的思维突破
2. ✅ **如何模拟切割线**：`startIndex` 是上一层已确定的分割线，`i` 是这一层试图寻找的新分割线
3. ✅ **递归终止**：`startIndex >= s.length()` 说明切到了末尾
4. ✅ **截取子串**：`s.substring(startIndex, i + 1)`，注意两个边界**必须一致**
5. ✅ **回文判断**：双指针 or DP 预处理

### 关键要点

1. **问题本质**：切割问题 = 在 `[startIndex, i]` 这个窗口里选切割点，等价于「组合」的变种
2. **回溯三要素**：
   - 参数：`(s, startIndex)`
   - 终止：`startIndex >= s.length()` → 收集方案
   - 单层：从 `startIndex` 到末尾枚举切点 `i`，若子串回文则递归 `i+1`
3. **回溯对称性**：`path.add(sub)` ↔ `path.removeLast()`，加了什么就撤什么
4. **天然剪枝**：不是回文直接 `continue`，不递归
5. **优化方向**：DP 预处理把每次 O(n) 回文判断降到 O(1)

### Java 易错点（我的踩坑）

- ⚠️ `s.length()` 不是 `s.length`（字符串是方法，数组才是属性）
- ⚠️ `substring(start, end)` 的 `end` 是**开区间**，注意 `+1`
- ⚠️ **两次 substring 边界必须一致**，否则会出现「检查通过但实际加入的不是回文」的隐蔽 bug
- ⚠️ 回溯撤销要撤销「刚加入 `path` 的元素」，不是别的临时变量
