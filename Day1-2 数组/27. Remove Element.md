# 27. Remove Element

> [LeetCode 27. Remove Element](https://leetcode.com/problems/remove-element/)

## 思路：快慢指针

**核心思想**：用两个指针遍历数组，`fast` 负责扫描每个元素，`slow` 负责记录下一个有效元素该写入的位置。

- 当 `nums[fast] != val` → 把值写入 `nums[slow]`，然后 `slow++`
- 当 `nums[fast] == val` → 跳过，`slow` 不动

**`slow` 的双重角色**：
- **写入指针**：告诉 fast "下一个有效元素放这里"
- **计数器**：循环结束后，`slow` 的值就是保留了多少个有效元素

## 代码

```java
class Solution {
    public int removeElement(int[] nums, int val) {
        int slowIndex = 0;
        for (int fastIndex = 0; fastIndex < nums.length; fastIndex++) {
            if (nums[fastIndex] != val) {
                nums[slowIndex] = nums[fastIndex];
                slowIndex++;
            }
        }
        return slowIndex;
    }
}
```

## 思路反思：我是怎么想岔的

**误区一：以为需要 Binary Search**
一开始以为要"找到"target 再删除，想到了 binary search。但这题数组不一定有序，而且核心不是"查找"而是"移动元素"。

**误区二：想用 ArrayList 中转**
担心在 array 中间删除元素后要把后面的元素逐个往前移（O(n) 操作），所以想创建 ArrayList 来做，最后再转回 array。但题目说 **顺序可以改变**，根本不需要"删除再移位"，只需要把有效元素往前"压缩"就行。

**误区三：以为 slow 也遍历了整个数组**
一开始不理解为什么返回 `slow`，觉得 slow 跟 fast 一样走完了整个数组。实际上 **slow 和 fast 的移动条件不同**：fast 无条件 `++`，slow 只在遇到非 target 时才 `++`。每跳过一个 target，slow 就落后 fast 一步。

**误区四：以为 slow 指向的一定是 target**
后来又以为 slow 当前位置存的一定是 target 元素。其实 slow 不关心自己位置存的是什么，它只代表"下一个该写入的位置"，当前值随时会被覆盖。

**误区五：以为这题只是返回一个 int**
以为只需要计数，如果要返回结果数组就得换做法。实际上快慢指针在遍历过程中已经把有效元素按顺序压缩到了数组前面，**计数只是附带结果**，数组本身已经被 in-place 修改好了。

## 总结

1. **快慢指针不需要数组有序**，适合 in-place 移除/压缩类问题
2. **slow 的本质是"写入指针"**，不是用来找 target 的，它标记的是"下一个有效元素该放哪"
3. **两个指针脱节的时刻**就是遇到 target 的时刻，最终 `slow = 数组长度 - target个数`
4. 这个模式可以推广到类似题目：如 **26. Remove Duplicates from Sorted Array**、**283. Move Zeroes**
