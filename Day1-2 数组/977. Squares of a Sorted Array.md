# 977. Squares of a Sorted Array

> [LeetCode 977](https://leetcode.com/problems/squares-of-a-sorted-array/)

## 核心思路

输入是一个**非递减**排序的数组，平方之后两头大、中间小。所以用双指针从两端向中间比较，每轮挑出当前最大的平方值，**从右往左**填入结果数组。

## 错误代码

```java
class Solution {
    public int[] sortedSquares(int[] nums) {
        int left = 0;
        int right = nums.length - 1;

        int[] arr = new int[nums.length];

        while (left <= right) {
            int leftSqr = nums[left] * nums[left];
            int rightSqr = nums[right] * nums[right];

            if (leftSqr <= rightSqr) {
                arr[right] = rightSqr;
                right--;
            } else if (leftSqr > rightSqr) {
                arr[right] = leftSqr;
                left++;
            }
        }

        return arr;
    }
}
```

## 错误分析：`right` 身兼两职导致覆盖

**问题：** `right` 同时充当了「读指针」和「写指针」，但这两件事的节奏不一样。

- 作为**读指针**：`right` 只在右边平方值更大时才 `right--`
- 作为**写指针**：每轮都确定了一个最大值，写入位置应该**每轮都往前移一格**

**用 `nums = [-4, -1, 0, 3, 10]` 举例：**

| 轮次 | left | right | leftSqr | rightSqr | 走哪个分支 | 写入操作 | arr 状态 |
|------|------|-------|---------|----------|-----------|---------|---------|
| 1 | 0 | 4 | 16 | 100 | `<=` | `arr[4]=100`, `right--` | `[0,0,0,0,100]` ✅ |
| 2 | 0 | 3 | 16 | 9 | `>` | `arr[3]=16`, `left++` | `[0,0,0,16,100]` ✅ |
| 3 | 1 | 3 | 1 | 9 | `<=` | `arr[3]=9`, `right--` | `[0,0,0,9,100]` ❌ |

**第3轮出错：** 第2轮走的是 `else` 分支，只做了 `left++`，`right` 没动（还是3）。第3轮又往 `arr[3]` 写入，把第2轮放好的 16 覆盖掉了。

## 正确代码

```java
class Solution {
    public int[] sortedSquares(int[] nums) {
        int left = 0;
        int right = nums.length - 1;
        int index = nums.length - 1; // 独立写指针

        int[] arr = new int[nums.length];

        while (left <= right) {
            int leftSqr = nums[left] * nums[left];
            int rightSqr = nums[right] * nums[right];

            if (leftSqr <= rightSqr) {
                arr[index] = rightSqr;
                right--;
            } else {
                arr[index] = leftSqr;
                left++;
            }
            index--; // 每轮无条件递减
        }

        return arr;
    }
}
```

## 关键点总结

1. **为什么 `left <= right`：** 当 `left == right` 时还剩一个元素没处理，必须用 `<=` 才能覆盖到
2. **为什么需要独立的 `index`：** `left` 和 `right` 是读指针，控制「比较谁」；`index` 是写指针，控制「放到哪」。每轮都会确定一个值，所以 `index` 每轮无条件 `index--`，和读指针的移动彻底解耦
3. **相等不需要单独处理：** `leftSqr == rightSqr` 时归到 `<=` 分支，移动 `right` 即可，下一轮 `left` 指向的相同值会自然被处理
