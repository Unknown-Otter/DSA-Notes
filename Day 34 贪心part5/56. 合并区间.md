
# 56. Merge Intervals

> [LeetCode 链接](https://leetcode.com/problems/merge-intervals/)

给定一个区间数组 `intervals`,合并所有**重叠**的区间,返回不重叠的区间数组。

**注意**:边界相等(如 `[1,4]` 和 `[4,5]`)也算**重叠**,需要合并成 `[1,5]`。

---

## 核心思路

和 **LC 452(用最少数量的箭引爆气球)** 是同一个思路家族:**按起点排序 + 遍历合并**。

但两道题的目标不同:

| 题目 | 目标 | 输出 |
|------|------|------|
| LC 452 | **统计**不重叠的组数 | `int`(箭的数量) |
| LC 56 | **合并**重叠的区间并收集 | `int[][]`(合并后的区间) |

所以 LC 56 不仅要判断重叠,还要:
1. 把重叠区间**合并**(更新终点为 `max`)
2. 把结果**动态收集**到 `List<int[]>`

---

## 算法步骤

### Step 1: 按起点排序

```java
Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));
```

**为什么排序**:让所有可能重叠的区间在数组中**相邻**,这样只需要线性扫一遍即可。

⚠️ **避免使用 `a[0] - b[0]`**:如果 `a[0] = Integer.MIN_VALUE`,`b[0] = 1`,相减会**整数溢出**。`Integer.compare` 是安全写法。

### Step 2: 用 List 动态收集结果

```java
List<int[]> result = new ArrayList<>();
```

**为什么用 List 而不是数组**:事先不知道合并后有几个区间,需要动态扩容。

### Step 3: 遍历每个区间,与 result 的最后一个比较

**关键技巧**:每次只和 `result` 中的**最后一个**区间比较,因为已经按起点排序,前面的区间不可能再和后面的重叠。

三种区间关系判断:

| 情况 | 例子 | `last[1]` vs `curr[0]` | 操作 |
|------|------|----------------------|------|
| 完全分开 | `[1,3]` 和 `[5,8]` | `3 < 5` | **不重叠** → 加入 |
| 边界相等 | `[1,4]` 和 `[4,5]` | `4 == 4` | **重叠** → 合并 |
| 部分重叠 | `[1,4]` 和 `[2,5]` | `4 > 2` | **重叠** → 合并 |

所以「**不重叠**」的判断条件是 **严格小于**:`last[1] < curr[0]`,**不能用 `<=`**。

### Step 4: List → int[][]

```java
return result.toArray(new int[result.size()][]);
```

---

## 完整代码(推荐写法)

```java
class Solution {
    public int[][] merge(int[][] intervals) {
        // 1. 按起点排序
        Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));
        
        // 2. 用 List 动态收集结果
        List<int[]> result = new ArrayList<>();
        
        // 3. 遍历每个区间
        for (int[] interval : intervals) {
            // 情况 A: result 为空 或 不重叠 → 直接加入
            if (result.isEmpty() || result.get(result.size() - 1)[1] < interval[0]) {
                result.add(interval);
            } 
            // 情况 B: 重叠 → 合并(更新最后一个区间的终点)
            else {
                int[] last = result.get(result.size() - 1);
                last[1] = Math.max(last[1], interval[1]);
            }
        }
        
        // 4. List<int[]> → int[][]
        return result.toArray(new int[result.size()][]);
    }
}
```

## 等价写法对比

下面这个写法和上面**完全等价**,只是判断方向反过来(从「重叠」入手):

```java
// 等价写法:从「重叠」角度写
for (int[] current : intervals) {
    int size = result.size();
    if (size > 0 && current[0] <= result.get(size - 1)[1]) {
        // 重叠 → 合并
        int[] last = result.get(size - 1);
        last[1] = Math.max(last[1], current[1]);
    } else {
        // 不重叠 → 加入
        result.add(current);
    }
}
```

**两种条件的等价关系**:
- 「不重叠」:`last[1] < current[0]`
- 「重叠」:`current[0] <= last[1]`(即 `last[1] >= current[0]`)
- 两者互为否定 ✅

## 推演验证

**输入**:`[[1,3],[2,6],[8,10],[15,18]]`(已按起点排序)

| 步骤 | current | result 状态 | 判断 | 操作 |
|------|---------|------------|------|------|
| 1 | `[1,3]` | `[]` | result 为空 | 加入 → `[[1,3]]` |
| 2 | `[2,6]` | `[[1,3]]` | `3 < 2`? **否** → 重叠 | 合并:`last[1] = max(3,6) = 6` → `[[1,6]]` |
| 3 | `[8,10]` | `[[1,6]]` | `6 < 8`? **是** → 不重叠 | 加入 → `[[1,6],[8,10]]` |
| 4 | `[15,18]` | `[[1,6],[8,10]]` | `10 < 15`? **是** → 不重叠 | 加入 → `[[1,6],[8,10],[15,18]]` |

**输出**:`[[1,6],[8,10],[15,18]]` ✅

## Bug 历史 🐛

### Bug 1: `<` vs `<=` 混淆

**错误**:用 `<=` 判断「不重叠」
```java
if (result.get(...)[1] <= interval[0]) {  // ❌ 错!
    result.add(interval);
}
```

**问题**:对于 `[[1,4],[4,5]]`,`4 <= 4` 为 `true`,会错误地把 `[4,5]` 当作不重叠加入,得到 `[[1,4],[4,5]]`,但正确答案是 `[[1,5]]`。

**原因**:边界相等(`==`)算重叠,所以「不重叠」必须**严格小于**。

**修复**:用 `<`
```java
if (result.get(...)[1] < interval[0]) {  // ✅ 对
    result.add(interval);
}
```

### Bug 2: 排序用 `a[0] - b[0]` 溢出

**错误**:
```java
Arrays.sort(intervals, (a, b) -> a[0] - b[0]);  // ⚠️ 可能溢出
```

**问题**:当 `a[0] = Integer.MIN_VALUE`,`b[0] = 1` 时,`MIN_VALUE - 1` 会溢出变成正数,导致排序错误。

**修复**:
```java
Arrays.sort(intervals, (a, b) -> Integer.compare(a[0], b[0]));  // ✅ 安全
```

### Bug 3: 误以为要和 `intervals[i-1]` 比较

**错误思路**:遍历时直接比较 `intervals[i]` 和 `intervals[i-1]`。

**问题**:如果出现 `[1,3], [2,4], [5,9]` 这种连续合并,`intervals[i-1]` 已经过时(应该和合并后的 `[1,4]` 比较,而不是原始的 `[2,4]`)。

**修复**:始终与 **`result` 的最后一个**比较。`result` 维护的是**已经合并好**的状态。

## 复杂度分析

- **时间复杂度**:`O(n log n)` — 排序主导,遍历是 `O(n)`
- **空间复杂度**:`O(n)` — `result` 最坏情况存所有区间(完全不重叠时)
  - 不计返回值的话,排序的栈空间是 `O(log n)`

## 与 LC 452 对比

**LC 452(用最少数量的箭引爆气球)**:

```java
public int findMinArrowShots(int[][] points) {
    Arrays.sort(points, (a, b) -> Integer.compare(a[0], b[0]));
    int count = 1;
    for (int i = 1; i < points.length; i++) {
        if (points[i][0] > points[i-1][1]) {
            count++;  // 不重叠 → 需要新箭
        } else {
            points[i][1] = Math.min(points[i][1], points[i-1][1]);  // 重叠 → 缩小公共区间
        }
    }
    return count;
}
```

**核心区别**:

| 维度 | LC 452 | LC 56 |
|------|--------|-------|
| 边界相等 | `>` 才算不重叠(`==` 重叠) | `<` 才算不重叠(`==` 重叠) |
| 重叠时操作 | 取**终点的 min**(缩小公共区间) | 取**终点的 max**(扩大合并区间) |
| 输出 | 不重叠组数 | 合并后的区间数组 |
| 数据结构 | 计数器 `int count` | `List<int[]>` |

**记忆口诀**:
- 452 找**公共重叠区域** → 用 `min`
- 56 找**合并后的总区间** → 用 `max`

---

## 总结

1. **排序是前提**:按起点排序,让可能重叠的区间相邻。
2. **`Integer.compare` 防溢出**:面试时优先用,不要写 `a[0] - b[0]`。
3. **与 result 的最后一个比较**,而不是原数组的前一个,处理连续合并。
4. **「不重叠」用 `<`**,不能用 `<=`(因为边界相等算重叠)。
5. **合并时取 `max(last[1], curr[1])`** 更新终点,起点不用动(已排序)。
6. **List → int[][]**:用 `result.toArray(new int[result.size()][])`,简洁地道。
7. **同思路兄弟题**:LC 452(用 `min`)、LC 435(无重叠区间)、LC 763(划分字母区间)。
