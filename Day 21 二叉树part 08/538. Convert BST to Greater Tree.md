# 538. Convert BST to Greater Tree

> [LeetCode 538](https://leetcode.com/problems/convert-bst-to-greater-tree/)

## 题目理解

给定一个 BST，将每个节点的值修改为：**原值 + 所有比它大的节点值之和**。

核心观察：BST 的**反中序遍历**（右 → 中 → 左）会按从大到小的顺序访问节点，因此只需维护一个累加变量 `sum`/`pre`，遍历过程中不断累加即可。

---

## 解法一：递归（反中序遍历）

用一个成员变量 `sum` 记录累加值，递归按 **右 → 中 → 左** 顺序遍历。

**为什么用 helper 函数？** `convertBST` 返回 `TreeNode` 是 LeetCode 接口要求，但递归本身不需要返回值——只是遍历+修改。用 `void` 的 helper 更准确地表达意图。

```java
class Solution {
    int sum;

    public TreeNode convertBST(TreeNode root) {
        sum = 0;
        helper(root);
        return root;
    }

    public void helper(TreeNode root) {
        if (root == null) return;

        helper(root.right);   // 右：先访问更大的节点
        sum += root.val;      // 中：累加
        root.val = sum;       //     更新当前节点值
        helper(root.left);    // 左：再访问更小的节点
    }
}
```

**也可以不用 helper，直接写在 `convertBST` 里：**

```java
class Solution {
    int sum = 0;

    public TreeNode convertBST(TreeNode root) {
        if (root == null) return null;
        convertBST(root.right);
        sum += root.val;
        root.val = sum;
        convertBST(root.left);
        return root; // 递归调用时没人接这个返回值，只有最外层调用者需要
    }
}
```

## 解法二：DFS 迭代（统一迭代法）

使用栈模拟反中序遍历，采用**标记法（null 标记）**统一处理入栈顺序和处理顺序。

**核心思路：**
- `curr != null` 时：只负责按 **左 → 中(标记null) → 右** 的顺序入栈（因为栈是 LIFO，实际弹出顺序就是 **右 → 中 → 左**）
- `curr == null` 时：说明下一个栈顶是被标记的节点，弹出并执行**单层逻辑**（累加 + 赋值）

```java
class Solution {
    public TreeNode convertBST(TreeNode root) {
        int pre = 0;
        Stack<TreeNode> stack = new Stack<>();
        if (root == null) return null;

        stack.add(root);

        while (!stack.isEmpty()) {
            TreeNode curr = stack.peek();

            if (curr != null) {
                // curr != null：只负责存 node 到 stack 中
                stack.pop();
                if (curr.left != null)        // 左（最后处理，先入栈）
                    stack.add(curr.left);
                stack.add(curr);              // 中
                stack.add(null);              // null 标记：下次遇到 null 就处理它上面的节点
                if (curr.right != null)       // 右（最先处理，最后入栈）
                    stack.add(curr.right);
            } else {
                // curr == null：只负责做单层逻辑
                stack.pop();                  // 弹出 null 标记
                TreeNode temp = stack.pop();  // 弹出实际要处理的节点
                temp.val += pre;
                pre = temp.val;
            }
        }
        return root;
    }
}
```

## 踩坑记录

- **helper 函数 void vs 有返回值**：判断标准是调用者有没有 `xxx = func(...)` 去**接住返回值**。这道题递归只是遍历+修改，没有人需要返回值，所以本质是 void 操作
- **统一迭代法的入栈顺序**：想要 **右 → 中 → 左** 的处理顺序，入栈时要反过来写成 **左 → 中(null) → 右**
- **null 标记的含义**：null 不是空节点，是一个"信号"，告诉循环"null 上面那个节点已经就位了，该处理了"

## 复杂度分析

| | 时间复杂度 | 空间复杂度 |
|---|---|---|
| 递归 | O(n) | O(h)，h 为树高 |
| 迭代 | O(n) | O(n) |

---

## 总结

1. **反中序遍历**（右 → 中 → 左）是解题关键，利用 BST 性质从大到小访问
2. 递归解法中，helper 用 **void** 更清晰地表达"只遍历不返回"的语义
3. 统一迭代法通过 **null 标记** 将"入栈"和"处理"分离，是一种通用的迭代遍历模板
4. 两种解法的核心逻辑完全一致：维护累加变量 `sum`/`pre`，遍历时更新节点值
