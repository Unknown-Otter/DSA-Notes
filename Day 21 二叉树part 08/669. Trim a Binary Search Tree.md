
# 669. Trim a Binary Search Tree

> [LeetCode 669](https://leetcode.com/problems/trim-a-binary-search-tree/)

## 题目理解

给定一棵BST和范围 `[low, high]`，裁剪这棵树，使所有节点的值都在 `[low, high]` 范围内。裁剪后的树仍然是一棵合法的BST。

## Java 解法

```java
class Solution {
    public TreeNode trimBST(TreeNode root, int low, int high) {
        if (root == null) {
            return null;
        }
        if (root.val < low) {
            return trimBST(root.right, low, high);
        }
        if (root.val > high) {
            return trimBST(root.left, low, high);
        }
        // root在[low,high]范围内
        root.left = trimBST(root.left, low, high);
        root.right = trimBST(root.right, low, high);
        return root;
    }
}
```

## 递归思路拆解

把 `trimBST` 想成一个**承诺**：给我任何一棵BST子树，我保证还你一棵裁剪好的、所有值都在 `[low, high]` 内的树。

每一层节点只需要做一个简单的判断：

- **`root.val < low`**（我太小了）：当前节点和整个左子树都不要（BST左边更小），去右子树里找合格的。执行 `return trimBST(root.right, low, high)`。
- **`root.val > high`**（我太大了）：当前节点和整个右子树都不要（BST右边更大），去左子树里找合格的。执行 `return trimBST(root.left, low, high)`。
- **`root.val` 在 `[low, high]` 内**（我合格）：保留自己，但左右子树里可能藏着不合格的节点，分别递归处理后接回来。

## 核心机制："接线"操作

这道题递归的精髓在于：**每一层真正的操作不是发生在被调用的函数里，而是发生在调用者的赋值语句上。**

```java
// 拆开来看这行代码：
TreeNode result = trimBST(root.left, low, high);  // 第一步：递归拿到结果
root.left = result;                                 // 第二步：赋值 = 接线操作
```

如果 `result` 和原来的 `root.left` 是同一个节点，什么都没变。但如果不是（比如原来的 `root.left` 不合格，被它的某个后代替代了），这一赋值就改变了树的结构——不合格的节点被跳过，合格的后代直接接了上去。

**每一层做判断，但判断的结果要到上一层的赋值才真正生效。** 被调用者负责"决定谁该留下"，调用者负责"把留下的人接上去"。

## 另一个关键理解：return 一个节点 = return 整棵子树

在树这个数据结构里，一个节点就是它下面整棵子树的"提手"。return 节点3，3.left 指向 2，3.right 指向 4，4.right 指向 5——整棵树通过引用链全都连着。return 一个节点就是把以它为根的一整棵树都带出去了。

## 完整运行模拟

**示例树**，调用 `trimBST(root, 2, 5)`：

```
         6
        / \
       3    7
      / \
     1    4
      \    \
       2    5
```

### Step 1：trimBST(6, 2, 5)

```
Call Stack:
┌─────────────────────────┐
│ trimBST(6)  ◀ 当前执行    │
└─────────────────────────┘
```

6 > high(5)，6太大了。6和右子树(7)全部丢弃。执行 `return trimBST(6.left, 2, 5)`，去看节点3。

### Step 2：trimBST(3, 2, 5)

```
Call Stack:
┌─────────────────────────┐
│ trimBST(6)   等节点3       │
├─────────────────────────┤
│ trimBST(3)  ◀ 当前执行    │
└─────────────────────────┘
```

3在[2,5]范围内，合格！走到 `root.left = trimBST(root.left, ...)` 分支，先处理左子树(节点1)。

### Step 3：trimBST(1, 2, 5)

```
Call Stack:
┌─────────────────────────────────┐
│ trimBST(6)   等节点3              │
├─────────────────────────────────┤
│ trimBST(3)   等结果 → 赋给 3.left │
├─────────────────────────────────┤
│ trimBST(1)  ◀ 当前执行           │
└─────────────────────────────────┘
```

1 < low(2)，太小了。1和左子树全部不要。执行 `return trimBST(1.right, 2, 5)`，去看节点2。

### Step 4：trimBST(2, 2, 5)

```
Call Stack:
┌───────────────────────────────────────┐
│ trimBST(6)   等节点3                    │
├───────────────────────────────────────┤
│ trimBST(3)   等结果 → 赋给 3.left       │
├───────────────────────────────────────┤
│ trimBST(1)   等节点2，拿到就直接return    │
├───────────────────────────────────────┤
│ trimBST(2)  ◀ 当前执行                  │
└───────────────────────────────────────┘
```

2在[2,5]范围内，合格！左右子节点都是null，递归返回null，2没有变化。**return 节点2。**

### Step 5：返回链 —— 节点2 → 节点1 → 节点3

trimBST(2) 返回节点2 → trimBST(1) 拿到节点2，直接 return 给上层（**节点1被跳过，从此消失**） → trimBST(3) 拿到节点2。

```
Call Stack:
┌─────────────────────────────────────────────┐
│ trimBST(6)   等节点3                          │
├─────────────────────────────────────────────┤
│ trimBST(3)   拿到节点2 → 执行 3.left = 节点2   │
└─────────────────────────────────────────────┘
```

**⚡ 接线操作发生！`3.left` 原来指向1，现在指向2。节点1从树里彻底断开。**

树的当前状态（节点1已消失）：

```
         6
        / \
       3    7
      / \
     2    4
           \
            5
```

### Step 6：trimBST(4, 2, 5)

3.left 处理完了，接下来处理右子树：`3.right = trimBST(4, 2, 5)`。

```
Call Stack:
┌──────────────────────────────────┐
│ trimBST(6)   等节点3               │
├──────────────────────────────────┤
│ trimBST(3)   等结果 → 赋给 3.right │
├──────────────────────────────────┤
│ trimBST(4)  ◀ 当前执行            │
└──────────────────────────────────┘
```

4在[2,5]范围内，合格！左子树null，右子树是5。

### Step 7：trimBST(5, 2, 5)

5在[2,5]范围内，无子节点，直接返回节点5。回到trimBST(4)：`4.right = 节点5`（和原来一样，没变）。**return 节点4（带着5一起，因为4.right指向5）。**

### Step 8：节点3完成

trimBST(3) 拿到节点4，`3.right = 节点4`（和原来一样）。左右都处理完了。**return 节点3（带着2、4、5一起）。**

```
Call Stack:
┌─────────────────────────────────────┐
│ trimBST(6)   拿到节点3 → 直接return  │
└─────────────────────────────────────┘
```

### Step 9：最终结果

trimBST(6) 执行的是 `return trimBST(6.left, ...)`，拿到节点3，直接return。**节点6和节点7都被丢弃，节点3成为新根。**

```
      3
     / \
    2    4
          \
           5
```

所有节点值都在 [2, 5] 范围内，裁剪完成。

## 一句话总结

**合格的节点原样返回自己，不合格的节点通过 return 下面某个合格的后代来完成自我淘汰，而上层的赋值语句负责把这个替换落实到树的结构上。**

## 复杂度分析

- **时间复杂度**：O(n)，每个节点最多被访问一次
- **空间复杂度**：O(n)，最坏情况下递归栈的深度（树退化为链表时）
