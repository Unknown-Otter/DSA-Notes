## 62. 不同路径

### 题目描述

一个机器人位于 `m x n` 网格的左上角，每次只能**向下**或**向右**移动一步。机器人试图到达网格的右下角，问总共有多少条不同的路径？

### 思路分析

#### 1. 定义 dp 数组

**`dp[i][j]` = 从起点 `(0,0)` 走到位置 `(i,j)` 一共有多少条不同的路径**

#### 2. 推导递推公式

到达 `(i, j)` 只有两种来源：
- 从上面 `(i-1, j)` 向下走一步
- 从左边 `(i, j-1)` 向右走一步

所以：

```java
dp[i][j] = dp[i-1][j] + dp[i][j-1];
```

#### 3. 初始化（关键理解点）

**为什么需要初始化第一行和第一列？**

递推公式用到了 `dp[i-1][j]` 和 `dp[i][j-1]`，当 `i=0` 或 `j=0` 时会数组越界。所以**第一行和第一列没办法用递推公式算出来，必须手动赋值**。

**为什么初始值是 1？**

- 第一行 `dp[0][j]`：因为不能向上走，只能从起点一路向右，路径**唯一**，所以是 1
- 第一列 `dp[i][0]`：因为不能向左走，只能从起点一路向下，路径**唯一**，所以是 1

```java
for (int i = 0; i < m; i++) dp[i][0] = 1;
for (int j = 0; j < n; j++) dp[0][j] = 1;
```

#### 4. 遍历顺序

因为 `dp[i][j]` 依赖左边和上边，所以从左到右、从上到下遍历即可。

#### 5. 举例推导（3×3 网格）

初始化后：

```
1 1 1
1 . .
1 . .
```

逐格填充：

```
最终 dp 表：
1 1 1
1 2 3
1 3 6  ← dp[m-1][n-1] = 6 即为答案
```

### 代码实现

```java
class Solution {
    public static int uniquePaths(int m, int n) {
        int[][] dp = new int[m][n];

        // 初始化第一列
        for (int i = 0; i < m; i++) {
            dp[i][0] = 1;
        }
        // 初始化第一行
        for (int j = 0; j < n; j++) {
            dp[0][j] = 1;
        }

        // 递推填表
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                dp[i][j] = dp[i-1][j] + dp[i][j-1];
            }
        }
        return dp[m-1][n-1];
    }
}
```

### 为什么 return `dp[m-1][n-1]`？

1. `dp[i][j]` 的定义就是"从起点到 `(i,j)` 的路径数"
2. 题目要求的是"从起点到**终点**的路径数"
3. 终点 = 右下角 = `(m-1, n-1)`（因为 Java 数组索引从 0 开始）
4. 所以答案就是 `dp[m-1][n-1]`

## 63. 不同路径 II

### 题目描述

在 62 题的基础上，网格中加入了**障碍物**（用 `1` 表示，空格用 `0` 表示）。机器人不能进入障碍物所在的格子，问从左上角到右下角有多少条不同的路径？

### 思路分析

整体框架与 62 题一致，但需要处理两个新情况：

#### 1. 初始化时遇到障碍要"截断"

第一列上机器人只能向下走，**如果中间某个格子是障碍，那么障碍之后的所有格子都到不了**，必须保持为 0。

利用 `&&` 短路终止循环是最优雅的写法：

```java
for (int i = 0; i < m && obstacleGrid[i][0] == 0; i++) {
    dp[i][0] = 1;
}
```

**反例（错误写法）**：

```java
// ❌ 错误：障碍后面的格子仍然被错误地设为 1
for (int i = 0; i < m; i++) {
    if (obstacleGrid[i][0] == 0) dp[i][0] = 1;
}
```

例如第一列是 `[0, 0, 1, 0, 0]`，正确结果应为 `[1, 1, 0, 0, 0]`，而错误写法会得到 `[1, 1, 0, 1, 1]`。

#### 2. 递推时遇到障碍要置 0

如果当前格是障碍物，路径数为 0（不能停留也不能穿过）：

```java
dp[i][j] = (obstacleGrid[i][j] == 0) ? dp[i-1][j] + dp[i][j-1] : 0;
```

#### 3. 特判

起点或终点是障碍物时直接返回 0。

#### 4. 举例推导（中间有障碍）

```
obstacleGrid:
0 0 0
0 1 0
0 0 0
```

填表过程：

```
最终 dp 表：
1 1 1
1 0 1
1 1 2  ← 答案 = 2
```

注意 `dp[1][1] = 0` 起到了"切断路径"的关键作用。

### 代码实现

```java
class Solution {
    public int uniquePathsWithObstacles(int[][] obstacleGrid) {
        int m = obstacleGrid.length;
        int n = obstacleGrid[0].length;
        int[][] dp = new int[m][n];

        // 特判：起点或终点是障碍直接返回 0
        if (obstacleGrid[0][0] == 1 || obstacleGrid[m-1][n-1] == 1) {
            return 0;
        }

        // 初始化第一列：遇到障碍立即停止
        for (int i = 0; i < m && obstacleGrid[i][0] == 0; i++) {
            dp[i][0] = 1;
        }

        // 初始化第一行：遇到障碍立即停止
        for (int j = 0; j < n && obstacleGrid[0][j] == 0; j++) {
            dp[0][j] = 1;
        }

        // 递推填表：障碍格保持 0
        for (int i = 1; i < m; i++) {
            for (int j = 1; j < n; j++) {
                dp[i][j] = (obstacleGrid[i][j] == 0) ? dp[i-1][j] + dp[i][j-1] : 0;
            }
        }

        return dp[m-1][n-1];
    }
}
```

### 常见 bug

循环边界 `m` 和 `n` 写反——尤其在**非方形网格**才会暴露：

```java
// ❌ 错：第一行有 n 列，应该是 j < n
for (int j = 0; j < m && obstacleGrid[0][j] == 0; j++) { ... }

// ✅ 对
for (int j = 0; j < n && obstacleGrid[0][j] == 0; j++) { ... }
```

## 通用总结

1. **dp 数组定义**：`dp[i][j]` = 从起点到 `(i,j)` 的不同路径数
2. **递推公式**：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`（障碍格则为 0）
3. **初始化逻辑**：递推公式覆盖不到的边界（第一行、第一列）需要手动赋值；有障碍时遇障碍立即截断
4. **遍历顺序**：从左到右、从上到下
5. **返回值**：`dp[m-1][n-1]`，因为右下角就是终点
6. **调试技巧**：把 dp 表打印出来直接看
   ```java
   for (int i = 0; i < m; i++) {
       System.out.println(Arrays.toString(dp[i]));
   }
   ```
7. **DP 通用思路**：**先想递推公式 → 看公式用到哪些前置状态 → 那些前置状态够不到的边界，就是需要初始化的地方**
