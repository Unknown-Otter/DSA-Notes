
# 93-Restore-IP-Addresses

> [LeetCode 93. 复原 IP 地址](https://leetcode.cn/problems/restore-ip-addresses/)

## 题目描述

给定一个只包含数字的字符串 `s`，复原它并返回所有可能的 **有效 IP 地址** 格式。

**有效 IP 地址** 正好由 **四个整数** 组成：
- 每个整数位于 `0` 到 `255` 之间
- 不能含有前导 0
- 整数之间用 `.` 分隔

**示例：**
```
输入：s = "25525511135"
输出：["255.255.11.135","255.255.111.35"]

输入：s = "101023"
输出：["1.0.10.23","1.0.102.3","10.1.0.23","10.10.2.3","101.0.2.3"]
```

## 解题思路

这是一道**切割问题**，本质上和 [131. 分割回文串](https://leetcode.cn/problems/palindrome-partitioning/) 是同一类型，都可以用**回溯算法**把所有可能性搜索出来。

切割问题可以抽象成一棵树：
- **树的宽度**：每一层从 `startIndex` 开始往后切的所有可能位置
- **树的深度**：切割了多少段（最多 4 段）

## 回溯三部曲

### 1. 递归参数

需要两个关键参数：
- `startIndex`：本层递归切割的起始位置（不能重复切割）
- `pointNum`：已经添加的逗点数量（用来判断是否切够 4 段）

### 2. 终止条件

和分割回文串不同，这里**不是用切割线切到末尾作为终止**，而是用**分段数量**作为终止条件。

当 `pointNum == 3` 时，说明字符串已经被分成了 4 段。此时还要验证**最后一段**是否合法，合法才加入结果。

```java
if (pointNum == 3) {
    if (isValid(s, startIndex, s.length() - 1)) {
        result.add(s);
    }
    return;
}
```

### 3. 单层搜索逻辑

在 `for` 循环中，`[startIndex, i]` 区间就是当前截取的子串：

- 如果**合法**：在 `i` 后面插入 `.`，`pointNum + 1`，递归时 `startIndex = i + 2`（跳过插入的点）
- 如果**不合法**：直接 `break` 结束本层循环（剪枝）

回溯时记得：删掉刚才插入的 `.`，`pointNum - 1`。

```java
for (int i = startIndex; i < s.length(); i++) {
    if (isValid(s, startIndex, i)) {
        s = s.substring(0, i + 1) + "." + s.substring(i + 1);
        pointNum++;
        backTrack(s, i + 2, pointNum);
        pointNum--;
        s = s.substring(0, i + 1) + s.substring(i + 2);
    } else {
        break;
    }
}
```

## 判断子串是否合法

`isValid` 函数需要考虑 **3 种不合法情况**：

1. **前导 0**：`0` 开头但长度大于 1（如 `01`、`001` 不合法，但单独的 `0` 合法）
2. **非数字字符**：含有非 `0-9` 的字符
3. **数值越界**：转换成数字后大于 `255`

```java
private Boolean isValid(String s, int start, int end) {
    if (start > end) return false;
    if (s.charAt(start) == '0' && start != end) return false;  // 前导 0
    int num = 0;
    for (int i = start; i <= end; i++) {
        if (s.charAt(i) > '9' || s.charAt(i) < '0') return false;  // 非数字
        num = num * 10 + (s.charAt(i) - '0');
        if (num > 255) return false;  // 越界
    }
    return true;
}
```

## 完整代码

### 方法一：使用 String（基础版）

```java
class Solution {
    List<String> result = new ArrayList<>();

    public List<String> restoreIpAddresses(String s) {
        if (s.length() > 12) return result; // 剪枝：超过 12 位不可能合法
        backTrack(s, 0, 0);
        return result;
    }

    // startIndex: 搜索的起始位置，pointNum: 添加逗点的数量
    private void backTrack(String s, int startIndex, int pointNum) {
        if (pointNum == 3) {  // 逗点数量为 3 时，分隔结束
            if (isValid(s, startIndex, s.length() - 1)) {
                result.add(s);
            }
            return;
        }
        for (int i = startIndex; i < s.length(); i++) {
            if (isValid(s, startIndex, i)) {
                s = s.substring(0, i + 1) + "." + s.substring(i + 1);  // 插入逗点
                pointNum++;
                backTrack(s, i + 2, pointNum);  // 下一个子串起始位置为 i+2
                pointNum--;                     // 回溯
                s = s.substring(0, i + 1) + s.substring(i + 2);  // 回溯：删掉逗点
            } else {
                break;
            }
        }
    }

    // 判断字符串 s 在左闭右闭区间 [start, end] 所组成的数字是否合法
    private Boolean isValid(String s, int start, int end) {
        if (start > end) return false;
        if (s.charAt(start) == '0' && start != end) return false;
        int num = 0;
        for (int i = start; i <= end; i++) {
            if (s.charAt(i) > '9' || s.charAt(i) < '0') return false;
            num = num * 10 + (s.charAt(i) - '0');
            if (num > 255) return false;
        }
        return true;
    }
}
```

### 方法二：使用 StringBuilder（优化版）

**优化点**：用 `StringBuilder` 避免每次 `substring` 创建新字符串，时间和空间复杂度都更优。

```java
class Solution {
    List<String> result = new ArrayList<>();

    public List<String> restoreIpAddresses(String s) {
        StringBuilder sb = new StringBuilder(s);
        backTracking(sb, 0, 0);
        return result;
    }

    private void backTracking(StringBuilder s, int startIndex, int dotCount) {
        if (dotCount == 3) {
            if (isValid(s, startIndex, s.length() - 1)) {
                result.add(s.toString());
            }
            return;
        }
        for (int i = startIndex; i < s.length(); i++) {
            if (isValid(s, startIndex, i)) {
                s.insert(i + 1, '.');
                backTracking(s, i + 2, dotCount + 1);
                s.deleteCharAt(i + 1);  // 回溯
            } else {
                break;
            }
        }
    }

    private boolean isValid(StringBuilder s, int start, int end) {
        if (start > end) return false;
        if (s.charAt(start) == '0' && start != end) return false;
        int num = 0;
        for (int i = start; i <= end; i++) {
            int digit = s.charAt(i) - '0';
            num = num * 10 + digit;
            if (num > 255) return false;
        }
        return true;
    }
}
```

### 方法三：更强的剪枝（最优版）

**优化点**：在 `for` 循环条件中直接限制每段最长 3 位、数值不超过 255，减少无效递归。

```java
class Solution {
    List<String> result = new ArrayList<String>();
    StringBuilder stringBuilder = new StringBuilder();

    public List<String> restoreIpAddresses(String s) {
        restoreIpAddressesHandler(s, 0, 0);
        return result;
    }

    // number 表示 stringBuilder 中 ip 段的数量
    public void restoreIpAddressesHandler(String s, int start, int number) {
        // start 到达末尾且已凑够 4 段，加入结果
        if (start == s.length() && number == 4) {
            result.add(stringBuilder.toString());
            return;
        }
        // 提前结束：要么 start 到末尾但段数不够，要么段数已满但 start 没到末尾
        if (start == s.length() || number == 4) {
            return;
        }
        // 剪枝：每段最长 3 位且值在 [0, 255]
        for (int i = start; i < s.length() && i - start < 3
                && Integer.parseInt(s.substring(start, i + 1)) >= 0
                && Integer.parseInt(s.substring(start, i + 1)) <= 255; i++) {
            // 前导 0 剪枝
            if (i + 1 - start > 1 && s.charAt(start) - '0' == 0) {
                break;
            }
            stringBuilder.append(s.substring(start, i + 1));
            // 前 3 段后面加 ".", 最后一段不加
            if (number < 3) {
                stringBuilder.append(".");
            }
            number++;
            restoreIpAddressesHandler(s, i + 1, number);
            number--;
            // 回溯：删除当前段（注意点的数量）
            stringBuilder.delete(start + number, i + number + 2);
        }
    }
}
```

## 复杂度分析

- **时间复杂度**：`O(3^4)` —— IP 地址最多 4 段，每段最多 3 种切法，搜索树深度为 4
- **空间复杂度**：`O(n)` —— 递归栈深度

## 总结

1. **本质是切割问题**：和 131 分割回文串套路完全一致，用回溯穷举所有切法
2. **终止条件用段数而非位置**：`pointNum == 3` 表示已切出 4 段
3. **不合法直接 break**：因为后面切得更长肯定也不合法（这是关键剪枝）
4. **isValid 三个判断**：前导 0、非数字、大于 255
5. **回溯要还原现场**：插入了 `.` 就要删掉，`pointNum++` 就要 `--`
6. **优化方向**：`String → StringBuilder`，再加上长度和数值的循环条件剪枝

> 💡 这道题可以看作是 131 分割回文串的**加强版**——多了"插入分隔符"和"区间合法性验证"两个操作。
