# 746. Min Cost Climbing Stairs

> [LeetCode 链接](https://leetcode.com/problems/min-cost-climbing-stairs/)

## 题目理解

给定一个数组 `cost`，`cost[i]` 表示**踩在第 i 级台阶上**需要支付的费用。

规则：
- 一旦付了费用，可以选择**向上爬 1 步或 2 步**
- 起点可以是下标 **0** 或下标 **1**（起点不收费）
- 目标：到达**楼顶**（下标 `cost.length`，越过最后一级台阶）的**最小总花费**

**关键认知**：楼顶不在数组里。如果 `cost.length = n`，那么楼顶的"位置"是 `n`，而台阶编号是 `0 ~ n-1`。

## 暴力思路（先理解题意）

每一级都有"跳 1 步"或"跳 2 步"两种选择，自然想到递归：

```java
// 到达位置 i 的最小花费
int minCost(int i) {
    if (i <= 1) return 0;  // 起点免费
    return Math.min(
        minCost(i - 1) + cost[i - 1],  // 从 i-1 跳 1 步过来
        minCost(i - 2) + cost[i - 2]   // 从 i-2 跳 2 步过来
    );
}
```

时间复杂度 **O(2^n)**，存在大量重复子问题 → **用 DP 优化**。

## DP 五步法

### 1. dp 数组定义

`dp[i]` 表示**到达**第 i 级台阶的**最小累计花费**（注意：到达但还没付 i 的费用）。

特别地：`dp[cost.length]` 表示到达楼顶的最小花费，即最终答案。

### 2. 递推公式

```java
dp[i] = Math.min(dp[i - 1] + cost[i - 1], dp[i - 2] + cost[i - 2]);
```

**两种到达方式取最小**：
- 从 `i-1` 跳 1 步：在 i-1 起跳，付 `cost[i-1]`
- 从 `i-2` 跳 2 步：在 i-2 起跳，付 `cost[i-2]`

### 3. 初始化

```java
dp[0] = 0;  // 起点，免费
dp[1] = 0;  // 起点，免费
```

题目明确说"可以从下标 0 或 1 开始"，所以两个起点的累计花费都是 0。

### 4. 遍历顺序

从前往后：`i` 从 2 到 `cost.length`（**包含** `cost.length`，因为楼顶才是终点）。

### 5. 举例推导

`cost = [10, 15, 20]`：

| i | dp[i] | 计算过程 |
|---|-------|---------|
| 0 | 0 | 起点 |
| 1 | 0 | 起点 |
| 2 | 10 | min(dp[1]+15, dp[0]+10) = min(15, 10) |
| 3 | 15 | min(dp[2]+20, dp[1]+15) = min(30, 15) ← 楼顶 |

答案 **15**：从下标 1 起跳，跳 2 步直接到楼顶，付 `cost[1] = 15`。

## 代码实现

### 解法一：dp 数组（O(n) 空间）

```java
class Solution {
    public int minCostClimbingStairs(int[] cost) {
        int len = cost.length;
        int[] dp = new int[len + 1];

        // 起点免费
        dp[0] = 0;
        dp[1] = 0;

        for (int i = 2; i <= len; i++) {
            dp[i] = Math.min(dp[i - 1] + cost[i - 1], dp[i - 2] + cost[i - 2]);
        }

        return dp[len];
    }
}
```

### 解法二：滚动变量（O(1) 空间）

注意到 `dp[i]` 只依赖 `dp[i-1]` 和 `dp[i-2]`，可以**只用两个变量滚动**：

```java
class Solution {
    public int minCostClimbingStairs(int[] cost) {
        int len = cost.length;
        int prev2 = 0;  // dp[i-2]
        int prev1 = 0;  // dp[i-1]

        for (int i = 2; i <= len; i++) {
            int curr = Math.min(prev1 + cost[i - 1], prev2 + cost[i - 2]);
            prev2 = prev1;
            prev1 = curr;
        }

        return prev1;
    }
}
```

## 关键易错点

### 易错点 1：`cost` 和 `dp` 的概念混淆

- `cost[i]` = 踩在第 i 级的**单步费用**（局部）
- `dp[i]` = 到达第 i 级的**累计费用**（全局）

转移方程的"加法"本质：**上一站的累计 + 这一步的过路费 = 这一站的累计**。

### 易错点 2：为什么是 `cost[i-1]` 而不是 `cost[i]`

题目定义"踩在 i 上才付 `cost[i]`"。从 i-1 跳到 i 的过程中：
- 在 **i-1 起跳**，付的是 `cost[i-1]`
- **还没站上 i**，所以不付 `cost[i]`

因此 `dp[i]` 表示"到达 i 但**未付 i 的费用**"。这个定义让 `dp[len]`（楼顶，无费用）天然就是答案。

### 易错点 3：循环边界 `i <= len` 而不是 `i < len`

楼顶位置是 `cost.length`，需要算到 `dp[len]` 才是答案。`dp` 数组大小要开 `len + 1`。

### 易错点 4：起点初始化

`dp[0] = dp[1] = 0` 不是因为"`cost[0]` 或 `cost[1]` 为 0"，而是因为**题目允许从这两个位置免费出发**。

## 复杂度分析

| 解法 | 时间 | 空间 |
|------|------|------|
| dp 数组 | O(n) | O(n) |
| 滚动变量 | O(n) | O(1) |

## 总结

1. **dp 定义**：`dp[i]` 是"到达 i 的最小累计花费"，**不是**"i 这一步的花费"。
2. **递推方程**："上一站累计 + 这一步过路费"取两种来源的最小值。
3. **边界**：起点 `dp[0] = dp[1] = 0`，终点是 `dp[len]`（楼顶）。
4. **空间优化**：转移只依赖前两项，可滚动到 O(1)。
5. **题目变种识别**：和 LC 70（爬楼梯）、LC 198（打家劫舍）同属"线性 DP，状态依赖前两项"系列。
