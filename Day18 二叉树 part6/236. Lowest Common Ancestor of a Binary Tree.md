# 236. Lowest Common Ancestor of a Binary Tree

> [LeetCode 236](https://leetcode.com/problems/lowest-common-ancestor-of-a-binary-tree/)

## 题目理解

给定一棵二叉树和两个节点 p、q，找到它们的**最低公共祖先（LCA）**——即最深的同时是 p 和 q 祖先的节点（节点可以是自己的祖先）。

```
        3
       / \
      5    1
     / \  / \
    6   2 0   8
       / \
      7   4
```

## 情况分析

一共有 **4 种核心情况**：

1. **p 和 q 分别在左右子树** → LCA 是当前 root（如 p=5, q=1 → LCA=3）
2. **p 是 q 的祖先**（或反过来）→ LCA 是 p 本身（如 p=5, q=4 → LCA=5）
3. **p 和 q 都在左子树** → LCA 在左子树中（如 p=7, q=4 → LCA=2）
4. **p 和 q 都在右子树** → LCA 在右子树中（如 p=0, q=8 → LCA=1）

## 思路：后序遍历 + 自底向上汇报

核心思想：用递归**自底向上**地汇报信息。每个节点问左右子树"你们找到 p 或 q 了吗？"，然后根据左右的返回值做判断。

**每个节点只做三件事：**

1. **我自己是 p 或 q 吗？** → 是的话直接返回自己（提前返回，同时作为 base case）
2. **递归左右子树** → 后序遍历，先左后右
3. **汇总左右返回值：**
   - 左右都非 null → **我就是 LCA**，返回自己
   - 只有一边非 null → **透传**那一边的结果给 parent
   - 都是 null → 返回 null（这棵子树里没有 p 也没有 q）

**为什么不需要显式记录 parent？** 递归调用栈本身就是 parent 链，每次 `return` 就是在向 parent 汇报结果。

## 递归模拟（情况3：p=7, q=4, LCA=2）

```
调用 LCA(3)
├── 左：LCA(5)
│   ├── 左：LCA(6)
│   │   ├── 左null，右null
│   │   └── 6 != p, 6 != q → 返回 null
│   ├── 右：LCA(2)
│   │   ├── 左：LCA(7) → 7 == p，命中！→ 返回 7
│   │   ├── 右：LCA(4) → 4 == q，命中！→ 返回 4
│   │   └── 左7（非null），右4（非null）→ 我(2)就是LCA → 返回 2
│   └── 左null，右返回2 → 透传 2
├── 右：LCA(1) → 整棵右子树找不到 → 返回 null
└── 左返回2，右null → 透传 2 ✅
```

## 代码

```java
class Solution {
    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        // base case: 空节点，或者当前节点就是 p 或 q
        if (root == null || root == p || root == q) return root;

        // 后序：先递归左右
        TreeNode left = lowestCommonAncestor(root.left, p, q);
        TreeNode right = lowestCommonAncestor(root.right, p, q);

        // 汇总：左右都找到了，当前节点就是 LCA
        if (left != null && right != null) return root;

        // 只有一边找到了，透传那一边的结果
        return left != null ? left : right;
    }
}
```

## 复杂度

- **时间复杂度**：O(n)，最坏遍历所有节点
- **空间复杂度**：O(h)，递归栈深度，h 为树高；最坏 O(n)（退化为链表）

## 踩坑记录

1. **前序 vs 后序的困惑**：`if (root == p || root == q) return root` 这一行看起来是前序（先判断自己），但整体逻辑是"前序判断 + 后序汇总"的混合体。先判断自己是为了提前剪枝，汇总左右结果是后序的思想。
2. **p 是 q 的祖先时为什么正确**：当节点5命中 p 后直接返回，不继续往下找 q（q=4 在5的子树里）。这没问题，因为如果 p 是 q 的祖先，那 LCA 就是 p 本身，返回 p 就是正确答案。
3. **不需要记录 parent**：递归的 return 机制天然就是"向 parent 汇报"，调用栈本身维护了 parent 链。

## 总结

1. 本质是**后序遍历 + 自底向上汇报**，利用递归调用栈隐式维护 parent 关系
2. 每个节点根据左右子树的返回值做判断：**两边都非 null → 我是 LCA；一边非 null → 透传；都 null → 返回 null**
3. `root == p || root == q` 的提前返回同时处理了"p 是 q 的祖先"和普通命中两种情况
4. 代码只有 6 行核心逻辑，但需要透彻理解递归的汇报机制才能真正掌握
