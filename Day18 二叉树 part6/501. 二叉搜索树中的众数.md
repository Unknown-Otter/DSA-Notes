# 501. 二叉搜索树中的众数

> [LeetCode 501 - Find Mode in Binary Search Tree](https://leetcode.com/problems/find-mode-in-binary-search-tree/)

## 核心思路：普通二叉树 vs BST

**普通二叉树**：遍历序列中相同值可能散落各处 → 必须用 HashMap 做全局统计

**BST**：中序遍历是有序的，相同值一定挨在一起 → 只需 `pre` 和 `cur` 相邻比较

> BST 的有序性把问题从"全局统计"降级成了"局部相邻比较"

## 做法一：暴力法（适用于任何二叉树）

**思路**：遍历整棵树 → HashMap 统计频率 → 找最大频率 → 收集所有等于最大频率的值

```java
public int[] findMode(TreeNode root) {
    if (root == null) return new int[0];

    // 1. 遍历树，统计频率
    Map<Integer, Integer> map = new HashMap<>();
    searchBST(root, map);

    // 2. 找到最大频率
    int maxFreq = 0;
    for (int freq : map.values()) {
        if (freq > maxFreq) maxFreq = freq;
    }

    // 3. 收集所有频率等于最大频率的值
    List<Integer> result = new ArrayList<>();
    for (Map.Entry<Integer, Integer> entry : map.entrySet()) {
        if (entry.getValue() == maxFreq) result.add(entry.getKey());
    }

    // 4. List<Integer> → int[]
    int[] res = new int[result.size()];
    for (int i = 0; i < result.size(); i++) res[i] = result.get(i);
    return res;
}

void searchBST(TreeNode curr, Map<Integer, Integer> map) {
    if (curr == null) return;
    map.put(curr.val, map.getOrDefault(curr.val, 0) + 1);
    searchBST(curr.left, map);
    searchBST(curr.right, map);
}
```

- **时间复杂度**：O(n log n)（如果排序）或 O(n)（两次遍历）
- **空间复杂度**：O(n)，HashMap 存所有节点

## 做法二：中序遍历 + pre指针（利用BST性质）

**关键洞察**：BST 中序遍历有序，相同值一定相邻。例如 `[1, 2, 2, 3, 3, 3]`，不可能出现 `[1, 2, 3, 2, 3, 3]`

### 技巧一：pre指针实现相邻比较

用 `pre` 指向前一个访问的节点，和 `cur` 比较：

- `pre == null` → 第一个节点，`count = 1`
- `pre.val == cur.val` → 还在同一组，`count++`
- `pre.val != cur.val` → 进入新的一组，`count = 1`

### 技巧二：一次遍历找出所有众数

朴素做法需要遍历两遍（第一遍找 maxCount，第二遍收集结果）。**一次遍历的技巧**：

- `count == maxCount` → 加入结果集
- `count > maxCount` → **更新 maxCount，清空结果集**（之前收集的都作废了），再加入当前值

> **`result.clear()` 是关键** — 发现更高频率时，之前的"伪众数"全部清掉

```java
class Solution {
    ArrayList<Integer> resList;
    int maxCount;
    int count;
    TreeNode pre;

    public int[] findMode(TreeNode root) {
        resList = new ArrayList<>();
        maxCount = 0;
        count = 0;
        pre = null;
        findMode1(root);
        int[] res = new int[resList.size()];
        for (int i = 0; i < resList.size(); i++) res[i] = resList.get(i);
        return res;
    }

    public void findMode1(TreeNode root) {
        if (root == null) return;
        findMode1(root.left);       // 左

        // —— 中：处理当前节点 ——
        int rootValue = root.val;
        if (pre == null || rootValue != pre.val) {
            count = 1;              // 新值或第一个节点
        } else {
            count++;                // 相同值，频率+1
        }

        if (count > maxCount) {     // 发现更高频率
            resList.clear();        // 清空之前的结果！
            resList.add(rootValue);
            maxCount = count;
        } else if (count == maxCount) {  // 并列最高频
            resList.add(rootValue);
        }
        pre = root;                 // 更新pre

        findMode1(root.right);      // 右
    }
}
```

- **时间复杂度**：O(n)，一次遍历
- **空间复杂度**：O(1)（不算递归栈）

## 踩坑记录

1. **忘记 `result.clear()`**：当 `count > maxCount` 时，如果只更新 maxCount 而不清空 result，结果里会混入之前频率较低的值
2. **pre 初始化为 null**：不能随意初始化为某个节点，null 用来判断"这是第一个被处理的节点"
3. **暴力法的 Stream 写法不好读**：用两次基础 for 循环（先找 maxFreq，再收集结果）更清晰，而且省掉了排序的 O(n log n)

## 总结

1. **BST 中序遍历有序** → 相同值一定相邻 → `pre` 指针做相邻比较就够了，不需要 HashMap
2. **pre 指针** 是在树上模拟"有序数组相邻元素比较"的标准技巧（530.最小绝对差 也用过）
3. **`count > maxCount` 时清空结果集** 是一次遍历找所有众数的核心技巧
4. 普通二叉树只能暴力 HashMap；BST 利用有序性把空间从 O(n) 降到 O(1)
