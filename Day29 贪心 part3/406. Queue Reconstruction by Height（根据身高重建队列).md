# 406. Queue Reconstruction by Height（根据身高重建队列）

> [LeetCode 链接](https://leetcode.com/problems/queue-reconstruction-by-height/)

## 题目理解

每个人用 `[h, k]` 表示：
- **h**：这个人的身高
- **k**：在他前面**身高 ≥ 他**的人数（**注意是大于等于，不是严格大于**）

输入是乱序的 `people` 数组，要求根据这些属性重建出真正的队列顺序。

**关键陷阱**：`k` 的定义是"≥"，所以同样身高的人会被算进计数里。比如 `[5, 2]` 表示前面有 2 个人身高 ≥ 5。

## 核心思路：贪心 + 插入

### 关键洞察

> **如果先把高的人放进队列，后面再插入矮的人，不会影响已放好的高个子的 k 值**——因为矮的人不会被算进"≥"的计数里。

基于这个观察，策略是：

1. **从高到低**处理每个人
2. 对当前要插入的人 `[h, k]`，由于已放进队列的人都 ≥ h，所以他的 **k 值就是他应该插入的下标**

### 为什么身高相同时 k 要升序？

考虑 `[7, 0]` 和 `[7, 1]`：

- `[7, 1]` 表示前面要有 1 个人 ≥ 7
- 如果先插 `[7, 1]`（位置 1），数组还为空，无法插入到位置 1
- 应该先插 `[7, 0]` 到位置 0，再插 `[7, 1]` 到位置 1
- 这样 `[7, 1]` 前面正好有一个 `[7, 0]`，满足条件

所以身高相同时，**k 小的先放**（k 升序）。

## 完整解题步骤

### Step 1：排序

按以下规则排序：
- 身高 **降序**（高的在前）
- 身高相同时，k **升序**（k 小的在前）

### Step 2：插入

遍历排好序的数组，对每个人 `[h, k]`，把他插入到结果列表的下标 `k` 处。

### Step 3：转换返回

把 `List<int[]>` 转成 `int[][]` 返回。

## 模拟过程（Example 1）

输入：`[[7,0],[4,4],[7,1],[5,0],[6,1],[5,2]]`

**排序后**：`[[7,0],[7,1],[6,1],[5,0],[5,2],[4,4]]`

**依次插入**：

```
插 [7,0] 到位置 0  →  [[7,0]]
插 [7,1] 到位置 1  →  [[7,0],[7,1]]
插 [6,1] 到位置 1  →  [[7,0],[6,1],[7,1]]
插 [5,0] 到位置 0  →  [[5,0],[7,0],[6,1],[7,1]]
插 [5,2] 到位置 2  →  [[5,0],[7,0],[5,2],[6,1],[7,1]]
插 [4,4] 到位置 4  →  [[5,0],[7,0],[5,2],[6,1],[4,4],[7,1]]
```

## Java 完整代码

```java
class Solution {
    public int[][] reconstructQueue(int[][] people) {
        // 1. 排序：身高降序，身高相同时 k 升序
        Arrays.sort(people,
            Comparator.comparingInt((int[] a) -> a[0]).reversed()
                      .thenComparingInt(a -> a[1])
        );

        // 2. 按 k 值依次插入到第 k 个位置
        List<int[]> queue = new LinkedList<>();
        for (int[] p : people) {
            queue.add(p[1], p);
        }

        return queue.toArray(new int[people.length][]);
    }
}
```

## 复杂度分析

- **时间复杂度**：O(n²)
  - 排序：O(n log n)
  - 插入循环 n 次，每次 `LinkedList.add(index, e)` 最坏 O(n)
- **空间复杂度**：O(n)，结果列表的开销

# 附录一：Java 排序写法详解

这道题的排序是难点，下面系统梳理 Java 中对 `int[][]` 排序的几种写法。

## 通用规则：Comparator 的返回值

`Comparator.compare(a, b)` 的返回值约定：
- **负数** → a 排在 b 前面
- **正数** → a 排在 b 后面
- **0** → 顺序不变

由此可以推出口诀：
- `a - b` → **升序**（a 小返回负数，a 在前）
- `b - a` → **降序**（a 大返回负数，a 在前）

## 写法 1：Lambda + if-else（最直观）

```java
Arrays.sort(people, (a, b) -> {
    if (a[0] != b[0]) {
        return b[0] - a[0];  // 身高降序：b 减 a
    } else {
        return a[1] - b[1];  // k 升序：a 减 b
    }
});
```

**适用场景**：逻辑稍复杂、需要多个分支判断时最清晰。

## 写法 2：Lambda + 三元表达式（紧凑）

```java
Arrays.sort(people, (a, b) ->
    a[0] != b[0] ? b[0] - a[0] : a[1] - b[1]
);
```

逻辑和写法 1 完全一致，只是用三元运算符压成一行。

**适用场景**：双键排序、一行能写清楚时。

## 写法 3：传统匿名类（啰嗦但清晰）

```java
Arrays.sort(people, new Comparator<int[]>() {
    @Override
    public int compare(int[] a, int[] b) {
        if (a[0] != b[0]) {
            return b[0] - a[0];
        }
        return a[1] - b[1];
    }
});
```

**适用场景**：Java 7 及以前的代码、或在不支持 Lambda 的语境下。Lambda 本质上就是这种写法的语法糖。

## 写法 4：Comparator 链式调用（最优雅）

```java
Arrays.sort(people,
    Comparator.comparingInt((int[] a) -> a[0]).reversed()
              .thenComparingInt(a -> a[1])
);
```

**逐步拆解**：

1. `Comparator.comparingInt((int[] a) -> a[0])`
   按 `a[0]`（身高）**升序**比较。

2. `.reversed()`
   反转顺序 → 变成**降序**。

3. `.thenComparingInt(a -> a[1])`
   身高相等时，再按 `a[1]`（k 值）**升序**比较。

**坑点**：第一个 Lambda 里必须显式写 `(int[] a)`，否则 Java 类型推断会失败。这是这种写法最容易踩的坑。

**适用场景**：写多键排序、想要代码读起来像英语。

## 写法 5：用 Integer.compare 避免溢出

```java
Arrays.sort(people, (a, b) -> {
    if (a[0] != b[0]) return Integer.compare(b[0], a[0]);
    return Integer.compare(a[1], b[1]);
});
```

**为什么需要它**：`b[0] - a[0]` 在两个数都很大或一正一负且绝对值很大时，可能**整数溢出**，导致比较结果错误。

例如 `a = Integer.MIN_VALUE`, `b = 1`，那么 `b - a` 会溢出为负数，排序就乱了。

**适用场景**：
- 数据范围接近 `Integer.MAX_VALUE` / `Integer.MIN_VALUE`
- 包含负数
- 面试中作为防御性写法

本题数据范围 `1 ≤ h ≤ 10⁶`，`0 ≤ k < 2000`，**不会溢出**，但养成习惯用 `Integer.compare` 是个好习惯。

## 排序写法对比总结

| 写法 | 简洁度 | 可读性 | 推荐场景 |
|------|--------|--------|---------|
| 写法 1（if-else） | 中 | ⭐⭐⭐⭐⭐ | 平时刷题、面试白板 |
| 写法 2（三元） | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 简短双键排序 |
| 写法 3（匿名类） | ⭐ | ⭐⭐⭐ | 老 Java 项目 |
| 写法 4（链式） | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 体现 Java 8 功底 |
| 写法 5（防溢出） | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 大数 / 含负数 / 严格代码 |

# 附录二：Java 知识点系统梳理

## 1. `Arrays.sort(T[] a, Comparator<? super T> c)`

对**对象数组**（包括 `int[][]`，因为 `int[]` 本身是对象）进行排序。

**注意**：基本类型数组如 `int[]` 不能用 `Comparator`，只能用 `Arrays.sort(int[] a)` 默认升序排。如果想降序，需要先转成 `Integer[]`。

```java
int[][] arr = {{3, 1}, {1, 2}};   // 可以用 Comparator
int[] arr2 = {3, 1, 2};            // 只能默认升序
Integer[] arr3 = {3, 1, 2};        // 可以用 Comparator
```

## 2. Lambda 表达式

### 基本语法

```java
(参数列表) -> 表达式或语句块
```

例如：
```java
(a, b) -> a - b              // 单表达式，自动 return
(a, b) -> { return a - b; }  // 带 return 的语句块
```

### Lambda 的本质

Lambda 是**函数式接口**（只有一个抽象方法的接口）的简写形式。

`Comparator<T>` 是函数式接口，只有一个抽象方法 `compare(T a, T b)`，所以可以用 Lambda 实现。

### 类型推断规则

Java 编译器会根据上下文推断 Lambda 的参数类型：

```java
Arrays.sort(people, (a, b) -> ...);   // a, b 推断为 int[]
```

但在某些情况下推断会失败（比如写法 4 的 `comparingInt`），这时需要**显式声明类型**：

```java
Comparator.comparingInt((int[] a) -> a[0])  // 必须写 (int[] a)
```

## 3. Comparator 接口家族

### 静态工厂方法

| 方法 | 用途 |
|------|------|
| `Comparator.comparing(keyFn)` | 按某个键排序（返回的键是对象） |
| `Comparator.comparingInt(keyFn)` | 按 int 键排序（避免装箱开销） |
| `Comparator.comparingLong(keyFn)` | 按 long 键排序 |
| `Comparator.comparingDouble(keyFn)` | 按 double 键排序 |
| `Comparator.naturalOrder()` | 自然顺序（升序） |
| `Comparator.reverseOrder()` | 反向自然顺序（降序） |

### 链式默认方法

| 方法 | 用途 |
|------|------|
| `.reversed()` | 反转当前比较器 |
| `.thenComparing(other)` | 第一个键相等时用 `other` 比较 |
| `.thenComparingInt(keyFn)` | 第一个键相等时按 int 键比较 |

### 链式调用示例对照

```java
// 单键升序
Comparator.comparingInt((int[] a) -> a[0])

// 单键降序
Comparator.comparingInt((int[] a) -> a[0]).reversed()

// 双键：第一键降序，第二键升序
Comparator.comparingInt((int[] a) -> a[0]).reversed()
          .thenComparingInt(a -> a[1])

// 双键：第一键升序，第二键降序
Comparator.comparingInt((int[] a) -> a[0])
          .thenComparing(Comparator.comparingInt((int[] a) -> a[1]).reversed())
```

## 4. List 的两种实现：ArrayList vs LinkedList

| 操作 | ArrayList | LinkedList |
|------|-----------|------------|
| `get(i)` 随机访问 | **O(1)** | O(n) |
| `add(e)` 末尾添加 | 均摊 O(1) | O(1) |
| `add(i, e)` 中间插入 | O(n)，需搬移元素 | O(n)，需遍历到位置 |
| `remove(i)` 中间删除 | O(n) | O(n) |
| 内存占用 | 较小 | 较大（每个节点有指针） |

### 本题为什么选 LinkedList？

主要是**习惯性选择**——感觉中间插入是 LinkedList 的强项。但实际上：

- LinkedList 的 `add(i, e)` 也需要 O(n) 遍历到位置
- ArrayList 的 `add(i, e)` 虽然要搬移元素，但底层是连续数组，**缓存友好，常数小**

**实测在 LeetCode 上，ArrayList 反而更快**。但从语义上 LinkedList 更"贴切"，所以教学代码常用 LinkedList。

## 5. List 转数组

```java
List<int[]> list = ...;
int[][] result = list.toArray(new int[list.size()][]);
```

**为什么传一个空数组？**

`toArray()` 的两种重载：
- `toArray()` 返回 `Object[]`，需要强转
- `toArray(T[] a)` 返回 `T[]`，类型安全

传 `new int[size][]` 是为了告诉编译器返回类型是 `int[][]`。注意第二维**不要指定长度**，否则会浪费内存。

## 6. 增强 for 循环（for-each）

```java
for (int[] p : people) {
    // p 是 people 中的每个元素
}
```

等价于：

```java
for (int i = 0; i < people.length; i++) {
    int[] p = people[i];
}
```

**适用场景**：只读遍历，不需要下标。如果需要下标或要修改数组元素引用，用传统 for 循环。

# 总结

1. **核心思想**：贪心 + 插入。先把身高高的人放好，因为后插入的矮个子不会影响他们的 k 值。
2. **排序规则**：身高降序，身高相同时 k 升序。
3. **插入位置**：每个人的 k 值就是他在结果数组中的目标下标。
4. **数据结构选择**：`LinkedList` 用来频繁中间插入（`ArrayList` 实测更快，但 LinkedList 语义更直观）。
5. **Java 知识点**：掌握至少两种排序写法（推荐 if-else 和链式），熟悉 `Comparator` 的链式 API，理解 Lambda 的类型推断规则。
