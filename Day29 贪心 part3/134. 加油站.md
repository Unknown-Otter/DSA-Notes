# 134. Gas Station 加油站

> [LeetCode 134](https://leetcode.com/problems/gas-station/)

## 题目描述

环形公路上有 `n` 个加油站：

- `gas[i]`：在第 `i` 个加油站可以加多少升油
- `cost[i]`：从第 `i` 个加油站开到第 `i+1` 个加油站消耗多少升油
- 油箱容量无限，初始为空

找一个起点，使得汽车能顺时针绕环路一周回到起点。如果存在则返回起点 index，否则返回 `-1`。题目保证解唯一。

---

## 题意理解的关键点

1. **station 编号 = 数组的 index**，从 0 开始
2. **初始油箱为空**，起点的"初始油量"来自起点加油站给的 `gas[start]`
3. `cost[i]` 是**从 i 开到 i+1 的路费**，不是"在 i 加油的费用"（加油是免费的）
4. **环形结构**：最后一个站 `n-1` 的下一站是 `0`
5. **任何一步油箱都不能 < 0**，否则中途抛锚

### 走一遍 Example 1

`gas = [1,2,3,4,5]`, `cost = [3,4,5,1,2]`，从 station 3 出发：

```
站 3：加油 gas[3]=4              → tank = 0 + 4 = 4
开到站 4：路费 cost[3]=1，加油 gas[4]=5  → tank = 4 - 1 + 5 = 8
开到站 0：路费 cost[4]=2，加油 gas[0]=1  → tank = 8 - 2 + 1 = 7
开到站 1：路费 cost[0]=3，加油 gas[1]=2  → tank = 7 - 3 + 2 = 6
开到站 2：路费 cost[1]=4，加油 gas[2]=3  → tank = 6 - 4 + 3 = 5
开回站 3：路费 cost[2]=5                 → tank = 5 - 5 = 0 ✅
```

---

## 核心递推公式

最简洁的写法是把"在站 i 加油"和"离开 i 时付路费"合并到一步：

```
diff[i] = gas[i] - cost[i]   // 在站 i 净赚/净亏的油量
tank   += diff[i]            // 累加油量
total  += diff[i]            // 累加全局收支
```

**物理意义**：到达站 i，加油 `gas[i]`，出发时付掉路费 `cost[i]`，剩下进油箱。

---

## 解法一：暴力 O(n²)

枚举每个起点，模拟一圈。**用 while 模拟环形遍历**。

```java
class Solution {
    public int canCompleteCircuit(int[] gas, int[] cost) {
        int n = gas.length;
        for (int i = 0; i < n; i++) {
            int rest = gas[i] - cost[i];
            int index = (i + 1) % n;
            while (rest > 0 && index != i) {  // 模拟绕一圈
                rest += gas[index] - cost[index];
                index = (index + 1) % n;
            }
            if (rest >= 0 && index == i) return i;
        }
        return -1;
    }
}
```

- 时间复杂度：O(n²)
- 空间复杂度：O(1)

**技巧**：`for` 循环适合从头到尾遍历，`while` 循环适合环形遍历。

## 解法二：贪心 O(n)（重点）

### 核心性质

> 如果从 station A 出发，累加到 station B 时 `tank` **第一次变负**，那么 A、A+1、…、B 这些站**都不可能**作为合法起点。

**证明**：
- 从 A 出发到 A+1 时，`tank ≥ 0`（设为 `x`）。如果改成从 A+1 出发，油箱从 0 开始，到 B 时油量比"A 版本"少了 `x`
- A 版本到 B 已经 `< 0`，所以 A+1 版本到 B `≤ A版本 - x < 0`
- 同理 A+2、…、B 都不行

所以一旦 `tank < 0`，**直接把起点跳到 `i + 1`**，跳过整段。

### 局部最优 → 全局最优

- **局部最优**：累加 `rest[i]` 一旦 `< 0`，起点至少是 `i+1`
- **全局最优**：找到能跑完一圈的起点
- **可行性判断**：`total = 总gas - 总cost ≥ 0`，否则无解

### 标准代码

```java
class Solution {
    public int canCompleteCircuit(int[] gas, int[] cost) {
        int curSum = 0;
        int totalSum = 0;
        int start = 0;
        for (int i = 0; i < gas.length; i++) {
            curSum += gas[i] - cost[i];
            totalSum += gas[i] - cost[i];
            if (curSum < 0) {       // 当前段亏本
                start = i + 1;      // 起点跳到 i+1
                curSum = 0;         // 油箱清零，重新开始
            }
        }
        if (totalSum < 0) return -1;  // 总油量 < 总消耗，无解
        return start;
    }
}
```

- 时间复杂度：O(n)
- 空间复杂度：O(1)

### 用 Example 1 验证

`gas = [1,2,3,4,5]`, `cost = [3,4,5,1,2]`

| i | gas[i]-cost[i] | curSum | totalSum | curSum<0? | start |
|---|----------------|--------|----------|-----------|-------|
| 0 | 1-3 = -2 | -2 | -2 | ✅ | **1**, curSum=0 |
| 1 | 2-4 = -2 | -2 | -4 | ✅ | **2**, curSum=0 |
| 2 | 3-5 = -2 | -2 | -6 | ✅ | **3**, curSum=0 |
| 3 | 4-1 = 3 | 3 | -3 | ❌ | 3 |
| 4 | 5-2 = 3 | 6 | 0 | ❌ | 3 |

`totalSum = 0 ≥ 0`，返回 `start = 3` ✅

### Bug 版本


**问题**：递推用了 `gas[(i+1) % n] - cost[i]`，物理意义错了。

- 我的本意是"开到下一站，付路费 + 加下一站的油"
- 但这样**永远漏算了起点 station 的 gas**
- 看似 Example 1 能 AC（因为错位是一致的，且总和不变），但遇到亏本段分散的 case 会翻车

### 教训

1. **递推的物理意义要清楚**。`gas[i] - cost[i]` 语义干净：在站 i，加油后付路费，净收支
2. **错位累加的总和虽然相同，但中间状态不同**——亏本段分散的时候判断点会错
3. 不要追求"看起来差不多对"的写法，写清楚物理量

---

## 关键 takeaway

1. **起点为什么是 `i+1` 不是 `i`**：因为 `i` 这个站本身就亏本（`gas[i] - cost[i] < 0`），从 `i` 出发自己都开不出去
2. **为什么只需要一次遍历**：贪心性质保证最后一次重置的 `start` 就是答案，配合 `totalSum ≥ 0` 验证整体可行性，不需要再模拟一圈
3. **DP vs 贪心**：这题路径唯一、决策唯一，没有重叠子问题，**贪心更自然**，DP 反而把问题搞复杂

## 总结模板

```java
// 贪心标准模板
int curSum = 0, totalSum = 0, start = 0;
for (int i = 0; i < n; i++) {
    int diff = gas[i] - cost[i];
    curSum += diff;
    totalSum += diff;
    if (curSum < 0) {
        start = i + 1;
        curSum = 0;
    }
}
return totalSum >= 0 ? start : -1;
```

1. **求差**：`diff[i] = gas[i] - cost[i]`
2. **累加 + 判负**：`curSum < 0` 就重置起点和 curSum
3. **全局判可行**：`totalSum < 0` 返回 -1
4. **返回最后的 start**
