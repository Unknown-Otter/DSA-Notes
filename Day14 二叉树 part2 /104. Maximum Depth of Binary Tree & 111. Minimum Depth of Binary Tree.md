# 104. Maximum Depth of Binary Tree

## 思路

求二叉树的最大深度,即从根节点到最远叶子节点的最长路径上的**节点数**。

使用**递归(后序遍历)**思想:

1. **相信递归**:假设 `maxDepth(root.left)` 已经能正确返回左子树的最大深度,`maxDepth(root.right)` 同理。
2. **base case**:`root == null` 时返回 0(空树深度为 0)。
3. **当前节点要做的事**:取左右子树深度的**较大值**,再 `+1`(算上自己这一层)。

**关键理解:下行不算,上行才算**

- 递归调用时,只是一路往下"试探",不做任何加法。
- 真正的计算发生在 `return` 时——每往上弹一层,就 `+1`。
- `+1` 跟节点存的值无关,数的是**层数**。

## 代码

```java
class Solution {
    /**
     * 递归法(后序遍历)
     */
    public int maxDepth(TreeNode root) {
        if (root == null) {
            return 0;
        }
        int leftDepth = maxDepth(root.left);
        int rightDepth = maxDepth(root.right);
        return Math.max(leftDepth, rightDepth) + 1;
    }
}
```

## 复杂度

- **时间复杂度**:`O(n)`,n 为节点总数,每个节点访问一次。
- **空间复杂度**:`O(h)`,h 为树的高度(递归栈深度)。最坏情况(链状树)为 `O(n)`,平衡树为 `O(log n)`。

## 走查示例

以下面这棵树为例:

```
        3
       / \
      9   20
         /  \
        15   7
```

| 调用 | leftDepth | rightDepth | 返回值 |
|------|-----------|------------|--------|
| `maxDepth(9)` | 0 (null) | 0 (null) | `max(0,0)+1 = 1` |
| `maxDepth(15)` | 0 (null) | 0 (null) | `max(0,0)+1 = 1` |
| `maxDepth(7)` | 0 (null) | 0 (null) | `max(0,0)+1 = 1` |
| `maxDepth(20)` | 1 | 1 | `max(1,1)+1 = 2` |
| `maxDepth(3)` | 1 | 2 | `max(1,2)+1 = 3` ✅ |

## 总结

1. **后序遍历模板**:先递归左右,最后处理当前节点。
2. **+1 发生在回程**:不是"往下走 +1",而是"往上返回时 +1"。
3. **对 max 来说,null 返回 0 是安全的**:因为 max 会自动选另一边更大的值,空孩子不会影响结果。

# 111. Minimum Depth of Binary Tree

## 思路

求二叉树的最小深度,即从根节点到**最近叶子节点**的最短路径上的节点数。

**关键定义**:**叶子节点 = 左右孩子都为 null 的节点**。这是 LC 111 的坑所在。

### 为什么不能直接把 LC 104 的 max 改成 min?

考虑这棵树:

```
    1
   /
  2
```

正确答案是 **2**(根 1 → 叶子 2)。

如果天真地写 `Math.min(leftDepth, rightDepth) + 1`:

- 节点 1 的右孩子是 null → `rightDepth = 0`
- `min(1, 0) + 1 = 1` ❌

错在哪?**节点 1 不是叶子**(它还有左孩子),但 `min` 把右边那个 null 返回的 0 当成了一条合法路径,提前"截断"了。

### 修正思路

需要在比较前判断左右孩子是否为 null:

1. `root == null` → 返回 0
2. **左孩子为 null** → 答案是 `右子树深度 + 1`(只能往右走)
3. **右孩子为 null** → 答案是 `左子树深度 + 1`(只能往左走)
4. **两边都存在** → 才能安全用 `min(leftDepth, rightDepth) + 1`

## 代码

```java
class Solution {
    public int minDepth(TreeNode root) {
        if (root == null) return 0;
        
        int leftDepth = minDepth(root.left);
        int rightDepth = minDepth(root.right);
        
        // 左孩子为 null,只能走右边(叶子情况也被这里覆盖:rightDepth=0,返回 1)
        if (root.left == null) return rightDepth + 1;
        
        // 右孩子为 null,只能走左边
        if (root.right == null) return leftDepth + 1;
        
        // 左右都存在,才能用 min
        return Math.min(leftDepth, rightDepth) + 1;
    }
}
```

## 复杂度

- **时间复杂度**:`O(n)`,每个节点访问一次。
- **空间复杂度**:`O(h)`,h 为树的高度。

## 走查示例

继续用反例:

```
    1
   /
  2
```

| 调用 | leftDepth | rightDepth | 走哪个分支 | 返回值 |
|------|-----------|------------|-----------|--------|
| `minDepth(2)` | 0 | 0 | `root.left == null` | `0 + 1 = 1` ✅ |
| `minDepth(1)` | 1 | 0 | `root.right == null` | `1 + 1 = 2` ✅ |

## 踩坑笔记 ⚠️

### 坑 1:重复递归调用

错误写法:

```java
int leftDepth = minDepth(root.left);
int rightDepth = minDepth(root.right);

if (root.left == null) return minDepth(root.right) + 1;  // ❌ 又算一遍!
```

正确写法:用上面已经存好的变量 `rightDepth + 1`,避免指数级重复计算。

### 坑 2:判断对象错了

错误写法:

```java
if (root.left.left == null && root.left.right == null)  // ❌ 这是判断 root.left 是不是叶子
```

正确做法:判断 `root.left == null`(当前节点的孩子是不是 null),不是孙子辈。

### 坑 3:漏掉叶子节点的情况

实际上不需要单独写 `if (root.left == null && root.right == null)`,因为:
- 当节点是叶子时,`leftDepth = 0`, `rightDepth = 0`
- 走第一个 if (`root.left == null`),返回 `0 + 1 = 1` ✅
- 自动被覆盖了

## 总结

| 题目 | 关键差异 |
|------|---------|
| **LC 104 最大深度** | null 返回 0 没问题,`max` 会自动忽略空孩子 |
| **LC 111 最小深度** | null 返回 0 会污染 `min`,**必须单独判断左右是否为 null** |

# 通用递归思维框架(树类问题)

这套思维可以套到大部分二叉树递归题:

1. **相信递归**:假设递归函数对子树已经能正确工作。
2. **找 base case**:通常是 `root == null` 返回什么(0、true、null 等)。
3. **下行不算,上行才算**:递归调用只是往下走,真正的计算在 `return` 那一行。
4. **当前节点的贡献**:常见是 `+1`(节点计数)或更新某个全局变量。
5. **如何汇总左右**:根据题目选 `max`、`min`、`+`、`&&`、`||` 等。

## 同类题目对比

| 题目 | 上行时做的事 |
|------|-------------|
| LC 104 最大深度 | `max(left, right) + 1` |
| LC 111 最小深度 | 判 null 后 `min(left, right) + 1` |
| LC 222 节点总数 | `left + right + 1` |
| LC 110 是否平衡 | 检查 `\|left - right\| <= 1` |
| LC 543 直径 | 更新全局答案 `left + right` |
