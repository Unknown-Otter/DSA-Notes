# 101. 对称二叉树

> [LeetCode 101. Symmetric Tree](https://leetcode.com/problems/symmetric-tree/)

## 思路

判断一棵树是否轴对称，本质是判断 **根的左子树和右子树是否互为镜像**。

镜像的定义：
- 两棵子树的**根节点值相等**
- 左子树的**外侧**（left.left）== 右子树的**外侧**（right.right）
- 左子树的**内侧**（left.right）== 右子树的**内侧**（right.left）

所以这道题不是单棵树的遍历，而是**两棵树成对比较**。无论用递归还是迭代，每次都要从队列/栈里**一次取出两个节点**做比较。

## 代码

### 递归法

```java
public boolean isSymmetric(TreeNode root) {
    return compare(root.left, root.right);
}

private boolean compare(TreeNode left, TreeNode right) {
    // 先穷尽 null 的情况，再比较值
    if (left == null && right == null) return true;
    if (left == null || right == null) return false;
    if (left.val != right.val) return false;

    boolean outside = compare(left.left, right.right);   // 外侧
    boolean inside  = compare(left.right, right.left);   // 内侧
    return outside && inside;
}
```

### 迭代法（队列）

队列必须用 **`LinkedList`**，不能用 `ArrayDeque`（详见踩坑 3）。

```java
public boolean isSymmetric(TreeNode root) {
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(root.left);
    queue.offer(root.right);

    while (!queue.isEmpty()) {
        TreeNode leftNode  = queue.poll();
        TreeNode rightNode = queue.poll();

        if (leftNode == null && rightNode == null) continue;
        if (leftNode == null || rightNode == null
                || leftNode.val != rightNode.val) {
            return false;
        }

        // 成对入队：外侧配外侧，内侧配内侧
        queue.offer(leftNode.left);
        queue.offer(rightNode.right);
        queue.offer(leftNode.right);
        queue.offer(rightNode.left);
    }
    return true;
}
```

### 迭代法（双端队列当两个栈用）

把 deque 两端各当成一个栈，左侧从外往里入，右侧从外往里入，pollFirst / pollLast 配对取出。

```java
public boolean isSymmetric(TreeNode root) {
    Deque<TreeNode> deque = new LinkedList<>();
    deque.offerFirst(root.left);
    deque.offerLast(root.right);

    while (!deque.isEmpty()) {
        TreeNode leftNode  = deque.pollFirst();
        TreeNode rightNode = deque.pollLast();

        if (leftNode == null && rightNode == null) continue;
        if (leftNode == null || rightNode == null
                || leftNode.val != rightNode.val) {
            return false;
        }

        deque.offerFirst(leftNode.left);
        deque.offerFirst(leftNode.right);
        deque.offerLast(rightNode.right);
        deque.offerLast(rightNode.left);
    }
    return true;
}
```

## 复杂度

- 时间：**O(n)**，每个节点最多入队/递归一次。
- 空间：**O(n)**，最坏情况下队列或递归栈存满一层节点。

## 踩坑记录

### 踩坑 1：误用 `==` 比较 TreeNode 的值

```java
// ❌ 错误：比较的是引用
if (leftNode == rightNode) return true;
if (leftNode != rightNode) return false;
```

`==` 在对象上比较的是**引用地址**，两个值相同的不同节点不会相等。判断节点值必须用 `.val`：

```java
// ✅ 正确
if (leftNode.val != rightNode.val) return false;
```

> 同类坑：`Integer` 的 `==` 在 `[-128, 127]` 范围外也会失效，要用 `.equals()` 或 `.intValue()`。

### 踩坑 2：循环条件用 `while(size-- > 0)` 而 size 只取一次

```java
// ❌ 错误：size 只在循环外算了一次（等于 2），后面入队的节点没机会比较
int size = deque.size();
while (size-- > 0) { ... }
```

- "按层处理"模板（如层序遍历需要分层时）：`size` 必须**每层开始时重新取**。
- 本题不需要分层，直接用 `while (!deque.isEmpty())` 就够了。

### 踩坑 3：`ArrayDeque` 不允许存 `null`，会抛 NPE

```java
Deque<TreeNode> deque = new ArrayDeque<>();
deque.offer(null);  // ❌ NullPointerException
```

源码里 `ArrayDeque.addLast` 第一行就是 `if (e == null) throw new NPE`。

| 实现 | 允许 null | 底层 | 速度 |
|------|----------|------|------|
| `ArrayDeque` | ❌ | 循环数组 | 更快 |
| `LinkedList` | ✅ | 双向链表 | 略慢 |

**原因**：`ArrayDeque` 用 `null` 作为"空槽位"标记，所以禁止用户存 null，否则无法区分"空槽"和"存了 null"。

**踩坑场景汇总**：凡是需要把 `null` 当作"空节点占位符"入队的题，都不能用 `ArrayDeque`：
- 对称二叉树（本题）
- 二叉树的序列化与反序列化（LC 297）
- 每层最右节点 / 完全二叉树判断等需要 null 占位的写法

**口诀**：树题涉及 null 占位 → `LinkedList`；纯非空对象 → `ArrayDeque`（更快）。

> 想绕开 `LinkedList` 也可以——只在两个节点都非 null 时才往队列里塞它们的子节点，但这样代码会更绕，不如直接换实现。

### 踩坑 4：null 判断顺序写反，导致 NPE

```java
// ❌ 错误顺序：先比较 .val，再判断 null
if (leftNode.val != rightNode.val) return false;   // 这里就 NPE 了
if (leftNode == null || rightNode == null) return false;
```

正确顺序是 **先穷尽 null，再访问 `.val`**：

```java
if (leftNode == null && rightNode == null) continue;          // 都为空 → 这一对对称
if (leftNode == null || rightNode == null) return false;      // 一空一不空 → 不对称
if (leftNode.val != rightNode.val) return false;              // 此时两者都非 null，安全访问
```

三条可以合并成一条（短路求值保证安全）：

```java
if (leftNode == null || rightNode == null || leftNode.val != rightNode.val) {
    return false;
}
```

### 踩坑 5：循环结束的 return 写错

```java
return ;  // ❌ 编译错误
return false;  // ❌ 逻辑错误
```

循环正常结束 = 所有对应节点对都比较通过 = 对称，应该 `return true`。**只在循环内部发现不匹配时才 `return false`**，这是"穷举不匹配，最后兜底成功"的经典模板。

## 总结

1. **成对比较**是这类"两棵树/两个子树是否相同"题目的核心模板，每次从队列里 `poll` 两次。
2. **入队顺序**决定逻辑正确性：外侧配外侧（left.left ↔ right.right），内侧配内侧（left.right ↔ right.left）。
3. **null 占位入队** → 必须用 `LinkedList`，不能用 `ArrayDeque`。
4. **判空在前，访问值在后**，否则必 NPE。
5. **循环结束 return true**，循环内 return false。
6. 同模板可解：LC 100（相同的树）、LC 572（另一棵树的子树）、LC 951（翻转等价二叉树）。
