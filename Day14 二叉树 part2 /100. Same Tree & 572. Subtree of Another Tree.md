## 一、100. Same Tree

### 思路

判断两棵树是否**结构相同 + 值相同**。两种写法：递归直接判，或迭代用队列**成对**比较。

迭代法和 `isSymmetric` 几乎一模一样，区别只在于**成对入队的顺序**：

- **对称二叉树**：比较「镜像」位置 → `left.left` 配 `right.right`，`left.right` 配 `right.left`
- **相同二叉树**：比较「相同」位置 → `p.left` 配 `q.left`，`p.right` 配 `q.right`

每次从队列取出两个节点，三种情况：

1. 两个都为 `null` → 当前位置相同，`continue`
2. 一个为 `null` 或值不相等 → 返回 `false`
3. 都不为 `null` 且值相等 → 把左右孩子**成对**入队

### 代码

**递归写法：**

```java
public boolean isSameTree(TreeNode p, TreeNode q) {
    if (p == null && q == null) return true;
    if (p == null || q == null) return false;
    if (p.val != q.val) return false;
    return isSameTree(p.left, q.left)
        && isSameTree(p.right, q.right);
}
```

**迭代写法（队列）：**

```java
public boolean isSameTree(TreeNode p, TreeNode q) {
    Queue<TreeNode> queue = new LinkedList<>();
    queue.offer(p);
    queue.offer(q);
    while (!queue.isEmpty()) {
        TreeNode node1 = queue.poll();
        TreeNode node2 = queue.poll();
        if (node1 == null && node2 == null) continue;
        if (node1 == null || node2 == null
                || node1.val != node2.val) {
            return false;
        }
        // 成对入队：左配左，右配右
        queue.offer(node1.left);
        queue.offer(node2.left);
        queue.offer(node1.right);
        queue.offer(node2.right);
    }
    return true;
}
```

### 复杂度

- **时间复杂度**：O(n)，n 为较小那棵树的节点数（一旦不同就提前返回）
- **空间复杂度**：O(n)，队列最坏情况下要存一层节点；递归栈深度为 O(h)

### 总结

1. **迭代法本质**：用队列/栈模拟递归，每次成对处理两个待比较的节点
2. **和 `isSymmetric` 的唯一区别**：入队顺序不同 —— 一个配镜像，一个配同侧
3. **`null` 处理顺序很重要**：先判断"都为 null"再判断"有一个为 null"，否则会 NPE / 漏判
4. **队列 vs 栈**：两者都可以，因为只在乎"成对取出"，不在乎是 BFS 还是 DFS 顺序。`Deque<TreeNode> stack = new ArrayDeque<>()` + `push/pop` 完全等价

## 二、572. Subtree of Another Tree

### 思路

这道题是 100 题的**进阶应用**，标准套路是 **「双递归 / 嵌套递归」**：

1. **外层 `isSubtree`**：在 `root` 里**遍历每一个节点**，把它当作"候选起点"
2. **内层 `isSameTree`**（就是 100 题）：判断"以当前节点为根的子树" 和 `subRoot` 是否**完全相同**

对每个节点，三种情况只要满足一个就返回 `true`：

- 当前节点开始的子树 == `subRoot`
- 左子树里能找到
- 右子树里能找到

### ⚠️ 易错点：两个函数职责不能混淆

写代码时最容易犯的错是把递归调用全写成 `isSame`：

```java
// ❌ 错误写法
return isSame(root, subRoot)
    || isSame(root.left, subRoot)     // 错！应该是 isSubtree
    || isSame(root.right, subRoot);   // 错！应该是 isSubtree
```

**为什么错？**

- `isSame(root.left, subRoot)`：只比较「`root.left` 这一个节点为根的子树」和 `subRoot` 是否相同 —— **比较一次就结束**
- `isSubtree(root.left, subRoot)`：在 `root.left` 这**整棵子树里继续遍历找**，对每一个节点都尝试匹配

反例：如果 `subRoot` 藏在 `root.left.right.right` 这种很深的位置，用 `isSame(root.left, subRoot)` 就会因为根不匹配直接返回 `false`，根本不会下钻去找。

> **记忆口诀：当前节点用 `isSame`（一次性比较），左右子树用 `isSubtree`（继续遍历找）**
>
> `isSame` 只在「**已经选定了候选起点**」时调用一次；`isSubtree` 负责「**移动起点**」。

### 代码

```java
class Solution {
    public boolean isSubtree(TreeNode root, TreeNode subRoot) {
        // root 走到空都没匹配上 → 找不到
        // (subRoot 题目保证非空，不需要单独判)
        if (root == null) return false;
        // 三种可能：当前节点匹配 / 左子树里有 / 右子树里有
        return isSame(root, subRoot)
            || isSubtree(root.left, subRoot)
            || isSubtree(root.right, subRoot);
    }

    // 复用 100 题的递归写法
    private boolean isSame(TreeNode root1, TreeNode root2) {
        if (root1 == null && root2 == null) return true;
        if (root1 == null || root2 == null) return false;
        if (root1.val != root2.val) return false;
        return isSame(root1.left, root2.left)
            && isSame(root1.right, root2.right);
    }
}
```

### 复杂度

- **时间复杂度**：O(m × n)，m 是 `root` 的节点数，n 是 `subRoot` 的节点数。最坏情况下 `root` 的每个节点都要触发一次完整的 `isSame`
- **空间复杂度**：O(max(h₁, h₂))，递归栈深度

### 总结

1. **核心套路：双函数嵌套**。外层负责**找起点**（遍历），内层负责**判相同**（比较）。这个模式在树形题里非常通用，比如「树的子结构（剑指 Offer 26）」「翻转等价二叉树」都是这个套路
2. **逻辑运算符的对称之美**：
   - `isSubtree` 里用 **`||`**：当前匹配 / 左子树找到 / 右子树找到，**任一**成立即可
   - `isSame` 里用 **`&&`**：左右子树**都**必须相同
   - 两者都享受短路求值的优化
3. **边界两个易错点**：
   - `isSubtree`：`root == null` 时返回 `false`（空树里不可能有非空子树）
   - `isSame`：**「都为 null → true」必须写在「有一个为 null → false」之前**
4. **进阶 O(m+n) 解法**：把两棵树**序列化成带 null 占位符的字符串**（前序 + `#` 标记空节点），然后用 **KMP** 判断 `subRoot` 的序列化是否为 `root` 序列化的子串。等 KMP 复习到了可以回来用这个思路再写一遍，是个很好的串联练习

## 三、两题对比与串联

| 维度 | 100. Same Tree | 572. Subtree of Another Tree |
|------|----------------|-------------------------------|
| 问题本质 | **比较**两棵树是否相同 | **查找** + **比较** |
| 函数数量 | 1 个 | 2 个（嵌套调用） |
| 主逻辑运算符 | `&&`（左右都要同） | `\|\|`（任一找到即可） |
| 100 题的角色 | —— | 作为 572 的**子函数被复用** |
| 时间复杂度 | O(n) | O(m × n) |

**核心启示**：很多树形题都可以拆解为「**遍历子问题** + **判断子问题**」两层。100 是 572 的**判断子问题**，572 是 100 的**遍历包装**。先把 100 写熟，572 就只是多套一层壳。
