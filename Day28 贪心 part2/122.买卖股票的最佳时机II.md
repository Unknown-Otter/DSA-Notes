# 122. 买卖股票的最佳时机 II

> [LeetCode 链接](https://leetcode.cn/problems/best-time-to-buy-and-sell-stock-ii/)

给定一个数组 `prices`，第 `i` 个元素是某支股票第 `i` 天的价格。设计算法计算所能获取的最大利润，可以尽可能多次买卖（但同一时刻只能持有一支股票，再次购买前必须先卖出）。

---

## 一、我的初始思路

**核心想法**：用差分数组 `prefix[i] = prices[i+1] - prices[i]` 记录每天的涨跌，然后分类讨论：

- 如果差分数组**全是正数** → 用最后一天减最便宜那天
- 如果差分数组**全是负数** → 返回 0
- **混合情况** → 把正数加起来

### 思路评估

**核心方向是对的**！这就是贪心解法的雏形 —— 累加所有相邻的正向差值。

但分类讨论是**多余的**：

1. 全正数：相邻正差值之和 = `prices[n-1] - prices[0]`，自动等于"最后减最小"
2. 全负数：正数之和 = 0，自动返回 0
3. 混合：直接加正数

所以**只需要一条逻辑**：把所有正的差值加起来。

### 初始代码的问题

```java
int[] prefix = int[prices.length];   // ❌ 缺少 new 关键字
for(int i = 0; i < prices.length; i++){
    prefix[i] = prices[i+1] - prices[i];   // ❌ i+1 越界
}
if(prefix[i])   // ❌ Java 的 if 必须是 boolean
```

| 问题 | 修正 |
|------|------|
| `int[] prefix = int[...]` | `int[] prefix = new int[n - 1]` |
| 数组长度用 `prices.length` | 差分数组长度应为 `n - 1` |
| 循环到 `i < prices.length` | 应为 `i < n - 1`，否则越界 |
| `if(prefix[i])` | Java 必须写成 `if(prefix[i] > 0)` |
| 没有 return | 必须返回 `profit` |

## 二、保留差分数组的写法（思路直观）

先算出差分数组，再累加正数：

```java
class Solution {
    public int maxProfit(int[] prices) {
        int n = prices.length;
        if (n <= 1) return 0;

        // 差分数组，长度是 n - 1
        int[] diff = new int[n - 1];
        for (int i = 0; i < n - 1; i++) {
            diff[i] = prices[i + 1] - prices[i];
        }

        // 把所有正的差值加起来
        int profit = 0;
        for (int d : diff) {
            if (d > 0) {
                profit += d;
            }
        }
        return profit;
    }
}
```

- **时间复杂度**：O(n)
- **空间复杂度**：O(n)

### 命名小提醒

**差分（difference）** 和 **前缀和（prefix sum）** 是两个不同概念，不要混用：

- 前缀和：`prefix[i] = arr[0] + arr[1] + ... + arr[i]`
- 差分：`diff[i] = arr[i+1] - arr[i]`

## 三、空间优化：一次遍历（贪心标准写法）

中间差分值用完就扔，没必要存数组，边算边加即可：

```java
class Solution {
    public int maxProfit(int[] prices) {
        int profit = 0;
        for (int i = 1; i < prices.length; i++) {
            int diff = prices[i] - prices[i - 1];
            if (diff > 0) {
                profit += diff;
            }
        }
        return profit;
    }
}
```

或者用 `Math.max` 写得更紧凑（教程的写法）：

```java
class Solution {
    public int maxProfit(int[] prices) {
        int result = 0;
        for (int i = 1; i < prices.length; i++) {
            result += Math.max(prices[i] - prices[i - 1], 0);
        }
        return result;
    }
}
```

- **时间复杂度**：O(n)
- **空间复杂度**：O(1)

### 两种写法对比

| 写法 | 时间 | 空间 | 备注 |
|------|------|------|------|
| 先算差分数组 | O(n) | O(n) | 思路直观，分两步走 |
| 一次遍历 | O(n) | O(1) | 时间相同，仅省空间 |

> ⚠️ 注意：**两种写法时间复杂度完全相同**，差别只在空间。差分数组版本严格说是遍历两次（2n），但仍是 O(n)。

### 优化的关键观察

如果中间结果**只被顺序使用一次**，就不需要存数组 —— 边算边用即可。这是常见的空间优化模式。

## 四、贪心思路推导（教程讲解）

### 利润可以拆分

假设第 0 天买入，第 3 天卖出，利润为 `prices[3] - prices[0]`。

这等价于：

```
(prices[3] - prices[2]) + (prices[2] - prices[1]) + (prices[1] - prices[0])
```

把整段利润**拆成每天为单位**，问题就转化为：在每天的涨跌序列里挑选哪些天的利润。

### 只收集正利润

由 `prices` 得到每天的利润序列：`prices[i] - prices[i - 1]`（注意利润序列比股票序列少一天，第一天没有利润）。

- **局部最优**：收集每天的正利润
- **全局最优**：求得最大利润

只取正的、跳过负的，找不出反例 → 贪心成立。

收集正利润的连续区间，对应的就是一次买卖区间，但我们只关心最终利润，**不需要真的记录买卖点**。

## 五、动态规划解法（教程附带，留作以后复习）

定义状态：

- `dp[i][0]`：第 `i` 天**持有现金**（不持股）的最大现金
- `dp[i][1]`：第 `i` 天**持有股票**的最大现金

状态转移：

- `dp[i][0] = max(dp[i-1][0], dp[i-1][1] + prices[i])`（不动 / 卖出）
- `dp[i][1] = max(dp[i-1][1], dp[i-1][0] - prices[i])`（不动 / 买入）

```java
class Solution {
    public int maxProfit(int[] prices) {
        // [天数][是否持有股票]
        int[][] dp = new int[prices.length][2];

        // base case
        dp[0][0] = 0;
        dp[0][1] = -prices[0];

        for (int i = 1; i < prices.length; i++) {
            dp[i][0] = Math.max(dp[i - 1][0], dp[i - 1][1] + prices[i]);
            dp[i][1] = Math.max(dp[i - 1][1], dp[i - 1][0] - prices[i]);
        }

        return dp[prices.length - 1][0];
    }
}
```

- **时间复杂度**：O(n)
- **空间复杂度**：O(n)（可优化到 O(1)，只保留前一天状态）

## 六、总结

1. **关键洞察**：把整段利润 `prices[j] - prices[i]` 拆成每天相邻差值之和，问题简化为"挑哪些天的差值"。
2. **贪心策略**：只累加正的相邻差值，负的跳过 —— 局部最优推出全局最优。
3. **从差分数组到一次遍历**：中间结果只顺序用一次时，可以省掉数组，把空间压到 O(1)。
4. **差分 vs 前缀和**：两个不同概念，命名要分清。
5. **本题贪心比 DP 更巧妙**，但 DP 是股票系列的通用解法，后面的变种题（含手续费、冷冻期、限制次数）需要 DP。
