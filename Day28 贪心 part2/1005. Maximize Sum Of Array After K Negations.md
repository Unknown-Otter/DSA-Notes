# 1005. Maximize Sum Of Array After K Negations

> [LeetCode 1005](https://leetcode.com/problems/maximize-sum-of-array-after-k-negations/)

## 题目描述

给定整数数组 `nums` 和整数 `k`，每次操作选择一个下标 `i` 把 `nums[i]` 替换为 `-nums[i]`。必须**恰好**执行 `k` 次操作（同一个下标可以重复选）。求操作后数组的最大和。

## 核心思想

> **贪心策略：每一步都翻转当前数组里最小的那个数。**

翻转一个数 `x` 对总和的贡献是 `-2x`：
- `x` 越小（越靠数轴左边），`-2x` 越大，越值得翻
- 不管 `x` 是负数、0、还是正数，**"翻最小的"永远是当前最优选择**

| x 的状态 | -2x（贡献） | 排序中的位置 |
|---------|-----------|-------------|
| 很大的负数（如 -10）| +20 | 最左边 |
| 较小的负数（如 -1）| +2 | 偏左 |
| 0 | 0 | 中间 |
| 较小的正数（如 1）| -2 | 偏右 |
| 很大的正数（如 10）| -20 | 最右边 |

## 关键观察：为什么贪心成立

题目允许**重复选位置**，这意味着：
- 翻同一个数两次 = 没翻（可以"撤销"）
- 多余的 `k` 总能通过"反复横跳"消耗掉

所以无论 `k` 多大，都不会卡住贪心策略：
- `k` 用不完？多出来的偶数次成对抵消，奇数次砸在损失最小的数上

## 解题步骤

### 第一步：从小到大排序

让"最小的"在最左边，方便从左往右处理。

### 第二步：翻转负数（从左往右）

只要还有负数且 `k > 0`，就翻它。每翻一个，`k--`。

**为什么不用每次重排？** 排序后翻负数 `-x` 得到 `+x`，由于 `x` 的绝对值通常 ≥ 剩余元素，新值会自动落到"已处理"的位置，剩余未处理部分仍然有序。所以"按位置走 = 翻最小的"在这个阶段成立。

### 第三步：处理剩余的 k

循环退出后，`k` 可能还没用完（数组没负数了，但 `k` 还 > 0）：

- **`k` 是偶数**：成对抵消，啥也不用做 ✅
- **`k` 是奇数**：必须实质翻 1 次，**重新排序**找到当前最小的，翻它（损失最少）

**为什么要重新排序？** 翻完负数后，原本最小的（绝对值大的负数）变成了最大的正数，"最小位置"已经变了。必须重排才能找到真正的最小。

## Java 代码

```java
class Solution {
    public int largestSumAfterKNegations(int[] nums, int k) {
        Arrays.sort(nums);  // 从小到大

        // 第一步：从左往右翻负数（最小的负数最优先）
        for (int i = 0; i < nums.length && k > 0 && nums[i] < 0; i++) {
            nums[i] = -nums[i];
            k--;
        }

        // 第二步：k 还有剩余且是奇数 → 翻当前数组里最小的那个
        if (k % 2 == 1) {
            Arrays.sort(nums);  // 重新排序，因为翻转后顺序变了
            nums[0] = -nums[0];
        }

        int sum = 0;
        for (int n : nums) sum += n;
        return sum;
    }
}
```

## 走例子验证

### 示例 1：`nums = [-4, -2, 1, 3], k = 5`

排序后：`[-4, -2, 1, 3]`

**第一步翻负数**：
- 翻 -4 → `[4, -2, 1, 3]`, k=4
- 翻 -2 → `[4, 2, 1, 3]`, k=3
- 遇到 1，正数，停止。

**第二步处理剩余 k=3**（奇数）：
- 重排 → `[1, 2, 3, 4]`
- 翻最小的 1 → `[-1, 2, 3, 4]`

总和 = -1 + 2 + 3 + 4 = **8** ✓

幕后 k=5 的分配：
| 操作 | 实际做了什么 | 是否实质翻转 |
|------|-------------|------------|
| 1 | 翻 -4 → 4 | 是 |
| 2 | 翻 -2 → 2 | 是 |
| 3 | 翻 1 → -1 | 是 |
| 4 | 翻 -1 → 1 | 否（撤销用） |
| 5 | 翻 1 → -1 | 否（撤销用） |

3 次实质翻转 + 2 次成对撤销 = 5 次，刚好用完 `k`。

### 示例 2：`nums = [-5, 1, 2], k = 2`（验证为什么要重排）

排序后：`[-5, 1, 2]`

**第一步**：翻 -5 → `[5, 1, 2]`, k=1，遇到 1 停止。

**第二步**：k=1 奇数。
- ❌ 不重排直接翻 `nums[0]`：翻 5 → `[-5, 1, 2]`，总和 = -2
- ✅ 重排后 `[1, 2, 5]`，翻 1 → `[-1, 2, 5]`，总和 = **6**

### 示例 3：`nums = [-3, 0, 2], k = 4`（验证 0 的特殊作用）

排序后：`[-3, 0, 2]`

**第一步**：翻 -3 → `[3, 0, 2]`, k=3，遇到 0 停止。

**第二步**：k=3 奇数，重排 `[0, 2, 3]`，翻最小的 0 → `[0, 2, 3]`（值不变！）。

总和 = **5**

> **0 是"免费消耗位"**：当数组里有 0，剩余 k 是奇数时，翻 0 没有任何损失。

## 常见疑问

### Q1：为什么要分两块代码？不能合并吗？

不能。两块代码做的事情本质不同：

| | 第一块 | 第二块 |
|---|--------|--------|
| 翻什么 | 只翻负数 | 翻当前最小的（任意类型）|
| 翻几次 | 每个最多一次 | 最多 1 次 |
| 触发条件 | 总是执行 | 仅当剩余 k 为奇数 |
| 排序需求 | 一次初始排序 | 需要**重新**排序 |

**第一块管"赚收益"，第二块管"止损"**。

### Q2：k 的多次操作可以撤销，会不会被忽略？

不会。最优解里不会包含"翻同一个数两次"这种浪费操作。但因为题目要求"恰好 k 次"，多余的 k 必须用掉，于是**成对抵消**变成处理"k 用不完"的工具：
- 偶数次 → 全部抵消
- 奇数次 → 抵消掉偶数对后，剩 1 次实质翻转

### Q3：k 每次的作用都不一样，怎么处理？

把 k 次操作看成**集合**而不是**序列**。由于操作可交换、可成对抵消，只需要决定"哪些位置被翻奇数次"，其余位置在某个数上反复横跳消耗即可。

### Q4：贪心体现在哪？

每一步都做**当前最优**的选择——翻数组里最小的那个数：
- 不规划未来（不考虑翻完会怎样）
- 不回头（翻了就不撤销）
- 局部最优 → 全局最优

代码里的"两块 + 奇偶判断"只是对"做 k 次相同的贪心选择"做了**数学优化**，骨架仍是朴素贪心。

## 复杂度分析

- **时间复杂度**：`O(n log n)`，主要是排序
- **空间复杂度**：`O(1)`（不考虑排序的栈空间）

## 总结

1. **从小到大排序**，让最小的元素在最左边
2. **从左往右翻负数**，每翻一个 `k--`，遇到非负数停止
3. **若剩余 k 为奇数**，重新排序后翻 `nums[0]`（当前最小的）
4. **若剩余 k 为偶数**，无需操作（成对抵消）
5. **求和返回**

**核心一句话**：每一步都翻数组里最小的那个数。
