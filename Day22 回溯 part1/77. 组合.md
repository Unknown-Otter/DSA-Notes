# 77. 组合 Combinations

> [LeetCode 77. Combinations](https://leetcode.com/problems/combinations/)

给定两个整数 `n` 和 `k`，返回 `1 ... n` 中所有可能的 `k` 个数的组合。

## 暴力思路：k 层 for 循环

最直接的想法：`k=2` 就两层循环，`k=3` 就三层循环……

```java
// k=2 时的暴力
for (int i = 1; i <= n; i++) {
    for (int j = i + 1; j <= n; j++) {
        result.add(Arrays.asList(i, j));
    }
}
```

**问题**：k 是动态的，没法写死循环层数。k=50 就要 50 层 for 循环，写不出来。

**解决**：用递归模拟 k 层嵌套 for 循环 → 回溯法。

## 回溯法核心思路

把组合问题抽象为**树形结构**：

- **树的宽度** = n（for 循环横向遍历）
- **树的深度** = k（递归纵向遍历）
- **叶子节点** = `path.size() == k`，收集结果

每次从集合中选取元素，可选择的范围随着选择的进行而**收缩**，靠 `startIndex` 控制。

## 回溯三部曲

### 1. 递归函数参数

```java
List<List<Integer>> result = new ArrayList<>();  // 结果集合
LinkedList<Integer> path = new LinkedList<>();     // 单一结果路径

void backtracking(int n, int k, int startIndex)
```

**`startIndex` 的作用**：保证组合不重复。选了 1 之后，下一层只能从 2 开始选，不能回头 → 和暴力解法中 `j = i + 1` 是同一个思路。

### 2. 终止条件

```java
if (path.size() == k) {
    result.add(new ArrayList<>(path));  // 注意要 new 一份拷贝
    return;
}
```

`path` 收集了 k 个数 → 到达叶子节点 → 收集结果并返回。

### 3. 单层搜索逻辑

```java
for (int i = startIndex; i <= n; i++) {
    path.add(i);                  // 处理：选这个数
    backtracking(n, k, i + 1);   // 递归：进入下一层，从 i+1 开始
    path.removeLast();            // 回溯：撤销选择
}
```

- **`path.add(i)`**：相当于暴力循环里进入更深一层
- **`backtracking(n, k, i + 1)`**：递归开下一层 for 循环
- **`path.removeLast()`**：回溯，撤销本次选择，让 for 循环尝试下一个 i

## 完整代码（未剪枝）

```java
class Solution {
    List<List<Integer>> result = new ArrayList<>();
    LinkedList<Integer> path = new LinkedList<>();

    public List<List<Integer>> combine(int n, int k) {
        backtracking(n, k, 1);
        return result;
    }

    public void backtracking(int n, int k, int startIndex) {
        if (path.size() == k) {
            result.add(new ArrayList<>(path));
            return;
        }
        for (int i = startIndex; i <= n; i++) {
            path.add(i);
            backtracking(n, k, i + 1);
            path.removeLast();
        }
    }
}
```

## 剪枝优化

**核心观察**：如果 for 循环起始位置之后的元素个数，已经不够凑齐 k 个了，就没必要继续搜索。

**推导**：

- 已经选择的元素个数：`path.size()`
- 还需要的元素个数：`k - path.size()`
- 集合中至多要从 `n - (k - path.size()) + 1` 开始遍历

**举例**：n=4, k=3, path.size()=0 → 至多从 `4 - (3 - 0) + 1 = 2` 开始 → [2,3,4] 是最后一个合法组合。从 3 开始就凑不齐 3 个数了。

**优化后的 for 循环**：

```java
for (int i = startIndex; i <= n - (k - path.size()) + 1; i++) {
    path.add(i);
    backtracking(n, k, i + 1);
    path.removeLast();
}
```

## 完整代码（剪枝优化）

```java
class Solution {
    List<List<Integer>> result = new ArrayList<>();
    LinkedList<Integer> path = new LinkedList<>();

    public List<List<Integer>> combine(int n, int k) {
        backtracking(n, k, 1);
        return result;
    }

    private void backtracking(int n, int k, int startIndex) {
        if (path.size() == k) {
            result.add(new ArrayList<>(path));
            return;
        }
        for (int i = startIndex; i <= n - (k - path.size()) + 1; i++) {
            path.add(i);
            backtracking(n, k, i + 1);
            path.removeLast();
        }
    }
}
```

## 踩坑记录

1. **`result.add(new ArrayList<>(path))`**：必须拷贝一份！直接 `result.add(path)` 加的是引用，后续 `removeLast` 会把已加入的结果也改掉
2. **`startIndex` 从 `i + 1` 开始**，不是 `startIndex + 1`。每次递归是从当前选的数的下一个开始，不是从本层起始位置的下一个
3. **剪枝公式 `n - (k - path.size()) + 1` 的 +1**：因为包含起始位置本身（左闭区间）

## 复杂度分析

- **时间复杂度**：O(C(n,k) × k) — 共 C(n,k) 个组合，每个组合拷贝需要 O(k)
- **空间复杂度**：O(k) — 递归深度为 k

## 总结

1. 组合问题本质是 **k 层 for 循环嵌套**，用**递归**解决动态层数问题
2. 回溯法搜索过程可以抽象为**树形结构**：for 循环横向遍历（树的宽度），递归纵向遍历（树的深度）
3. **回溯模板**：处理节点 → 递归 → 撤销处理（三步走）
4. **剪枝**：当剩余元素不够凑齐 k 个时，提前终止 for 循环
