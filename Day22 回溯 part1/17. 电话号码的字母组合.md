# 17. 电话号码的字母组合

> https://leetcode.cn/problems/letter-combinations-of-a-phone-number/

## 题目核心

给定一个只包含数字 `2-9` 的字符串 `digits`，返回**所有它能表示的字母组合**。数字到字母的映射与电话按键相同（2→abc，3→def，...，9→wxyz）。

示例：`digits = "23"` → `["ad","ae","af","bd","be","bf","cd","ce","cf"]`

这是**回溯算法（backtracking）**的经典入门题。

---

## 一、解题思路

### 1. 把"数字 → 字母"预先打成表

用一个 `String[]` 数组，**下标即数字**：

```java
String[] numString = {"", "", "abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"};
```

- `numString[0]` 和 `numString[1]` 留空，因为键盘上 0 和 1 不对应字母
- `numString[2] = "abc"`，`numString[3] = "def"`，以此类推

**关键的字符到数字转换**：`digits.charAt(num) - '0'`

因为 `'2'` 的 ASCII 是 50，`'0'` 是 48，两者相减得到整数 2。这样 `numString[digits.charAt(num) - '0']` 就能查到字母表。

### 2. 问题的本质是一棵决策树

以 `digits = "23"` 为例：
- 第 0 位：从 `"abc"` 中选一个字母
- 第 1 位：从 `"def"` 中选一个字母

把所有选择画成树：

```
                    ""
           /         |         \
          a          b          c
        / | \      / | \      / | \
       d  e  f    d  e  f    d  e  f
      ad ae af   bd be bf   cd ce cf
```

从根到叶子的每条路径，就是一个完整的字母组合。**回溯算法的任务：遍历所有根到叶子的路径**。

## 二、完整代码

```java
class Solution {
    List<String> list = new ArrayList<>();
    StringBuilder temp = new StringBuilder();

    public List<String> letterCombinations(String digits) {
        if (digits == null || digits.length() == 0) return list;

        String[] numString = {"", "", "abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"};
        backTracking(digits, numString, 0);
        return list;
    }

    private void backTracking(String digits, String[] numString, int num) {
        if (num == digits.length()) {
            list.add(temp.toString());
            return;
        }

        String str = numString[digits.charAt(num) - '0'];
        for (int i = 0; i < str.length(); i++) {
            temp.append(str.charAt(i));
            backTracking(digits, numString, num + 1);
            temp.deleteCharAt(temp.length() - 1);
        }
    }
}
```

## 三、变量含义详解

### 成员变量

| 变量 | 类型 | 作用 | 生命周期 |
|------|------|------|----------|
| `list` | `List<String>` | **结果集**，存放所有完整的字母组合 | 整个递归过程共享 |
| `temp` | `StringBuilder` | **当前路径**，记录正在尝试的字母组合 | 整个递归过程共享，动态变化 |

**`temp` 是一个共用的"草稿纸"，`list` 是最终的"答案簿"。**

### 方法参数

| 参数 | 含义 | 取值范围（以 "23" 为例） |
|------|------|-------------------------|
| `digits` | 用户输入的数字串（**不变**） | `"23"` |
| `numString` | 数字到字母的映射表（**不变**） | 固定的 10 元素数组 |
| `num` | **当前处理到 digits 的第几位**（指针） | 0 → 1 → 2 |

### for 循环里的局部变量

| 变量 | 含义 | 取值范围 |
|------|------|----------|
| `str` | 当前数字对应的字母表 | 如 `num=0` 时 `str="abc"` |
| `i` | **当前字母表 str 的索引** | 0 到 `str.length()-1` |

### 关键：`num` vs `i` 的区别

这是回溯最容易混淆的点，必须牢牢记住：

- **`num` 代表 digits 的位置**：每递归一层 `+1`，往 digits 的下一位走
- **`i` 代表当前字母表的位置**：只在当前 for 循环里有意义，**不往下传递**

**记忆口诀：`num` 跨层走，`i` 层内走，两个不能混。**

## 四、回溯模板（三步曲）

这道题用的是回溯最基础的模板：

```java
void backtrack(路径, 选择列表) {
    if (满足结束条件) {
        结果.add(路径);
        return;
    }
    for (选择 : 选择列表) {
        做选择;              // temp.append()
        backtrack(...);      // 递归进入下一层
        撤销选择;            // temp.deleteCharAt()
    }
}
```

### 三步曲的具体含义

1. **做选择**：`temp.append(str.charAt(i))` —— 把当前选的字母加到路径末尾
2. **递归**：`backTracking(digits, numString, num + 1)` —— 带着路径走向下一层
3. **撤销选择**：`temp.deleteCharAt(temp.length() - 1)` —— 把刚加的字母删掉

### 为什么必须撤销？

因为 `temp` 是**整个递归共用**的一个 StringBuilder。如果不撤销，下一轮 for 循环（比如从 `a` 换到 `b`）时，temp 里就会残留前一次的选择，导致结果错乱。

**一句话：来时添加，归时撤回。**

## 五、终止条件详解

```java
if (num == digits.length()) {
    list.add(temp.toString());
    return;
}
```

### 这三行代码分别在做什么？

| 行 | 动作 | 作用对象 |
|----|------|---------|
| `if (num == digits.length())` | 判断：选够了吗？ | 无 |
| `list.add(temp.toString())` | 把 temp 的**快照**存到 list | list |
| `return` | 返回上一层 | 调用栈 |

### 重要：这里**不往 temp 里加字母**

`temp` 里的内容是 for 循环里 `append` 上去的。当 `num == digits.length()` 触发时，`temp` 早就已经拼好了（比如 `"ad"`）。这个 if 块只做一件事：**把 temp 当前的样子拍张照，放进 list 里**。

### 为什么用 `temp.toString()` 而不是 `list.add(temp)`？

```java
list.add(temp.toString());  // ✓ 正确
list.add(temp);             // ✗ 错误
```

`temp.toString()` 生成一个**新的 String 对象**，和 `temp` 解耦。直接 `list.add(temp)` 会让 list 存的是 temp 的**引用**，后续回溯修改 temp，list 里的内容也会跟着变，最终 list 里全是空字符串或同一个字符串。

## 六、运行过程追踪（`digits = "23"`）

完整执行顺序：

```
进入 backTracking(num=0)，temp="", list=[]
  num(0) != length(2)，通过
  str = numString[2] = "abc"

  i=0: temp.append('a') → temp="a"
    进入 backTracking(num=1)
      num(1) != length(2)，通过
      str = numString[3] = "def"

      i=0: temp.append('d') → temp="ad"
        进入 backTracking(num=2)
          num(2) == length(2) ✓ 触发终止
          list.add("ad") → list=["ad"]
          return
        temp.deleteCharAt() → temp="a"

      i=1: temp.append('e') → temp="ae"
        进入 backTracking(num=2)
          list.add("ae") → list=["ad","ae"]
          return
        temp.deleteCharAt() → temp="a"

      i=2: temp.append('f') → temp="af"
        进入 backTracking(num=2)
          list.add("af") → list=["ad","ae","af"]
          return
        temp.deleteCharAt() → temp="a"

      for 循环结束，num=1 栈帧弹出
    temp.deleteCharAt() → temp=""  (删掉 'a')

  i=1: temp.append('b') → temp="b"
    进入 backTracking(num=1)
      i=0,1,2 依次拼出 "bd","be","bf" 加入 list
    temp.deleteCharAt() → temp=""

  i=2: temp.append('c') → temp="c"
    进入 backTracking(num=1)
      i=0,1,2 依次拼出 "cd","ce","cf" 加入 list
    temp.deleteCharAt() → temp=""

  for 循环结束，num=0 栈帧弹出

最终 list = ["ad","ae","af","bd","be","bf","cd","ce","cf"]
```

### temp 和 list 的变化表

| 步骤 | 操作 | temp | list |
|------|------|------|------|
| 1 | 进入 num=0 | `""` | `[]` |
| 2 | append 'a' | `"a"` | `[]` |
| 3 | append 'd' | `"ad"` | `[]` |
| 4 | **触发终止，list.add** | `"ad"` | `["ad"]` |
| 5 | return，delete 'd' | `"a"` | `["ad"]` |
| 6 | append 'e' → 终止 → list.add | `"ae"` | `["ad","ae"]` |
| 7 | return，delete 'e' | `"a"` | `["ad","ae"]` |
| 8 | append 'f' → 终止 → list.add | `"af"` | `["ad","ae","af"]` |
| 9 | return，delete 'f'、'a' | `""` | `["ad","ae","af"]` |
| 10 | append 'b'，继续递归... | `"b"` | ... |
| ... | ... | ... | ... |
| 最后 | 所有递归结束 | `""` | 9 个结果 |

### 关键观察

- **终止触发时 `temp` 不变**：list 只是复制了一份，temp 本身没有修改
- **删除发生在"返回后"**：不是"退出时"删除，而是**回到上一层的 for 循环里**，作为迭代的一部分执行删除
- **删除的目的是腾位置**：为了让下一轮 for 循环（尝试兄弟节点）能在干净的 temp 上工作

## 七、写代码的坑（踩坑笔记）

### 坑 1：把 `num + 1` 写成 `i + 1` ⚠️ 高频错误

```java
// ❌ 错误
backTracking(digits, numString, i + 1);

// ✅ 正确
backTracking(digits, numString, num + 1);
```

**后果**：`StackOverflowError`（栈溢出）

**原因分析**：
- 第一层递归时，`i=0`，`i+1=1`，恰好等于 `num+1=1`，巧合性地"看起来对"
- 第二层递归时，`i` 从 0 重新计数，`i+1` 又变成 1，但这时应该传 2
- 导致 `num` 永远停在 1，终止条件 `num == digits.length()` 永远不满足，无限递归

**判断方法**：问自己"这个参数代表的是**什么东西的**索引？"
- `num` = digits 的索引 → 跨层传递时用 `num + 1`
- `i` = 字母表 str 的索引 → 只在当前 for 循环内有意义

### 坑 2：用 `list.add(temp)` 而不是 `list.add(temp.toString())`

```java
// ❌ 错误：list 里存的是 temp 的引用
list.add(temp);

// ✅ 正确：生成新的 String 对象
list.add(temp.toString());
```

**后果**：最终 list 里全是空字符串（或最后一次 temp 的状态）

**原因**：StringBuilder 是可变对象。list 存引用的话，后续 `temp.deleteCharAt()` 会把 list 里的内容也改掉。

### 坑 3：忘记处理空输入

```java
// 必须加这一行
if (digits == null || digits.length() == 0) return list;
```

**后果**：如果不判空，`digits = ""` 时会直接命中终止条件 `num == 0 == digits.length()`，把空字符串 `""` 加入 list，结果变成 `[""]` 而不是 `[]`。

### 坑 4：忘记撤销选择（删除末尾字符）

```java
// ❌ 错误：缺少 deleteCharAt
for (int i = 0; i < str.length(); i++) {
    temp.append(str.charAt(i));
    backTracking(digits, numString, num + 1);
    // 忘了这一行！
}

// ✅ 正确
for (int i = 0; i < str.length(); i++) {
    temp.append(str.charAt(i));
    backTracking(digits, numString, num + 1);
    temp.deleteCharAt(temp.length() - 1);
}
```

**后果**：temp 越积越长，结果全错

**原因**：`temp` 是共享的 StringBuilder，必须手动清理

### 坑 5：对 `num` 做 `++` 操作而不是传 `num + 1`

```java
// ❌ 不推荐
num++;
backTracking(digits, numString, num);
// 后面还得 num-- 回来，容易出错

// ✅ 推荐：直接传递新值
backTracking(digits, numString, num + 1);
```

**原因**：`num + 1` 是生成一个新值传递，不改变本层的 `num`。这也是为什么 `num` 不需要像 `temp` 那样手动撤销——**基本类型按值传递**，每层递归都有独立的副本。

### 坑 6：`charAt(num) - '0'` 忘了减字符零

```java
// ❌ 错误：char 当 int 用，得到的是 ASCII 码
String str = numString[digits.charAt(num)];  // 越界！

// ✅ 正确
String str = numString[digits.charAt(num) - '0'];
```

**原因**：`digits.charAt(num)` 返回的是 `char`，值是 ASCII 码（如 '2' = 50）。必须减去 `'0'`（48）才能得到真正的数字 2。

## 八、总结

### 回溯三要素
1. **路径**（`temp`）：已经做出的选择
2. **选择列表**（`str`，由当前 `num` 决定）：当前还可以做的选择
3. **结束条件**（`num == digits.length()`）：到达叶子时收集结果

### 为什么 `num` 不需要撤销，而 `temp` 需要？
- `num` 是 `int`（基本类型），**按值传递**，每层递归有独立副本
- `temp` 是 `StringBuilder`（引用类型），**共享同一个对象**，必须手动清理

### 本题的核心模式
这是一棵**宽度为每层字母数、深度为 digits 长度**的决策树的 DFS 遍历。

### 延伸题目
掌握这个模板后，可以用完全相同的思路解：
- **LC 77 组合** —— 在 `[1..n]` 中选 k 个数
- **LC 78 子集** —— 所有子集
- **LC 46 全排列** —— 所有排列（需要 `used[]` 数组标记已选）
- **LC 39 组合总和** —— 在数组中选若干数使和为 target
- **LC 51 N 皇后** —— 在棋盘上放 N 个皇后

只要替换"选择列表"和"结束条件"即可，三步曲的骨架不变。

## 附：最终完整代码

```java
class Solution {
    List<String> list = new ArrayList<>();
    StringBuilder temp = new StringBuilder();

    public List<String> letterCombinations(String digits) {
        if (digits == null || digits.length() == 0) return list;

        String[] numString = {"", "", "abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"};
        backTracking(digits, numString, 0);
        return list;
    }

    private void backTracking(String digits, String[] numString, int num) {
        if (num == digits.length()) {
            list.add(temp.toString());
            return;
        }

        String str = numString[digits.charAt(num) - '0'];
        for (int i = 0; i < str.length(); i++) {
            temp.append(str.charAt(i));
            backTracking(digits, numString, num + 1);
            temp.deleteCharAt(temp.length() - 1);
        }
    }
}
```

**时间复杂度**：O(3^m × 4^n)，m 是对应 3 个字母的数字个数（2,3,4,5,6,8），n 是对应 4 个字母的数字个数（7,9）
**空间复杂度**：O(m + n)，递归调用栈的深度
