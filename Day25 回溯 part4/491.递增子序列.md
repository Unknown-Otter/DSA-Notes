# LC 491. Non-Decreasing Subsequences

> https://leetcode.com/problems/non-decreasing-subsequences/

给定一个整型数组，找到所有该数组的递增子序列，递增子序列的长度至少是 2。

**示例：**
```
输入: [4, 6, 7, 7]
输出: [[4, 6], [4, 7], [4, 6, 7], [4, 6, 7, 7], [6, 7], [6, 7, 7], [7, 7], [4, 7, 7]]
```

---

## 核心思路

本题类似"取有序子集"，同样涉及子集 + 去重，容易与 **LC 90 子集 II** 混淆。

**关键区别：**

- LC 90 可以先 `Arrays.sort()` + `used[]` 标记数组去重
- **本题不能排序**，排序后的数组所有子序列都是递增的，破坏原始顺序语义
- 必须改用**同层 `HashSet` 去重**

> 树形结构中，同一父节点下的同层上使用过的元素不能再使用。每进入新一层递归，`HashSet` 重新定义（清空），只负责**本层去重**，不影响纵向路径。

## 我写错的代码（Debug 记录）

```java
class Solution {

    List<List<Integer>> result = new ArrayList<>();
    LinkedList<Integer> path = new LinkedList<>();
    boolean[] used;

    public List<List<Integer>> findSubsequences(int[] nums) {
        Arrays.sort(nums);  // ❌ 错误：不能排序，破坏原始顺序

        used = new boolean[nums.length];
        helper(nums, 0);
        return result;
    }

    public void helper(int[] nums, int startIndex) {
        if (path.length() >= 2)  // ❌ 错误：LinkedList 没有 .length()，应用 .size()
            result.add(new ArrayList<>(path));

        if (startIndex >= nums.length) {
            return;
        }

        for (int i = startIndex; i < nums.length; i++) {
            if (i > 0 && nums[i] == nums[i - 1] && !used[i - 1]) {  // ❌ 错误：这是排序后才适用的去重逻辑
                continue;
            }

            path.add(nums[i]);
            used[i] = true;
            helper(nums, i + 1);
            path.removeLast();
            used[i] = false;
        }
    }
}
```

**三处错误总结：**

| # | 错误 | 原因 | 修复 |
|---|------|------|------|
| 1 | `Arrays.sort(nums)` | 本题要求保留原始顺序的子序列，排序后逻辑完全错误 | 删除排序 |
| 2 | `path.length()` | `LinkedList` 没有 `.length()` 方法，编译报错 | 改为 `path.size()` |
| 3 | `!used[i-1]` 去重逻辑 | 这套逻辑依赖排序后相邻重复元素，本题不适用 | 改为同层 `HashSet` 去重 |

## 修复后的代码

```java
class Solution {

    List<List<Integer>> result = new ArrayList<>();
    LinkedList<Integer> path = new LinkedList<>();

    public List<List<Integer>> findSubsequences(int[] nums) {
        helper(nums, 0);
        return result;
    }

    public void helper(int[] nums, int startIndex) {
        if (path.size() >= 2) {
            result.add(new ArrayList<>(path));
        }

        if (startIndex >= nums.length) {
            return;
        }

        HashSet<Integer> set = new HashSet<>(); // 同层去重，每层新建

        for (int i = startIndex; i < nums.length; i++) {

            // 同层已用过该值，跳过
            if (set.contains(nums[i])) {
                continue;
            }

            // 保证非递减
            if (!path.isEmpty() && nums[i] < path.getLast()) {
                continue;
            }

            set.add(nums[i]);
            path.add(nums[i]);
            helper(nums, i + 1);
            path.removeLast();
        }
    }
}
```

> **注意：** `set.add(nums[i])` 之后**没有对应的 remove 操作**。这是正确的——`set` 只负责本层，下一层递归会重新 `new HashSet<>()`，所以不需要回溯 `set`。

## 教程中的三种 Java 解法

### 解法一：HashSet 去重（标准写法）

```java
class Solution {
    List<List<Integer>> result = new ArrayList<>();
    List<Integer> path = new ArrayList<>();

    public List<List<Integer>> findSubsequences(int[] nums) {
        backTracking(nums, 0);
        return result;
    }

    private void backTracking(int[] nums, int startIndex) {
        if (path.size() >= 2)
            result.add(new ArrayList<>(path));

        HashSet<Integer> hs = new HashSet<>();
        for (int i = startIndex; i < nums.length; i++) {
            if (!path.isEmpty() && path.get(path.size() - 1) > nums[i] || hs.contains(nums[i]))
                continue;
            hs.add(nums[i]);
            path.add(nums[i]);
            backTracking(nums, i + 1);
            path.remove(path.size() - 1);
        }
    }
}
```

---

### 解法二：数组哈希去重（优化版，推荐）

题目说明数值范围为 `[-100, 100]`，可以用长度 201 的数组代替 `HashSet`，避免频繁的哈希映射开销，性能更好。

```java
class Solution {
    private List<Integer> path = new ArrayList<>();
    private List<List<Integer>> res = new ArrayList<>();

    public List<List<Integer>> findSubsequences(int[] nums) {
        backtracking(nums, 0);
        return res;
    }

    private void backtracking(int[] nums, int start) {
        if (path.size() > 1) {
            res.add(new ArrayList<>(path));
        }

        int[] used = new int[201]; // 数值范围 [-100, 100]，偏移 +100
        for (int i = start; i < nums.length; i++) {
            if (!path.isEmpty() && nums[i] < path.get(path.size() - 1) ||
                    (used[nums[i] + 100] == 1)) continue;
            used[nums[i] + 100] = 1;
            path.add(nums[i]);
            backtracking(nums, i + 1);
            path.remove(path.size() - 1);
        }
    }
}
```

> **为什么 `nums[i] + 100`？** 数值范围 `[-100, 100]`，直接用作数组下标会有负数。偏移 `+100` 后范围变为 `[0, 200]`，可以安全用作下标。

### 解法三：HashMap 去重

```java
class Solution {
    List<List<Integer>> res = new ArrayList<>();
    LinkedList<Integer> path = new LinkedList<>();

    public List<List<Integer>> findSubsequences(int[] nums) {
        getSubsequences(nums, 0);
        return res;
    }

    private void getSubsequences(int[] nums, int start) {
        if (path.size() > 1) {
            res.add(new ArrayList<>(path));
            // 注意这里不要加 return，要取树上的所有节点
        }
        HashMap<Integer, Integer> map = new HashMap<>();
        for (int i = start; i < nums.length; i++) {
            if (!path.isEmpty() && nums[i] < path.getLast()) {
                continue;
            }
            if (map.getOrDefault(nums[i], 0) >= 1) {
                continue;
            }
            map.put(nums[i], map.getOrDefault(nums[i], 0) + 1);
            path.add(nums[i]);
            getSubsequences(nums, i + 1);
            path.removeLast();
        }
    }
}
```

## 三种去重方式对比

| 方式 | 数据结构 | 适用场景 | 性能 |
|------|----------|----------|------|
| `HashSet` | `HashSet<Integer>` | 通用 | 中等，有哈希映射开销 |
| 数组哈希 | `int[201]` | 数值范围已知且小 | **最快**，推荐 |
| `HashMap` | `HashMap<Integer, Integer>` | 需要计数场景 | 最慢 |

- **时间复杂度：** O(n · 2ⁿ)
- **空间复杂度：** O(n)

## 关键总结

1. **不能排序** —— 本题要求保留原始顺序，排完序所有子序列都是递增的，去重逻辑也会失效
2. **`path.size() >= 2`** —— 收集结果，但**不加 `return`**，需要继续遍历树上所有节点
3. **同层 `HashSet`/数组局部去重** —— 每层进入时新建，只管本层，不影响纵向路径
4. **`set` 不需要回溯** —— 因为每层都是新建的局部变量，天然隔离
5. **`LinkedList` 用 `.size()`，不是 `.length()`**
