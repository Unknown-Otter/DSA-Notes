# 完全背包理论基础 - 二维DP数组

> 卡码网第 52 题: [https://kamacoder.com/problempage.php?pid=1052](https://kamacoder.com/problempage.php?pid=1052)
>
> 力扣上无原题，所有完全背包问题都是需要从这套理论转化而来的应用题。

## 题目描述

有 N 件物品和一个最多能背重量为 W 的背包。第 i 件物品的重量是 `weight[i]`，价值是 `value[i]`，**每件物品都有无限个**。求将哪些物品装入背包里物品价值总和最大。

### 本文使用的例子

背包最大重量 `bagWeight = 4`，物品信息如下：

| 物品 | 重量 (weight) | 价值 (value) |
|------|--------------|--------------|
| 物品 0 | 1 | 15 |
| 物品 1 | 3 | 20 |
| 物品 2 | 4 | 30 |

**每件商品都有无限个**，问背包能装的物品最大价值是多少？

---

## 完全背包 vs 01 背包

**完全背包和 01 背包问题唯一不同的地方就是：每种物品有无限件。**

这看似很小的变化，会影响：
1. **第 0 行的初始化**：物品 0 可以放多次，所以容量够大时要循环累加
2. **递推公式的"放物品 i"那一项**：从 `dp[i-1][...]` 变成 `dp[i][...]`

下面用动规五部曲展开讲解。

---

## 一、确定 dp 数组以及下标的含义

`dp[i][j]` 表示从下标为 `[0, i]` 的物品中（**每个物品可以取无限次**），放进容量为 `j` 的背包，价值总和的最大值。

- **行 i**：表示"只允许从物品 0 到物品 i 中挑"
- **列 j**：表示"当前背包的容量"
- **dp 数组大小**：`n` 行 × `(bagWeight + 1)` 列
  - 行数 = 物品数量
  - 列数 = `bagWeight + 1`，因为 j 的取值需要从 0 到 bagWeight，**共 bagWeight + 1 个值**

最终答案位于 `dp[n-1][bagWeight]`（右下角格子）。

---

## 二、确定递推公式

在 `dp[i][j]` 这一格，我们只问一个问题：**当前这个物品 i，放还是不放？**

### 选项 A：不放物品 i

不放 i，那么背包里的最大价值就是"用前 i-1 个物品装容量 j 的最大价值"：

```java
dp[i-1][j]
```

### 选项 B：放物品 i

**放物品 i 的总价值 = 物品 i 自身的价值 + 剩余空间装的最大价值**

- 物品 i 自身价值：`value[i]`
- 放进物品 i 后剩余空间：`j - weight[i]`
- 剩余空间装的最大价值：从 dp 表查 → `dp[i][j - weight[i]]`

为什么查的是 `dp[i][...]` 而不是 `dp[i-1][...]`？**这是完全背包和 01 背包的关键区别**：

> 完全背包中物品 i 有无限个，放了一个物品 i 之后，剩余空间里**还可以继续选物品 i**。所以剩余空间的子问题，物品 i 依然在候选池里 → 查 `dp[i][...]`。
>
> 而 01 背包中物品 i 只有一个，放完就用掉了，剩余空间里不能再选 i → 查 `dp[i-1][...]`。

### 完整递推公式

```java
dp[i][j] = Math.max(dp[i-1][j], dp[i][j - weight[i]] + value[i]);
```

### 两个公式并排对比

```java
// 01 背包
dp[i][j] = max(dp[i-1][j], dp[i-1][j - weight[i]] + value[i]);
//                          ^^^^^ 物品 i 用完了
// 完全背包
dp[i][j] = max(dp[i-1][j], dp[i  ][j - weight[i]] + value[i]);
//                          ^^^   物品 i 还能继续选
```

**记忆口诀：01 看上一行（i-1），完全看本行（i）**。

---

## 三、dp 数组如何初始化

### 第一列 `dp[i][0]` 全为 0

背包容量为 0，啥都装不下，价值就是 0。代码里如果创建数组时已经全 0，这步可省略。

### 第一行 `dp[0][j]`：只用物品 0

这是完全背包和 01 背包**第一个不同点**。

- 当 `j < weight[0]` 时，装不下物品 0 → `dp[0][j] = 0`
- 当 `j >= weight[0]` 时，因为物品 0 有无限个，**能塞几个就塞几个**：

```java
for (int j = weight[0]; j <= bagWeight; j++) {
    dp[0][j] = dp[0][j - weight[0]] + value[0];
}
```

这里的关键是**右边引用的是 `dp[0][...]`（同一行）**，意思是"在已经装了若干物品 0 的基础上，再塞一个物品 0"。

**对比 01 背包**：01 背包里物品 0 只能放 1 个，所以是 `dp[0][j] = value[0]`，直接赋值。

### 逐个推导第 0 行（weight[0]=1, value[0]=15）

| j | 计算 | 结果 |
|---|------|------|
| 0 | 装不下 | **0** |
| 1 | `dp[0][0] + 15 = 0 + 15` | **15** |
| 2 | `dp[0][1] + 15 = 15 + 15` | **30** |
| 3 | `dp[0][2] + 15 = 30 + 15` | **45** |
| 4 | `dp[0][3] + 15 = 45 + 15` | **60** |

---

## 四、确定遍历顺序

二维 DP 数组下，**先遍历物品还是先遍历背包都可以**，因为递推公式所需的值在二维 dp 数组里都能找到。

通常先遍历物品更直观：

```java
for (int i = 1; i < n; i++) {            // 遍历物品（i=0 已初始化）
    for (int j = 0; j <= bagWeight; j++) { // 遍历背包容量
        if (j < weight[i]) {
            dp[i][j] = dp[i - 1][j];
        } else {
            dp[i][j] = Math.max(
                dp[i - 1][j],
                dp[i][j - weight[i]] + value[i]
            );
        }
    }
}
```

## 五、举例推导 dp 数组

用例子完整推导一遍。物品信息：物品 0(1,15)，物品 1(3,20)，物品 2(4,30)，bagWeight=4。

### 推导第 1 行（物品 1：w=3, v=20）

| j | 判断 | 计算 | 结果 |
|---|------|------|------|
| 0 | j<3，继承上一行 | `dp[0][0]` | **0** |
| 1 | j<3，继承上一行 | `dp[0][1]` | **15** |
| 2 | j<3，继承上一行 | `dp[0][2]` | **30** |
| 3 | j≥3 | `max(dp[0][3], dp[1][0]+20) = max(45, 20)` | **45** |
| 4 | j≥3 | `max(dp[0][4], dp[1][1]+20) = max(60, 35)` | **60** |

### 推导第 2 行（物品 2：w=4, v=30）

| j | 判断 | 计算 | 结果 |
|---|------|------|------|
| 0 | j<4，继承上一行 | `dp[1][0]` | **0** |
| 1 | j<4，继承上一行 | `dp[1][1]` | **15** |
| 2 | j<4，继承上一行 | `dp[1][2]` | **30** |
| 3 | j<4，继承上一行 | `dp[1][3]` | **45** |
| 4 | j≥4 | `max(dp[1][4], dp[2][0]+30) = max(60, 30)` | **60** |

### 最终 dp 表

|         | j=0 | j=1 | j=2 | j=3 | j=4 |
|---------|-----|-----|-----|-----|-----|
| 物品 0   | 0   | 15  | 30  | 45  | **60** |
| 物品 0~1 | 0   | 15  | 30  | 45  | **60** |
| 物品 0~2 | 0   | 15  | 30  | 45  | **60** |

`dp[2][4] = 60` 即为答案 —— 4 个物品 0 全部装入（4 × 15 = 60）。物品 0 性价比最高（每单位 15），完全背包中可以无限放，所以最优解就是堆满物品 0。

## 六、关键问题答疑

### Q1：dp 表为什么有 5 列（j=0 到 j=4）？

因为 j 表示背包容量，取值范围是 0 到 bagWeight（包含两端），共 `bagWeight + 1` 个值。代码里 `new int[n][bagWeight + 1]` 就是这个原因。`j=0` 这一列虽然结果都是 0，但**在递推时会被用到**（例如 `dp[i][j - weight[i]]` 当 `j == weight[i]` 时就查到了 `dp[i][0]`），不能省略。

### Q2：递推公式只 +1 个 `value[i]`，那"放多个物品 i"在哪里？

**通过子问题的递归展开自动覆盖**。

举例：推导 `dp[1][6]` 时，"放物品 1" 项是 `dp[1][3] + 20`。而 `dp[1][3]` 自己又是通过 `max(dp[0][3], dp[1][0]+20)` 得来的，**如果选了后者，就相当于在 `dp[1][6]` 里放了 2 个物品 1**。

**核心思想**：DP 不需要显式枚举"放几个"，每一格只做"放/不放"的二选一，子问题自己会继续做同样的决策，所有组合通过递归展开被覆盖。这就是 DP = Recursion + Memoization 的本质。

### Q3：`dp[i-1][j-w[i]]` 和 `dp[i][j-w[i]]` 的本质区别

两者代表的"剩余空间里能选什么物品"不同：

- `dp[i-1][...]`：物品 0 到 i-1 都能选，**不包括 i**（i 用完了）
- `dp[i][...]`：物品 0 到 i 都能选，**包括 i**（i 还有无限个）

用杂货店比喻：
- 01 背包：货架上苹果只有 1 个。拿走后货架上没苹果了，剩余空间只能选其他物品 → `dp[i-1][...]`
- 完全背包：货架上苹果无限。拿走 1 个后货架上还有，剩余空间还能继续选苹果 → `dp[i][...]`

### Q4：放物品 i 的表达式整体含义

`dp[i][j-w[i]] + v[i]` 整体表示"**假如我决定放物品 i 进背包，那总价值是多少？**"

拆解：
- `v[i]`：当前这一步把物品 i 放进去带来的价值（直接贡献）
- `j - w[i]`：放进去后剩余的空间
- `dp[i][j - w[i]]`：剩余空间装的最大价值（查表得到，子问题最优解）
- 三者关系：**总价值 = 物品 i 的价值 + 剩余空间装的最大价值**

## 七、完整 Java 代码

```java
import java.util.Scanner;

public class CompleteKnapsack {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        int n = scanner.nextInt();
        int bagWeight = scanner.nextInt();

        int[] weight = new int[n];
        int[] value = new int[n];

        for (int i = 0; i < n; i++) {
            weight[i] = scanner.nextInt();
            value[i] = scanner.nextInt();
        }

        int[][] dp = new int[n][bagWeight + 1];

        // 初始化第 0 行：物品 0 可以放多次
        for (int j = weight[0]; j <= bagWeight; j++) {
            dp[0][j] = dp[0][j - weight[0]] + value[0];
        }

        // 动态规划
        for (int i = 1; i < n; i++) {
            for (int j = 0; j <= bagWeight; j++) {
                if (j < weight[i]) {
                    dp[i][j] = dp[i - 1][j];
                } else {
                    dp[i][j] = Math.max(
                        dp[i - 1][j],
                        dp[i][j - weight[i]] + value[i]
                    );
                }
            }
        }

        System.out.println(dp[n - 1][bagWeight]);
        scanner.close();
    }
}
```

## 八、Python 代码

```python
def knapsack(n, bag_weight, weight, value):
    dp = [[0] * (bag_weight + 1) for _ in range(n)]

    # 初始化第 0 行：物品 0 可以放多次
    for j in range(weight[0], bag_weight + 1):
        dp[0][j] = dp[0][j - weight[0]] + value[0]

    # 动态规划
    for i in range(1, n):
        for j in range(bag_weight + 1):
            if j < weight[i]:
                dp[i][j] = dp[i - 1][j]
            else:
                dp[i][j] = max(
                    dp[i - 1][j],
                    dp[i][j - weight[i]] + value[i]
                )

    return dp[n - 1][bag_weight]


if __name__ == "__main__":
    n, bag_weight = map(int, input().split())
    weight = []
    value = []
    for _ in range(n):
        w, v = map(int, input().split())
        weight.append(w)
        value.append(v)
    print(knapsack(n, bag_weight, weight, value))
```

## 九、总结

1. **dp 定义**：`dp[i][j]` 表示从物品 0~i 中（每个无限取）放入容量 j 的背包的最大价值。
2. **递推公式**：`dp[i][j] = max(dp[i-1][j], dp[i][j - w[i]] + v[i])`，**注意"放"的那项查同一行 i，不是 i-1**。
3. **初始化**：第 0 列全 0；第 0 行循环累加 `dp[0][j] = dp[0][j - w[0]] + v[0]`，因为物品 0 可以放多次。
4. **遍历顺序**：二维写法下，先物品后背包、先背包后物品都可以。
5. **核心区别（vs 01 背包）**：只有"放物品 i"那一项的查表行不同 —— 01 用 `i-1`（物品用完），完全用 `i`（物品还在）。
6. **隐式枚举**：DP 不需要写"放几个 i"的循环，子问题递归展开自动覆盖所有组合数量。
