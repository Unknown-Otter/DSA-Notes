# 235. Lowest Common Ancestor of a Binary Search Tree

> [LeetCode 235](https://leetcode.com/problems/lowest-common-ancestor-of-a-binary-search-tree/)

## 题目理解

给定一棵**二叉搜索树（BST）**和两个节点 p、q，找到它们的最低公共祖先（LCA）。BST 的性质：左子树所有值 < root，右子树所有值 > root。

```
        6
       / \
      2    8
     / \  / \
    0   4 7   9
       / \
      3   5
```

## 与 236 题（普通二叉树）的对比

**236 普通二叉树**：不知道 p 和 q 在哪边，只能左右都递归（盲找），所以会走到 null，需要 `root == null` 兜底。

**235 BST**：利用大小关系可以**精准导航**，只往有目标的方向走。因为题目保证 p 和 q 存在于树中，往那个方向走一定有节点，**永远不会碰到 null**，所以不需要判空。

## 思路：利用 BST 性质精准导航

从 root 开始，当前节点（每层递归中的 `root`，不一定是最初的根节点）只有三种情况：

1. **p 和 q 都比当前节点小** → 两个都在左子树，往左走
2. **p 和 q 都比当前节点大** → 两个都在右子树，往右走
3. **其他情况（else）** → p 和 q 分布在两侧，或者当前节点本身就是 p 或 q，当前节点就是 LCA

**为什么 else 能处理"当前节点就是 p 或 q"的情况？** 比如 p 就是当前节点，那 `p.val < root.val` 不成立，`p.val > root.val` 也不成立，自然落到 else，返回 root，正确。

## 递归模拟

```
        6
       / \
      2    8
     / \  / \
    0   4 7   9
       / \
      3   5
```

**例1：p=0, q=4 → LCA=2**

```
root=6：0<6 且 4<6 → 都在左边，往左
root=2：0<2 但 4>2 → 分叉了，else → 返回 2 ✅
```

**例2：p=3, q=5 → LCA=4**

```
root=6：3<6 且 5<6 → 往左
root=2：3>2 且 5>2 → 往右
root=4：3<4 但 5>4 → 分叉了，else → 返回 4 ✅
```

**例3：p=2, q=4 → LCA=2（p 本身是 LCA）**

```
root=6：2<6 且 4<6 → 往左
root=2：2 不< 2，走不进第一个 if；4>2，走不进第二个 if → else → 返回 2 ✅
```

## 代码

**递归法：**

```java
class Solution {
    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        if (root.val > p.val && root.val > q.val) return lowestCommonAncestor(root.left, p, q);
        if (root.val < p.val && root.val < q.val) return lowestCommonAncestor(root.right, p, q);
        return root;
    }
}
```

**迭代法：**

```java
class Solution {
    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        while (true) {
            if (root.val > p.val && root.val > q.val) {
                root = root.left;
            } else if (root.val < p.val && root.val < q.val) {
                root = root.right;
            } else {
                break;
            }
        }
        return root;
    }
}
```

迭代法和递归法逻辑完全一致，只是把递归调用改成了 while 循环中移动 root 指针。因为是**尾递归**（递归调用是函数最后一步操作），可以直接改成迭代。

## 复杂度

- **时间复杂度**：O(h)，h 为树高；平衡 BST 下 O(log n)，退化为链表时 O(n)
- **空间复杂度**：递归法 O(h)（调用栈），迭代法 O(1)

## 踩坑记录

1. **为什么不需要 `root == null`**：BST 通过值的比较精准导航，只往目标存在的方向走，题目保证 p 和 q 存在，所以递归方向上一定有节点，不会碰到 null。而 236 题普通二叉树必须盲找两边，一定会走到 null。
2. **`root` 不是固定的根节点**：每层递归中的 root 是当前正在访问的节点，它会随着递归不断往下移动。else 分支的含义是"从当前节点的视角看，p 和 q 不在同一侧"。
3. **和 236 的本质区别**：236 需要后序遍历（自底向上汇报），因为不知道 p 和 q 在哪边；235 只需要自顶向下单路径搜索，因为 BST 的大小关系告诉你该往哪走。

## 总结

1. BST 的 LCA 本质是**自顶向下沿一条路径搜索**，找到第一个让 p 和 q 分叉的节点
2. 三种情况：**都在左 → 往左；都在右 → 往右；分叉或命中 → 当前节点就是 LCA**
3. 因为精准导航，不需要判空，不需要后序遍历，不需要左右都递归
4. 尾递归结构可以直接改写为迭代，空间优化到 O(1)
