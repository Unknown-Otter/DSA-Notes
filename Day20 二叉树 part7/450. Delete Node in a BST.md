# 450. Delete Node in a BST

> [LeetCode 450](https://leetcode.com/problems/delete-node-in-a-bst/)

## 整体思路

这道题的核心机制是**递归 + return 值被父节点接住**。

函数 `deleteNode(root, key)` 的含义是：处理完删除之后，**这个位置现在应该是谁？** 每一层递归 return 的东西，就是告诉父节点"你的 left 或 right 现在应该接谁"。

关键配合在这两行：

```java
root.left = deleteNode(root.left, key);   // 左边返回什么，左边就变成什么
root.right = deleteNode(root.right, key); // 右边返回什么，右边就变成什么
```

- 没找到要删的节点？`return root`，告诉父节点"我还是我，不用变"
- 找到了要删的节点？return 替代者，告诉父节点"别指向我了，指向它"

## 函数结构总览

函数分为两大部分：

1. **`root.val == key`**：找到了要删的节点，根据左右孩子情况分三种处理
2. **`root.val > key` / `root.val < key`**：还没找到，利用 BST 性质往左或往右递归查找

## 找到目标节点后的三种情况（if - else if - else）

这里用了 **if → else if → else** 链，关键特性是：**走到后面的分支，隐含着前面条件不成立**。

### 情况 1：左孩子为空（`root.left == null`）

不管右孩子是什么，直接 `return root.right`：

- 左空，右有值 → return 右孩子，用右孩子替代自己
- 左空，右也空 → return null，相当于直接删掉这个节点

两种情况下 `return root.right` 都是正确的，所以不需要额外判断右孩子的状态。

### 情况 2：右孩子为空（`else if (root.right == null)`）

能走到这里，说明第一个条件不成立，**左孩子一定不为空**。再加上右孩子为空，所以就是"左不空，右空"，直接 `return root.left`。

### 情况 3：左右孩子都不空（`else`）

能走到这里，前两个条件都不成立，所以左右都不为空。策略是找**右子树的最小节点**（右子树一直往左走到底），把左子树挂到它下面，然后用右孩子替代当前节点。

| left | right | 走哪个分支 |
|------|-------|-----------|
| 空 | 空 | 情况 1（return null） |
| 空 | 不空 | 情况 1（return right） |
| 不空 | 空 | 情况 2（return left） |
| 不空 | 不空 | 情况 3（找右子树最小值） |

## 每种情况的代码运行路径

### 路径 1：root 是 null

```java
if (root == null) return root;  // return null，结束
```

### 路径 2：当前节点不是要删的，key 更小，往左找

```java
if (root == null) return root;                  // 跳过
if (root.val == key)                            // 不成立，跳过
if (root.val > key)                             // 成立！
    root.left = deleteNode(root.left, key);     // 往左递归，接住返回值
if (root.val < key)                             // 不成立，跳过
return root;                                    // 返回当前节点
```

### 路径 3：当前节点不是要删的，key 更大，往右找

```java
if (root == null) return root;                  // 跳过
if (root.val == key)                            // 不成立，跳过
if (root.val > key)                             // 不成立，跳过
if (root.val < key)                             // 成立！
    root.right = deleteNode(root.right, key);   // 往右递归，接住返回值
return root;                                    // 返回当前节点
```

### 路径 4：找到了，左孩子为空

```java
if (root.val == key)        // 成立！
    if (root.left == null)  // 成立！
        return root.right;  // 直接返回右孩子，函数结束，下面代码不执行
```

### 路径 5：找到了，左不空右为空

```java
if (root.val == key)                // 成立！
    if (root.left == null)          // 不成立
    else if (root.right == null)    // 成立！
        return root.left;           // 返回左孩子，函数结束
```

### 路径 6：找到了，左右都不空

```java
if (root.val == key)                // 成立！
    if (root.left == null)          // 不成立
    else if (root.right == null)    // 不成立
    else {
        TreeNode cur = root.right;  // 从右孩子开始
        while (cur.left != null)    // 一直往左走
            cur = cur.left;         // 找到右子树最小值
        cur.left = root.left;       // 把左子树挂到右子树最小节点下面
        root = root.right;          // 用右孩子替代自己
        return root;                // 返回，函数结束
    }
```

**注意**：路径 4、5、6 执行了 return 后函数就结束了，下面的 `root.val > key`、`root.val < key`、`return root` 完全不会执行。只有路径 2 和路径 3（当前节点不是要删的）才会走到最后的 `return root`。

## 具体例子：删除节点 3

```
      5
     / \
    3    8
     \
      4
```

1. `deleteNode(5, 3)`：5 != 3，5 > 3，走路径 2 → `5.left = deleteNode(3, 3)`，**暂停等结果**
2. `deleteNode(3, 3)`：3 == 3，找到了！左孩子为空，走路径 4 → `return root.right` 即 return 4
3. 回到第 1 步：`5.left = 4`，然后 `return 5`

结果：

```
      5
     / \
    4    8
```

## 为什么是 return 而不是 root = root.right？

因为 `root = root.right` 只改了当前函数里的局部变量，父节点的 left/right 指针不知道这件事，树的结构不会变。必须通过 return 把新节点传回去，让父节点用 `root.left = deleteNode(...)` 接住，指针才真正更新。**在递归里，return 就是替代的方式。**

## 完整代码（带注释）

```java
class Solution {
    public TreeNode deleteNode(TreeNode root, int key) {
        // base case：空节点，直接返回
        if (root == null) return root;

        // 找到了要删除的节点
        if (root.val == key) {
            // 情况1：左孩子为空，直接返回右孩子（包含左右都空的情况）
            if (root.left == null) {
                return root.right;
            // 情况2：左孩子不空，右孩子为空，返回左孩子
            } else if (root.right == null) {
                return root.left;
            // 情况3：左右孩子都不空
            } else {
                // 找右子树的最左节点（右子树中最小值）
                TreeNode cur = root.right;
                while (cur.left != null) {
                    cur = cur.left;
                }
                // 把被删节点的左子树挂到右子树最小节点的左边
                cur.left = root.left;
                // 用右孩子替代当前节点
                root = root.right;
                return root;
            }
        }

        // 没找到，利用BST性质继续查找，并用返回值重新连接
        if (root.val > key) root.left = deleteNode(root.left, key);
        if (root.val < key) root.right = deleteNode(root.right, key);
        return root;
    }
}
```

## 踩坑记录

- **else 的归属**：else 是跟内层的 `else if` 配对的，不是跟外层的 `if (root.val == key)` 配对，要看清楚括号结构
- **if-else 链的隐含信息**：走到后面的分支时，前面的条件一定不成立，所以不需要显式写出所有条件
- **return 的本质**：在递归中，return 不是"结束程序"，而是"把值交给调用我的人"，父节点通过 `root.left/right = deleteNode(...)` 接住这个值来完成指针重连
