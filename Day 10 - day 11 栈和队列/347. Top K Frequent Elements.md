# 347. Top K Frequent Elements

> [LeetCode 347. Top K Frequent Elements](https://leetcode.com/problems/top-k-frequent-elements/)

给定整数数组 `nums` 和整数 `k`，返回出现频率前 `k` 高的元素。结果可以按任意顺序返回。

```
Input: nums = [1,1,1,2,2,3], k = 2
Output: [1,2]
```

## 解题思路

题目可以拆成两个子问题:

1. **统计每个数字的出现频率** → 用 `HashMap<Integer, Integer>`
2. **从所有 (num, freq) 中挑出频率最大的 k 个** → 用 `PriorityQueue`（小顶堆）

**关键决策**: 用**小顶堆**而不是大顶堆，堆里只维护 k 个元素。

- 堆顶是当前 top k 里**频率最小**的那个
- 新元素进来时和堆顶比: 大于堆顶就替换堆顶，否则丢弃
- 最终堆里剩下的就是频率前 k 大

**为什么反直觉地用小顶堆?** 因为我们要"淘汰小的，保留大的"。堆顶是最容易被访问和移除的位置，所以把"最可能被淘汰的"（当前 top k 里最小的）放在堆顶最方便。

## Step 1: 统计频率

用 HashMap 遍历一次数组统计频率。

```java
Map<Integer, Integer> freqMap = new HashMap<>();
for (int num : nums) {
    freqMap.put(num, freqMap.getOrDefault(num, 0) + 1);
}
```

## Step 2: 建立小顶堆

用 `PriorityQueue` 存 `Map.Entry`，按 value（频率）升序排列 → 小顶堆。

```java
PriorityQueue<Map.Entry<Integer, Integer>> pq = new PriorityQueue<>(
    (a, b) -> a.getValue() - b.getValue()
);
```

**Comparator 方向口诀**:

- `(a, b) -> a - b` → 升序 → **小顶堆**（堆顶最小）
- `(a, b) -> b - a` → 降序 → **大顶堆**（堆顶最大）

## Step 3: 维护大小为 k 的堆

遍历 freqMap 的每个 entry，offer 进堆；一旦 size > k，poll 掉堆顶（频率最小的）。

```java
for (Map.Entry<Integer, Integer> entry : freqMap.entrySet()) {
    pq.offer(entry);
    if (pq.size() > k) {
        pq.poll();
    }
}
```

**注意**: 是 `size() > k` 而不是 `>= k`。先 offer 进去（此时可能是 k+1 个），再把最小的 poll 出去，剩下稳定 k 个。

## Step 4: 提取结果

堆里剩下的 k 个就是答案，依次 poll 出来取 key。

```java
int[] result = new int[k];
for (int i = 0; i < k; i++) {
    result[i] = pq.poll().getKey();
}
return result;
```

## 完整代码

```java
public int[] topKFrequent(int[] nums, int k) {
    // Step 1: 统计频率
    Map<Integer, Integer> freqMap = new HashMap<>();
    for (int num : nums) {
        freqMap.put(num, freqMap.getOrDefault(num, 0) + 1);
    }

    // Step 2: 小顶堆，按频率升序
    PriorityQueue<Map.Entry<Integer, Integer>> pq = new PriorityQueue<>(
        (a, b) -> a.getValue() - b.getValue()
    );

    // Step 3: 维护大小为 k 的堆
    for (Map.Entry<Integer, Integer> entry : freqMap.entrySet()) {
        pq.offer(entry);
        if (pq.size() > k) {
            pq.poll();
        }
    }

    // Step 4: 提取结果
    int[] result = new int[k];
    for (int i = 0; i < k; i++) {
        result[i] = pq.poll().getKey();
    }
    return result;
}
```

## 其他写法参考

### 写法 1: 大顶堆 (全部入堆，poll 前 k 个)

思路更直观但空间更大: 所有 (num, freq) 全部入堆，堆顶永远是频率最高的，poll k 次即可。

**Comparator 说明**:
> 返回负数，第一个参数排在前面；返回正数，第二个参数排在前面。
> 对于 PriorityQueue: 从队头到队尾按从小到大排是小顶堆，从大到小排是大顶堆（队头 = 堆的根节点）。

```java
public int[] topKFrequent1(int[] nums, int k) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int num : nums) {
        map.put(num, map.getOrDefault(num, 0) + 1);
    }
    // 大顶堆: pair2[1] - pair1[1] 降序，出现次数多的在队头
    PriorityQueue<int[]> pq = new PriorityQueue<>((pair1, pair2) -> pair2[1] - pair1[1]);
    // 全部元素入堆
    for (Map.Entry<Integer, Integer> entry : map.entrySet()) {
        pq.add(new int[]{entry.getKey(), entry.getValue()});
    }
    // 依次弹出前 k 个
    int[] ans = new int[k];
    for (int i = 0; i < k; i++) {
        ans[i] = pq.poll()[0];
    }
    return ans;
}
```

- **时间**: O(n log n)，所有元素都要入堆
- **空间**: O(n)

### 写法 2: 小顶堆 (用 int[] 存 tuple，size < k 和 peek 比较)

和主解法思路一致，区别是用 `int[]{num, freq}` 代替 `Map.Entry`，并且先判断 `size < k` 再决定是否入堆（而不是先入堆再 poll）。

```java
public int[] topKFrequent2(int[] nums, int k) {
    Map<Integer, Integer> map = new HashMap<>();
    for (int num : nums) {
        map.put(num, map.getOrDefault(num, 0) + 1);
    }
    // 小顶堆: pair1[1] - pair2[1] 升序，出现次数少的在队头
    PriorityQueue<int[]> pq = new PriorityQueue<>((pair1, pair2) -> pair1[1] - pair2[1]);

    for (Map.Entry<Integer, Integer> entry : map.entrySet()) {
        if (pq.size() < k) {
            // 堆还没满，直接加
            pq.add(new int[]{entry.getKey(), entry.getValue()});
        } else if (entry.getValue() > pq.peek()[1]) {
            // 当前频率大于堆顶（top k 里最小的），替换
            pq.poll();
            pq.add(new int[]{entry.getKey(), entry.getValue()});
        }
    }
    // 从队头到队尾频率递增，倒序填充让结果按频率从高到低
    int[] ans = new int[k];
    for (int i = k - 1; i >= 0; i--) {
        ans[i] = pq.poll()[0];
    }
    return ans;
}
```

### 写法 3: 小顶堆简洁版 (先加后判断)

和写法 2 等价，逻辑更紧凑: 先无脑 offer，超过 k 就 poll。这就是主解法，只是这里用 `int[]` 代替 `Map.Entry`。

```java
public int[] topKFrequent(int[] nums, int k) {
    PriorityQueue<int[]> pq = new PriorityQueue<>((o1, o2) -> o1[1] - o2[1]);  // 小顶堆
    Map<Integer, Integer> map = new HashMap<>();
    for (int num : nums) map.put(num, map.getOrDefault(num, 0) + 1);

    for (var x : map.entrySet()) {
        pq.offer(new int[]{x.getKey(), x.getValue()});
        if (pq.size() > k) {
            pq.poll();  // 超过 k 就弹出最小的
        }
    }

    int[] res = new int[k];
    for (int i = 0; i < k; i++) {
        res[i] = pq.poll()[0];
    }
    return res;
}
```

### 三种写法对比

| 写法 | 堆类型 | 堆大小 | 时间 | 空间 | 特点 |
|------|--------|--------|------|------|------|
| 写法 1 | 大顶堆 | n | O(n log n) | O(n) | 直观，不适合大数据 |
| 写法 2 | 小顶堆 | k | O(n log k) | O(k) | 用 peek 提前剪枝 |
| 写法 3 | 小顶堆 | k | O(n log k) | O(k) | 先加后判断，代码最简洁 ⭐ |

**推荐写法 3**: 代码最简洁，复杂度最优，逻辑清晰。写法 2 用 peek 提前判断，常数稍快但代码稍繁。

## 复杂度分析

- **时间复杂度**: O(n log k)
  - 统计频率: O(n)
  - 堆操作: 每次 offer/poll 是 O(log k)，共遍历 n 个不同 key（最多 n 个）
- **空间复杂度**: O(n + k)
  - HashMap: O(n)
  - 堆: O(k)

## 总结

1. **频率统计用 HashMap**，是 "数量/出现次数" 类问题的标配
2. **Top K 大用小顶堆，Top K 小用大顶堆**，这是反直觉但正确的关键结论
3. **堆只维护 k 个元素**，空间 O(k) 比 O(n) 的大顶堆更优
4. **`size() > k` 再 poll**，保证堆最终稳定在 k 个
5. **Comparator 口诀**: `a - b` 升序（小顶堆），`b - a` 降序（大顶堆）
6. **进阶解法**: 桶排序可以做到 O(n)，按频率作为桶的索引
