# 239. Sliding Window Maximum

> [LeetCode 239. 滑动窗口最大值](https://leetcode.com/problems/sliding-window-maximum/)

给定一个数组 `nums` 和滑动窗口大小 `k`，返回每个窗口中的最大值。

---

## 核心思路：单调队列

### 为什么不用暴力 / 堆？

| 方法 | 时间复杂度 | 问题 |
|------|-----------|------|
| 暴力扫描每个窗口 | O(nk) | 每次窗口滑动都要**重新扫一遍 k 个元素**，大量重复计算 |
| Max Heap (PriorityQueue) | O(n log n) | 删除滑出窗口的任意元素是 O(n)，需要**懒删除**技巧 |
| **单调队列** | **O(n)** | 每个元素最多进队、出队各一次，均摊 O(1) |

### 单调队列的本质

> **单调队列不是新的数据结构，而是一种"使用 Deque 的约定"**：通过特定的入队/出队规则，让队列里的元素始终保持单调性。

**关键洞察 🔑**：如果窗口里有 `i < j` 且 `nums[i] <= nums[j]`，那么 `nums[i]` 永远不可能是最大值了（因为 j 更晚滑出且值更大），可以直接扔掉。

扔掉所有"废物"后，剩下的元素**自然单调递减**，队首就是当前窗口的最大值。

### 三步走模板（灵神模板）

1. **入**：新元素入队前，把队尾所有 ≤ 它的元素弹出（维护单调性）
2. **出**：队首下标如果已经滑出窗口（`<= i - k`），弹出
3. **记录答案**：当窗口形成后（`i >= k - 1`），队首即为最大值

## 写法一：封装 MyQueue 类（存值）

**思路**：自己封装一个 `MyQueue` 类，里面维护一个单调递减的 Deque，**存的是值**。通过 `poll(val)` 判断是否需要弹出队首。

```java
class MyQueue {
    Deque<Integer> deque = new LinkedList<>();

    // 弹出元素时，比较当前要弹出的数值是否等于队列出口的数值，如果相等则弹出
    void poll(int val) {
        if (!deque.isEmpty() && val == deque.peek()) {
            deque.poll();
        }
    }

    // 添加元素时，如果要添加的元素大于入口处的元素，就将入口元素弹出
    // 保证队列元素单调递减
    void add(int val) {
        while (!deque.isEmpty() && val > deque.getLast()) {
            deque.removeLast();
        }
        deque.add(val);
    }

    // 队列队顶元素始终为最大值
    int peek() {
        return deque.peek();
    }
}

class Solution {
    public int[] maxSlidingWindow(int[] nums, int k) {
        if (nums.length == 1) {
            return nums;
        }
        int len = nums.length - k + 1;
        int[] res = new int[len];
        int num = 0;
        MyQueue myQueue = new MyQueue();

        // 先将前 k 个元素放入队列
        for (int i = 0; i < k; i++) {
            myQueue.add(nums[i]);
        }
        res[num++] = myQueue.peek();

        for (int i = k; i < nums.length; i++) {
            // 滑动窗口移除最前面的元素（如果它还在队首才移除）
            myQueue.poll(nums[i - k]);
            // 滑动窗口加入最后面的元素
            myQueue.add(nums[i]);
            // 记录对应的最大值
            res[num++] = myQueue.peek();
        }
        return res;
    }
}
```

**优点**：逻辑清晰，把单调队列的"入队、出队、取最大"三个操作封装起来，可复用性强。

**注意点**：
- `poll(val)` 只在**队首值等于要移出的值**时才真正弹出。因为如果要移出的值早就被 `add` 弹掉了（被更大的值取代），根本不在队里。
- 存的是**值**，所以判断是否滑出窗口靠"主动传入要移出的 `nums[i-k]`"。

## 写法二：直接用 Deque 存下标（推荐）

**思路**：直接在 `Solution` 里用 `ArrayDeque`，**存下标**而不是值。这样可以直接通过下标判断是否滑出窗口。

```java
class Solution {
    public int[] maxSlidingWindow(int[] nums, int k) {
        ArrayDeque<Integer> deque = new ArrayDeque<>();
        int n = nums.length;
        int[] res = new int[n - k + 1];
        int idx = 0;

        for (int i = 0; i < n; i++) {
            // 1. 出：队首下标如果滑出窗口 [i-k+1, i]，弹出
            while (!deque.isEmpty() && deque.peek() < i - k + 1) {
                deque.poll();
            }

            // 2. 入：维护单调递减，弹出队尾所有小于当前值的下标
            while (!deque.isEmpty() && nums[deque.peekLast()] < nums[i]) {
                deque.pollLast();
            }
            deque.offer(i);

            // 3. 记录答案：窗口形成后，队首下标对应的值就是最大值
            if (i >= k - 1) {
                res[idx++] = nums[deque.peek()];
            }
        }
        return res;
    }
}
```

**优点**：
- **存下标**可以直接用 `i - k + 1` 判断是否过期，不用额外传参
- 代码更简洁，是面试首选写法

**队列形态示意**（`head → tail`，存的是下标，对应的值递减）：
```
head → 3 → 2 → 1 → 0 ← tail
      （对应 nums 值单调递减）
```

## 写法三：灵神三步走模板（最标准）

**思路**：严格按照"入 → 出 → 记录答案"三步走，逻辑最清晰。

```java
class Solution {
    public int[] maxSlidingWindow(int[] nums, int k) {
        int n = nums.length;
        int[] res = new int[n - k + 1];
        Deque<Integer> dq = new ArrayDeque<>();

        for (int i = 0; i < n; i++) {
            // 1. 入：维护单调性，把比当前小的都踢掉
            while (!dq.isEmpty() && nums[dq.peekLast()] <= nums[i]) {
                dq.pollLast();
            }
            dq.offerLast(i);

            // 2. 出：队首下标滑出窗口就移除
            if (dq.peekFirst() <= i - k) {
                dq.pollFirst();
            }

            // 3. 记录答案：窗口形成后，队首就是最大值
            if (i >= k - 1) {
                res[i - k + 1] = nums[dq.peekFirst()];
            }
        }
        return res;
    }
}
```

**与写法二的区别**：
- 写法二用 `while` 循环处理"出"（其实 `if` 就够了，因为每次最多滑出一个）
- 写法三用 `if`，更贴合实际逻辑
- 写法三的"入"用 `<=`，写法二用 `<`。**两者都对**：相等的元素扔不扔都不影响答案，`<=` 队列更短一点

---

## 三种写法对比

| 维度 | 写法一（封装类） | 写法二（存下标 + while 出队） | 写法三（灵神三步走） |
|------|----------------|---------------------------|-------------------|
| 队列存什么 | 值 | 下标 | 下标 |
| 判断滑出方式 | 主动传入 `nums[i-k]` 比较 | 下标比较 `< i-k+1` | 下标比较 `<= i-k` |
| 入/出顺序 | 先出后入 | 先出后入 | **先入后出**（更符合直觉） |
| 代码量 | 较多 | 中 | 少 |
| 面试推荐 | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

---

## 复杂度分析

- **时间复杂度**：`O(n)`
  - 每个元素最多入队一次、出队一次，均摊 O(1)
  - 虽然有 `while` 循环，但总操作数 ≤ 2n
- **空间复杂度**：`O(k)`
  - Deque 最多存 k 个元素（超出窗口会被弹出）

---

## 总结

1. **问题特征识别**：滑动窗口 + 求最值 → 单调队列
2. **核心思想**：及时淘汰"不可能成为答案的废物"，让候选者自动单调排列
3. **淘汰规则**（求最大值时）：新元素让队尾所有 ≤ 它的元素变废物
4. **数据结构选择**：`ArrayDeque`（比 `LinkedList` 性能更好）
5. **存下标 vs 存值**：**存下标更灵活**，能直接判断是否滑出窗口
6. **三步走模板**：**入 → 出 → 记录答案**，所有单调队列题目通用
