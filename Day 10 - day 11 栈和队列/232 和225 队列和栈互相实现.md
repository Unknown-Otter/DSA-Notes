# 232. 用栈实现队列

> [LeetCode 链接](https://leetcode.com/problems/implement-queue-using-stacks/)

## 核心思路

用两个栈模拟队列的 FIFO。关键洞察:**栈倒进栈会反转顺序,反转两次正好变成队列顺序**。

## 错误历程

### ❌ 版本 1:类型不匹配 + 逻辑缺失

```java
class MyQueue {
    Deque<Integer> main = new Stack<>(); 
    Deque<Integer> helper = new Stack<>(); 

    public void push(int x) {
        main.push(x);
    }
    
    public int pop() {
        while(!main.empty()){
            helper.push(main.pop());
        }
        helper.pop();   // 没有 return
    }
    
    public int peek() {
        while(!main.isEmpty()){
            helper.push(main.pop());
        }
        int temp = helper.pop();
        helper.push(temp);
        return temp; 
    }

    public boolean empty() {
        if(main.isEmpty() && helper.isEmpty()) return true; 
        return false; 
    }
}
```

**问题:**
1. `Stack` 不是 `Deque` 的实现类,类型不兼容,编译失败
2. `Deque` 没有 `empty()` 方法,只有 `isEmpty()`
3. `pop()` 缺少 `return` 语句
4. **核心逻辑缺陷**:每次 pop/peek 都把 main 倒进 helper,但**从不倒回来**,导致新 push 的元素和 helper 里已有元素顺序混乱

### ❌ 版本 2:修了类型,但核心逻辑还错

```java
public int pop() {
    while(!main.isEmpty()){
        helper.push(main.pop());
    }
    return helper.pop();
}
```

**问题复现:**
```
push(1), push(2)     → main=[2,1], helper=[]
pop()                → helper=[1,2], 弹出 1 ✓
push(3)              → main=[3],   helper=[2]
pop()                → 把 main 倒进 helper → helper=[3,2]
                     → 弹出 3 ✗  (应该是 2!)
```

**根本原因**:helper 里已经是队列顺序(栈顶 = 队首),再往里倒新元素会污染顺序。

### ❌ 版本 3:return 之后的死代码

```java
public int pop() {
    while(!main.isEmpty()){
        helper.push(main.pop());
    }
    return helper.pop();

    while(!helper.isEmpty()){       // unreachable statement,编译报错
        main.push(helper.pop());
    }
}
```

**问题:** `return` 已经结束方法,后面的 `while` 编译器直接报错。

### ⚠️ 版本 4(可通过但效率差):每次都倒回去

```java
public int pop() {
    while (!main.isEmpty()) {
        helper.push(main.pop());
    }
    int res = helper.pop();
    while (!helper.isEmpty()) {      // 倒回去保持不变量
        main.push(helper.pop());
    }
    return res;
}

public int peek() {
    while (!main.isEmpty()) {
        helper.push(main.pop());
    }
    int res = helper.peek();
    while (!helper.isEmpty()) {
        main.push(helper.pop());
    }
    return res;
}
```

**能通过,但每次 pop/peek 都是 O(n),n 次操作 O(n²)。**

## ✅ 最优解:懒搬运(摊还 O(1))

### 核心思想

**两个栈分工固定,不要来回倒:**
- `in` 栈:只负责接收新元素(push)
- `out` 栈:只负责出队(pop / peek)
- 一旦元素从 `in` 搬到 `out`,**绝对不会再倒回 `in`**

### 关键不变量

`out` 栈里的元素,**从栈顶到栈底就是队列的出队顺序**(栈顶 = 队首)。

### 搬运时机

**只有当 `out` 空了,才把 `in` 里所有元素一次性倒进 `out`。**

为什么?如果 `out` 还有元素,说明这些是更早 push 的,必须先出队。此时往 `out` 塞新元素会让新元素压在栈顶,抢在老元素前面出队 —— 顺序乱了。

### ❌ 踩坑:搬运循环条件写反

```java
while(out.isEmpty()){          // 搬一个进去 out 就不空了,循环退出
    out.push(in.pop());         // 只搬了一个!
}
```

### ✅ 正确写法:外层 `if` 管触发,内层 `while` 管搬运量

```java
class MyQueue {
    Deque<Integer> in = new ArrayDeque<>();
    Deque<Integer> out = new ArrayDeque<>();

    public MyQueue() {}

    public void push(int x) {
        in.push(x);
    }

    public int pop() {
        if (out.isEmpty()) {
            while (!in.isEmpty()) {
                out.push(in.pop());
            }
        }
        return out.pop();
    }

    public int peek() {
        if (out.isEmpty()) {
            while (!in.isEmpty()) {
                out.push(in.pop());
            }
        }
        return out.peek();
    }

    public boolean empty() {
        return in.isEmpty() && out.isEmpty();
    }
}
```

### 执行示例(栈顶在左)

```
push(1), push(2), push(3)        in=[3,2,1], out=[]

peek()
  out 空 → 搬运                  in=[], out=[1,2,3]
  返回 out.peek() = 1

pop()
  out 非空 → 不搬                in=[], out=[1,2,3]
  返回 out.pop() = 1              in=[], out=[2,3]

push(4), push(5)                 in=[5,4], out=[2,3]

pop()
  out 非空 → 不搬                in=[5,4], out=[2,3]
  返回 2                          in=[5,4], out=[3]

pop()                            返回 3, in=[5,4], out=[]

pop()
  out 空 → 搬运                  in=[], out=[4,5]
  返回 4                          in=[], out=[5]
```

### 复杂度分析

- `push`: O(1)
- `pop` / `peek`: **摊还 O(1)**

**摊还分析**:每个元素一生只经历 4 次栈操作 —— push 到 `in`、从 `in` 弹出、压入 `out`、从 `out` 弹出。n 次队列操作总共 O(n) 次栈操作。

单次 pop 最坏 O(n)(触发搬运时),但紧接着 n-1 次 pop 都是 O(1),均摊 O(1)。

## 总结

1. **用栈实现队列**:利用"栈倒栈 = 反转"的特性,反转两次得到 FIFO
2. **最优解是懒搬运**:分工固定,`out` 空了才搬,搬过去就不动
3. **易错点**:搬运的触发条件(`if out 空`)和搬运量(`while in 非空`)是两层不同的判断

# 225. 用队列实现栈

> [LeetCode 链接](https://leetcode.com/problems/implement-stack-using-queues/)

## 核心思路

**这题不能套懒搬运!** 因为队列倒进队列顺序不变,反转两次还是原序,永远拼不出 LIFO。

## 错误历程

### ❌ 版本 1:照搬懒搬运模板

```java
class MyStack {
    Queue<Integer> in = new LinkedList<>();
    Queue<Integer> out = new LinkedList<>(); 

    public void push(int x) {
        in.offer(x);
    }
    
    public int pop() {
        while(out.isEmpty()){
            while(!in.isEmpty()){
                out.offer(in.poll());
            }
        }
        int res = out.poll();
        return res; 
    }
    // ...
}
```

**问题复现:**
```
push(1), push(2), push(3)    in=[1,2,3], out=[]
pop()
  out 空 → 搬 → in=[], out=[1,2,3]
  out.poll() = 1 ✗  应该是 3!
```

**根本原因:顺序没有被反转。**

## 关键洞察:为什么懒搬运对这题失效

| 操作 | 顺序变化 |
|------|----------|
| 栈 push → pop | **反转** (LIFO) |
| 队列 offer → poll | **不变** (FIFO) |

- **栈 → 栈 → 目标**:反转两次 = 原序 = 队列 ✅
- **队列 → 队列 → 目标**:原序两次 = 原序 = 队列,**永远不是栈** ❌

懒搬运的全部魔法在于"倒一次 = 反转一次"这个副作用。队列没有这个副作用,所以这招对它失效。

## ✅ 正解:push 时预排序

既然队列改不了 FIFO,干脆**在 push 时就把顺序预先排成栈的样子**(栈顶 = 队首)。

**每次 push 新元素后,把它前面所有元素重新排到它后面。**

```java
class MyStack {
    Queue<Integer> q = new LinkedList<>();

    public MyStack() {}

    public void push(int x) {
        q.offer(x);
        // 把 x 之前的所有元素挪到 x 后面
        for (int i = 0; i < q.size() - 1; i++) {
            q.offer(q.poll());
        }
    }

    public int pop() {
        return q.poll();
    }

    public int top() {
        return q.peek();
    }

    public boolean empty() {
        return q.isEmpty();
    }
}
```

### 执行示例

```
push(1)
  offer 1              → q=[1]
  循环 0 次(size-1=0)

push(2)
  offer 2              → q=[1,2]
  循环 1 次:
    poll 1, offer 1    → q=[2,1]    ← 2 变成队首

push(3)
  offer 3              → q=[2,1,3]
  循环 2 次:
    poll 2, offer 2    → q=[1,3,2]
    poll 1, offer 1    → q=[3,2,1]  ← 3 变成队首

pop() → 3 ✓
pop() → 2 ✓
pop() → 1 ✓
```

### 复杂度

- `push`: **O(n)**(每次要重排)
- `pop` / `top` / `empty`: O(1)

**用队列实现栈,push 和 pop 必有一方是 O(n),这是数据结构差异决定的。** 可以把贵的放在 push(上面做法),也可以放在 pop(push 直接 offer,pop 时把前 n-1 个倒到另一个队列),效果等价。

一个队列就够了,两个队列反而多余。

## 总结对比

| 题目 | 数据结构特性 | 策略 | 复杂度 |
|------|--------------|------|--------|
| 232. 栈实现队列 | 栈倒栈 → 反转 | **懒搬运** | pop 摊还 O(1) |
| 225. 队列实现栈 | 队列倒队列 → 不变 | **push 时预排序** | push O(n) |

## 通用思考框架

遇到"用 A 实现 B"类题目,先问:

1. **A 的基本操作会不会改变顺序?改变成什么样?**
2. **目标 B 需要什么顺序?**
3. **A 的顺序变换能不能拼出 B 的顺序?**

- 能拼 → 懒搬运(省力)
- 不能拼 → 主动排序(硬掰,必有一端 O(n))
