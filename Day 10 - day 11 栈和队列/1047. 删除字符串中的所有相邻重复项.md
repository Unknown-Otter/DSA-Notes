# 1047. 删除字符串中的所有相邻重复项

> [LeetCode 1047. Remove All Adjacent Duplicates In String](https://leetcode.com/problems/remove-all-adjacent-duplicates-in-string/)

## 题目思路

用**栈**来做：遍历字符串，如果当前字符和栈顶相同，就把栈顶 `pop` 出去（抵消）；否则 `push` 进栈。最后栈里剩下的就是答案。

Java 中推荐用 `ArrayDeque` 作为栈（`Stack` 类已被官方不推荐）。

---

## 我的初版错误代码

```java
class Solution {
    public String removeDuplicates(String s) {
        Deque<Character> deque = new ArrayDeque<>(); 

        for(char c : s.toCharArray()){
            if(deque.peek() == 'c'){
                deque.pop(); 
            }
            deque.push(c);
        }

        return s;
    }
}
```

### 错误点汇总

1. `deque.peek() == 'c'` —— 比较的是字面量字母 c，不是当前变量
2. 没有判空，`peek()` 返回 `null` 时会 NPE
3. `return s` 返回的是原字符串，没用到栈的结果
4. `if` 没配 `else`，导致 pop 之后又 push 回来

---

## 误区 1：`'c'` 和 `c` 的区别

### 我的错误思路

我以为 `deque.peek() == 'c'` 就是在比较栈顶和当前字符。

### 正解

| 写法 | 含义 | 值 |
|------|------|-----|
| `c` | 循环变量（不带引号） | 随遍历变化 |
| `'c'` | **字符字面量**（带单引号） | 永远是字母 c |

**带单引号的就是字面值，不是变量**。就像 `"name"` 是字符串，而 `name` 是变量。

### 正确写法

```java
if (deque.peek() == c) {   // 栈顶 == 当前遍历到的字符
    deque.pop();
}
```

---

## 误区 2：为什么必须加 `else`

### 我的错误思路

我以为可以照搬之前写过的模式：

```java
for (int num : arr) {
    if (map.containsKey(num)) {
        // 做点什么
    }
    map.put(num, ...);   // 不管 if 成不成立，都要 put
}
```

这里 `map.put` 无论如何都要做，所以不用 `else`。我以为这题也一样，满足条件就 pop，否则一直 push。

### 正解：**互斥逻辑 vs 共存逻辑**

- **共存逻辑**：`if` 里的操作做完后，后面的操作**还该做** → 不用 else
- **互斥逻辑**：`if` 里的操作和后面的操作**冲突** → 必须用 if-else

这题里：
- `pop` 的语义是"c 和栈顶抵消，两个都消失"
- `push` 的语义是"c 留下来"

**两件事自相矛盾，不能同时发生**。

### 用 `"aa"` 验证错误代码

| 轮次 | c | 操作 | deque |
|------|---|------|-------|
| 1 | `a` | 栈空 → push | `[a]` |
| 2 | `a` | 栈顶==a → pop，**然后又 push** | `[a]` ❌ |

正确答案应该是 `""`，但错误代码输出 `"a"`（pop 之后又把 a 塞回来，白抵消了）。

### 正确写法

```java
if (!deque.isEmpty() && deque.peek() == c) {
    deque.pop(); 
} else {
    deque.push(c);
}
```

### 判断口诀

> **"if 成立之后，后面那行还该不该跑？"**
> - 该跑 → 单独 `if`
> - 不该跑 / 冲突 → `if-else`

## 误区 3：Deque 转 String，顺序是反的

### 我的疑问

栈里剩下的元素怎么转成字符串返回？

### 关键观察：deque 的内部顺序

`push` 是往**头部**插入的，所以 `"abbaca"` 处理完后：

| 步骤 | c | 操作 | deque（头 ← → 尾） |
|------|---|------|-------------------|
| 1 | `a` | push | `[a]` |
| 2 | `b` | push | `[b, a]` |
| 3 | `b` | 栈顶==b，pop | `[a]` |
| 4 | `a` | 栈顶==a，pop | `[]` |
| 5 | `c` | push | `[c]` |
| 6 | `a` | push | `[a, c]` |

**正确答案是 `"ca"`**，也就是 **从栈底到栈顶**（c 先入栈，在字符串前面）。

但 `for (char ch : deque)` 的遍历顺序是 **头 → 尾**（栈顶 → 栈底），得到的是 `"ac"`，**反了**。

### 三种转换方法

**方法 1：遍历后 reverse（推荐）**

```java
StringBuilder sb = new StringBuilder();
for (char ch : deque) {
    sb.append(ch);
}
return sb.reverse().toString();
```

**方法 2：用 `pollLast()` 从尾部取**

```java
StringBuilder sb = new StringBuilder();
while (!deque.isEmpty()) {
    sb.append(deque.pollLast());
}
return sb.toString();
```

**方法 3：pop 全部然后 reverse**

```java
StringBuilder sb = new StringBuilder();
while (!deque.isEmpty()) {
    sb.append(deque.pop());
}
return sb.reverse().toString();
```

## 误区 4：Deque 到底是栈还是别的什么？

### 我的疑问

把 deque 看成 stack 吗，还是可以当成普通容器遍历？

### 正解：**Deque 是"双面人"**

`Deque` 本身是双端队列，**同一个 deque，用不同方法就呈现不同行为**：

| 用法 | 调用的方法 | 心智模型 |
|------|-----------|---------|
| 当栈 | `push` / `pop` / `peek` | 后进先出，只操作头部 |
| 当队列 | `offer` / `poll` / `peek` | 先进先出，尾进头出 |
| 当普通集合 | `for-each` 遍历 | 从头到尾线性扫一遍 |

**这题里**：
- **构建阶段** → 当栈用（`push` / `pop` / `peek`）
- **输出阶段** → 当普通集合遍历（`for-each`）

这种组合非常常见，不矛盾。只要清楚自己当前是"哪种视角"就行。

### Deque 方法速查

| 方法 | 操作位置 |
|------|---------|
| `push(x)` / `pop()` / `peek()` | **头部**（栈顶） |
| `addLast(x)` / `pollLast()` / `peekLast()` | **尾部**（栈底） |
| `for (char ch : deque)` | 遍历方向：**头 → 尾** |

## 最终正确代码

```java
class Solution {
    public String removeDuplicates(String s) {
        Deque<Character> deque = new ArrayDeque<>();

        for (char c : s.toCharArray()) {
            if (!deque.isEmpty() && deque.peek() == c) {
                deque.pop();
            } else {
                deque.push(c);
            }
        }

        StringBuilder sb = new StringBuilder();
        for (char ch : deque) {
            sb.append(ch);
        }
        return sb.reverse().toString();
    }
}
```

### 优化版：直接用 StringBuilder 当栈

省去最后的 `reverse()`，代码更简洁：

```java
class Solution {
    public String removeDuplicates(String s) {
        StringBuilder sb = new StringBuilder();
        for (char c : s.toCharArray()) {
            int len = sb.length();
            if (len > 0 && sb.charAt(len - 1) == c) {
                sb.deleteCharAt(len - 1);
            } else {
                sb.append(c);
            }
        }
        return sb.toString();
    }
}
```

## 总结

1. **`'c'` vs `c`**：带单引号是字符字面量，不带引号是变量
2. **互斥用 if-else**：pop 和 push 不能共存，必须二选一
3. **空栈保护**：`!deque.isEmpty()` 放在 `peek()` 前，利用短路求值
4. **Deque 当栈**：`push` / `pop` / `peek` 都在头部操作
5. **顺序反转**：栈底 → 栈顶才是结果字符串的正确顺序，遍历后需要 `reverse()`
6. **Deque 是双面人**：同一个数据结构，用不同方法呈现不同行为（栈 / 队列 / 可遍历集合）
