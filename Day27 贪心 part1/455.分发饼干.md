# 455. Assign Cookies（分配饼干）

> https://leetcode.com/problems/assign-cookies/

---

## 贪心算法基础回顾

### 核心思想

**一句话定义：每一步都做当前看起来最好的选择，且不回头。**

| | 动态规划 | 贪心 |
|---|---|---|
| 决策方式 | 枚举所有可能，记录子问题结果 | 每步只选一个"最优"方向 |
| 回头看 | 会考虑之前的所有状态 | 不回头，不后悔 |
| 正确性保证 | 最优子结构 + 重叠子问题 | 需要证明**贪心选择性质** |
| 复杂度 | 通常更高 | 通常更低（线性或 nlogn）|

### 贪心不是万能的

反例：硬币面值为 `[1, 3, 4]`，凑 6 分：
- 贪心选法：4 + 1 + 1 = **3枚**
- 最优解：3 + 3 = **2枚** ❌

所以贪心需要问自己：**"每步选局部最优，会不会破坏后面的解？"**

## 题目分析

### 贪心选择性质

**错误直觉**：把尽量大的饼干 match 给胃口大的孩子。

**正确思路**：目标是满足**尽量多的孩子**，不是让每个孩子吃得尽量饱。

> 关键问题：一块刚好够大的饼干，应该给胃口大还是胃口小的孩子？

**答案：给胃口小的。** 能满足小胃口的饼干，就不要用它去满足大胃口——留给更大的饼干去做那件事。

即：**用最小的能满足需求的饼干，去满足当前最小的未满足胃口。**

## 错误代码分析

### Bug 版本 1：`idx` 无条件推进

```java
for (int i = 0; i < g.length & idx < s.length; i++) {
    if (g[i] <= s[idx]) count++;
    idx++;  // ❌ 不管满没满足，饼干都被消耗掉了
}
```

**反例**：`g = [2, 3]`, `s = [1, 2]`
- `i=0`：`g[0]=2 > s[0]=1`，没满足，但 `idx` 还是 `++`，饼干 1 被白白跳过
- `i=1`：`g[1]=3 > s[1]=2`，没满足
- 结果：count=0，正确答案应为 1 ❌

**根本原因**：饼干不够时，应该换饼干，而不是换孩子。

---

### Bug 版本 2：`i` 无条件推进

```java
for (int i = 0; i < g.length & idx < s.length; i++) {
    if (g[i] <= s[idx]) {
        count++;
        idx++;
    } else if (g[i] > s[idx]) {
        idx++;  // 换饼干，但 i 也跟着 ++ 了
    }
}
```

**反例**：`g = [2, 3]`, `s = [1, 2]`
- `i=0`：`g[0]=2 > s[0]=1`，走 `else if`，`idx=1`，然后 `i++` 变成 `i=1`
- `i=1`：`g[1]=3 > s[1]=2`，没满足
- **`g[0]=2` 的孩子被跳过了！** 饼干 2 本可以满足他 ❌

**根本原因**：饼干不够时，孩子指针不应该移动，这个孩子下一轮继续等。

## 三种正确解法

### 解法一：for 遍历饼干（推荐写法）

**策略**：for 驱动饼干指针，`start`（孩子指针）只在满足时推进。

```java
class Solution {
    public int findContentChildren(int[] g, int[] s) {
        Arrays.sort(g);
        Arrays.sort(s);
        int count = 0;
        int start = 0;

        for (int i = 0; i < s.length && start < g.length; i++) {
            if (s[i] >= g[start]) {
                start++;
                count++;
            }
        }

        return count;
    }
}
```

**走一遍**：`g = [2, 3]`, `s = [1, 2]`
- `i=0`：`s[0]=1 >= g[0]=2`？ ❌ 跳过，换更大的饼干
- `i=1`：`s[1]=2 >= g[0]=2`？ ✅ count=1, start=1
- 结果：1 ✅

---

### 解法二：while 双指针（两指针独立控制）

**适用场景**：需要同时独立控制两个指针时，`while` 比 `for` 更灵活。

```java
class Solution {
    public int findContentChildren(int[] g, int[] s) {
        Arrays.sort(g);
        Arrays.sort(s);
        int count = 0;
        int i = 0, j = 0;

        while (i < g.length && j < s.length) {
            if (s[j] >= g[i]) {
                count++;
                i++;    // 孩子满足了，推进孩子
            }
            j++;        // 饼干无论如何都换
        }

        return count;
    }
}
```

> 与解法一逻辑完全相同，只是用 `while` 显式控制两个指针。

---

### 解法三：从大到小遍历（反向贪心）

**策略**：优先满足胃口最大的孩子，用最小的能满足他的饼干。

```java
class Solution {
    public int findContentChildren(int[] g, int[] s) {
        Arrays.sort(g);
        Arrays.sort(s);
        int count = 0;
        int start = s.length - 1;

        for (int index = g.length - 1; index >= 0; index--) {
            if (start >= 0 && g[index] <= s[start]) {
                start--;
                count++;
            }
        }

        return count;
    }
}
```

**走一遍**：`g = [2, 3]`, `s = [1, 2]`
- `index=1`：`g[1]=3 <= s[1]=2`？ ❌ 跳过
- `index=0`：`g[0]=2 <= s[1]=2`？ ✅ count=1, start=0
- 结果：1 ✅

---

## 三种解法对比

| 解法 | 遍历方向 | 贪心策略 | 外层循环驱动 |
|---|---|---|---|
| 解法一（for） | 从小到大 | 最小可用饼干满足最小胃口 | 饼干指针 |
| 解法二（while） | 从小到大 | 同上，双指针独立控制 | 饼干指针 |
| 解法三（反向） | 从大到小 | 最大饼干优先满足最大胃口 | 孩子指针 |

> 三种策略本质相同：**不用大饼干去满足本可用小饼干满足的孩子**，即不浪费。

---

## 解题模板总结

1. 两个数组排序
2. **外层 for/while 驱动饼干**，饼干每轮必须换
3. 够了 → 孩子指针推进，count++
4. 不够 → 这个孩子等着，换更大的饼干来试

> 注意：Java 中条件判断用 `&&`（短路运算符），而非 `&`（位运算符）。`&&` 左边为 false 时右边不执行，更安全高效。
