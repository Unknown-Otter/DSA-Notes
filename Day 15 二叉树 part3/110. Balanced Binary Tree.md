# 110. Balanced Binary Tree

> https://leetcode.com/problems/balanced-binary-tree/

给定一棵二叉树，判断它是否是高度平衡的（即每个节点的左右子树高度差不超过 1）。

---

## 我的初始思路（自顶向下，有问题）

最初想法：写一个 `getHeight` helper，在主函数里对每个节点分别调用，比较左右高度差。

**问题**：这是自顶向下的遍历。对 root 调用 `getHeight`，它会递归计算所有子节点的高度；然后对 `root.left` 再调用，又重算一遍 `root.left` 以下的所有节点。**底层节点的高度被反复计算多次**，时间复杂度是 O(n²)。

---

## 正确思路：后序遍历，自底向上

**核心思想**：让 `getHeight` 同时承担两件事——

1. 返回当前子树的**高度**
2. 如果子树不平衡，返回**哨兵值 -1**

后序遍历天然满足"先算子节点，结果向上传"的顺序，每个节点只访问一次，时间复杂度 **O(n)**。

### 哨兵值设计

| 返回值 | 含义 |
|--------|------|
| `>= 0` | 子树平衡，值就是高度 |
| `-1` | 子树某处不平衡，向上传递报警信号 |

正常高度永远 `>= 0`，所以 `-1` 不会和真实高度冲突。

---

## 最终代码

```java
class Solution {
    public boolean isBalanced(TreeNode root) {
        return getHeight(root) != -1;
    }

    private int getHeight(TreeNode root) {
        if (root == null) return 0;  // base case：空节点高度为 0

        int left = getHeight(root.left);
        if (left == -1) return -1;       // 左子树已不平衡，直接剪枝

        int right = getHeight(root.right);
        if (right == -1) return -1;      // 右子树已不平衡，直接剪枝

        if (Math.abs(left - right) > 1) return -1;  // 当前节点不平衡

        return Math.max(left, right) + 1;  // 平衡则返回真实高度
    }
}
```

## 我踩的坑和误区

### 误区一：想单独处理 null 的情况

**错误写法**：
```java
if (root.left == null) return ???
if (root.right == null) return ???
```

**为什么不需要**：`getHeight(null)` 会直接命中 `if (root == null) return 0`，base case 已经处理了 null。

**递归的正确思维**：不要在调用前判断，**放心去调用，base case 会兜底**。

---

### 误区二：不理解 `left == -1` 这个条件从哪来

困惑：`left` 不就是高度吗？高度怎么会等于 -1？

**关键**：`left` 存的是 `getHeight(root.left)` 的**返回值**，而 `getHeight` 不只返回高度，还可能返回 -1。

所以 `left` 可能是：
- 正常高度（0, 1, 2, 3...）
- **-1**（左子树深处某个节点已经触发了不平衡）

---

### 误区三：不明白 -1 是从哪产生的

**-1 的唯一出生地**：
```java
if (Math.abs(left - right) > 1) return -1;
```

**null 不产生 -1**，null 永远返回 0。只有真正不平衡的节点才产生 -1。

然后通过：
```java
if (left == -1) return -1;
if (right == -1) return -1;
```

一层一层往上传递，直到 `isBalanced` 收到它。

---

## -1 传播的完整追踪

以这棵不平衡的树为例：
```
    1
   /
  2
 /
3
```

| 调用 | left | right | 操作 | 返回值 |
|------|------|-------|------|--------|
| `getHeight(3)` | 0 | 0 | 差为 0，平衡 | 1 |
| `getHeight(2)` | 1 | 0 | 差为 1，平衡 | 2 |
| `getHeight(1)` | 2 | 0 | 差为 **2 > 1**，不平衡 | **-1** |
| `isBalanced` | — | — | 收到 -1 | `false` |

## 解法二：迭代法（暴力，O(n²)）

**思路**：用栈模拟后序遍历，遍历到每个节点时，调用 `getHeight` 分别计算左右子树高度并比较。`getHeight` 用 BFS 层序遍历实现。

**问题所在**：每次调用 `getHeight` 都从该节点往下完整遍历一遍，和解法一（自顶向下递归）一样存在重复计算，时间复杂度 O(n²)。

```java
class Solution {
    public boolean isBalanced(TreeNode root) {
        if (root == null) return true;
        Stack<TreeNode> stack = new Stack<>();
        TreeNode pre = null;
        while (root != null || !stack.isEmpty()) {
            while (root != null) {
                stack.push(root);
                root = root.left;
            }
            TreeNode inNode = stack.peek();
            // 右节点为 null 或已经遍历过 → 当前节点可以处理了（后序时机）
            if (inNode.right == null || inNode.right == pre) {
                if (Math.abs(getHeight(inNode.left) - getHeight(inNode.right)) > 1) {
                    return false;
                }
                stack.pop();
                pre = inNode;
                root = null;
            } else {
                root = inNode.right;
            }
        }
        return true;
    }

    // BFS 层序遍历求高度，每次调用都是 O(n)
    public int getHeight(TreeNode root) {
        if (root == null) return 0;
        Deque<TreeNode> deque = new LinkedList<>();
        deque.offer(root);
        int depth = 0;
        while (!deque.isEmpty()) {
            int size = deque.size();
            depth++;
            for (int i = 0; i < size; i++) {
                TreeNode poll = deque.poll();
                if (poll.left != null) deque.offer(poll.left);
                if (poll.right != null) deque.offer(poll.right);
            }
        }
        return depth;
    }
}
```

## 解法三：优化迭代法（O(n)）⭐

**核心优化**：把每个节点的高度**缓存在 `node.val` 上**，这样 `getHeight` 只需 O(1) 直接读子节点的 val，而不用重新遍历。

**关键前提**：后序遍历保证了处理父节点时，子节点的 val 已经被更新为正确高度。

```java
class Solution {
    public boolean isBalanced(TreeNode root) {
        if (root == null) return true;
        Stack<TreeNode> stack = new Stack<>();
        TreeNode pre = null;
        while (root != null || !stack.isEmpty()) {
            while (root != null) {
                stack.push(root);
                root = root.left;
            }
            TreeNode inNode = stack.peek();
            if (inNode.right == null || inNode.right == pre) {
                if (Math.abs(getHeight(inNode.left) - getHeight(inNode.right)) > 1) {
                    return false;
                }
                stack.pop();
                pre = inNode;
                root = null;
            } else {
                root = inNode.right;
            }
        }
        return true;
    }

    // O(1)：直接读子节点已缓存的高度，同时更新当前节点的 val
    public int getHeight(TreeNode root) {
        if (root == null) return 0;
        int leftHeight = root.left != null ? root.left.val : 0;   // 读缓存
        int rightHeight = root.right != null ? root.right.val : 0; // 读缓存
        int height = Math.max(leftHeight, rightHeight) + 1;
        root.val = height;  // ← 把高度写入 val，供父节点直接读取
        return height;
    }
}
```

**注意**：这个方法修改了原始节点的 val，在实际工程中需谨慎；面试中如果题目允许修改树结构则可以使用。

---

## 三种解法全面对比

| | 解法一：递归（推荐）| 解法二：迭代暴力 | 解法三：迭代优化 |
|---|---|---|---|
| **实现方式** | 递归后序遍历 | 栈模拟后序 + BFS 求高度 | 栈模拟后序 + val 缓存高度 |
| **时间复杂度** | O(n) | O(n²) | O(n) |
| **空间复杂度** | O(n) 递归栈 | O(n) | O(n) |
| **重复计算** | 无（哨兵值剪枝）| 有（每次 getHeight 重新遍历）| 无（读缓存 O(1)）|
| **核心技巧** | 哨兵值 -1 | — | 用 node.val 缓存高度 |
| **修改原始数据** | 否 | 否 | **是（val 被覆盖）** |
| **代码简洁度** | ⭐⭐⭐ | ⭐ | ⭐⭐ |

## 关键总结

```
null        → 返回 0（正常，高度为 0）
不平衡节点   → 产生 -1（报警信号的唯一来源）
祖先节点     → 看到 -1 就继续往上传，不做无用功（剪枝）
```

**三种解法的本质区别**就是"如何避免重复计算高度"：
- 解法一：递归自底向上，顺路把高度和平衡状态一起返回，天然无重复
- 解法二：迭代但没缓存，每次重新算高度，退化成 O(n²)
- 解法三：迭代 + 手动缓存到 val，把求高度降到 O(1)，总体 O(n)

**面试首选解法一**，代码最简洁，逻辑最清晰，时间复杂度最优。
