# LC 257 · 二叉树的所有路径

> https://leetcode.cn/problems/binary-tree-paths/

## 题意

给定一棵二叉树的根节点 `root`，返回所有从根节点到叶子节点的路径，路径格式为 `"1->2->5"`。

**叶子节点**：左右孩子都为 `null` 的节点。

示例树：

```
      1
     / \
    2   3
     \
      5
```

输出：`["1->2->5", "1->3"]`

---

## 核心思路

- 需要从根到叶的完整路径 → 用**前序遍历**（中 → 左 → 右），这样父节点先于孩子节点入 path。
- 路径有"探索 + 撤回"的特点 → 需要**回溯**：走完一条路径后，把路径末尾的节点删掉，再去探索其他分支。
- 递归和回溯是一一对应的关系，**有一个递归调用，就必须有一个对应的回溯（`remove`）**，且两者必须在同一个花括号内。

---

## 解法一：递归 + 显式回溯（推荐初学）

**执行结构：**

```
traversal(node):
    path.add(node.val)          // ① 进来就加（前序）

    if 叶子节点:
        拼 StringBuilder → result  // ② 中间收割路径
        return

    if 有左孩子:
        traversal(left)         // ③ 递归下去
        path.remove(最后一个)   // ④ 回来就删（回溯）

    if 有右孩子:
        traversal(right)        // ③ 递归下去
        path.remove(最后一个)   // ④ 回来就删（回溯）
```

**关键点：**
- `path.add()` 在函数**入口**，`path.remove()` 在递归调用**返回后**，叶子节点的拼接在**中间**的终止条件里。三件事互不干扰。
- `remove()` 不需要"知道"叶子里发生了什么。叶子收割结果写进 `result` 后，path 里的叶子值还在，由**上一层**的 `remove()` 负责清掉。
- 每一层只负责清理"自己递归出去的那个孩子"，职责对称清晰。

**完整代码：**

```java
class Solution {
    public List<String> binaryTreePaths(TreeNode root) {
        List<String> result = new ArrayList<>();
        List<Integer> path = new ArrayList<>();
        if (root == null) return result;
        traversal(root, path, result);
        return result;
    }

    private void traversal(TreeNode cur, List<Integer> path, List<String> result) {
        path.add(cur.val); // 前序：先把当前节点加入 path

        // 终止条件：到达叶子节点
        if (cur.left == null && cur.right == null) {
            StringBuilder sb = new StringBuilder();
            for (int i = 0; i < path.size() - 1; i++) {
                sb.append(path.get(i)).append("->");
            }
            sb.append(path.get(path.size() - 1));
            result.add(sb.toString());
            return;
        }

        // 递归左子树 + 回溯
        if (cur.left != null) {
            traversal(cur.left, path, result);
            path.remove(path.size() - 1); // 回溯！删掉刚才加进去的左孩子
        }

        // 递归右子树 + 回溯
        if (cur.right != null) {
            traversal(cur.right, path, result);
            path.remove(path.size() - 1); // 回溯！删掉刚才加进去的右孩子
        }
    }
}
```

**为什么用 `List<Integer>` 而不是 `String`？**
- 每次 `remove()` 恰好删掉一个 `int` 元素，无论节点值是几位数都只调用一次。
- 若用 `String` 引用传递，删 `"->"` 需要 `pop_back` 两次，删节点值还要按位数来，代码冗余。

## 解法二：递归 + 隐式回溯（值传递）

利用 Java 的**值传递**特性：`String path` 作为参数，每次调用时传入 `path + "->"` 会创建一个新字符串，不会修改外层的 `path`，函数返回后 `path` 自动还原。**回溯被隐藏在参数传递里。**

```java
class Solution {
    List<String> result = new ArrayList<>();

    public List<String> binaryTreePaths(TreeNode root) {
        if (root == null) return result;
        dfs(root, "");
        return result;
    }

    private void dfs(TreeNode node, String path) {
        if (node == null) return;

        // 叶子节点：直接收割
        if (node.left == null && node.right == null) {
            result.add(path + node.val);
            return;
        }

        // path + "->" 是新字符串，外层 path 不变 → 等价于回溯
        String next = path + node.val + "->";
        dfs(node.left, next);
        dfs(node.right, next);
    }
}
```

**对比解法一：**

| | 解法一（显式回溯） | 解法二（隐式回溯） |
|---|---|---|
| 回溯方式 | 手动 `path.remove()` | 利用值传递，函数返回自动还原 |
| 回溯可见性 | 高，每步可见 | 隐藏在参数传递里 |
| 代码量 | 稍多 | 简洁 |
| 推荐场景 | 初学，理解回溯本质 | 熟悉原理后的写法 |

## 解法三：迭代法（栈模拟）

用两个栈分别保存**节点**和**对应的路径字符串**，模拟前序遍历。

```java
class Solution {
    public List<String> binaryTreePaths(TreeNode root) {
        List<String> result = new ArrayList<>();
        if (root == null) return result;

        Stack<Object> stack = new Stack<>();
        // 节点和路径字符串成对入栈
        stack.push(root);
        stack.push(root.val + "");

        while (!stack.isEmpty()) {
            String path = (String) stack.pop();
            TreeNode node = (TreeNode) stack.pop();

            // 遇到叶子节点，收割路径
            if (node.left == null && node.right == null) {
                result.add(path);
            }

            // 右孩子先入栈（后处理）
            if (node.right != null) {
                stack.push(node.right);
                stack.push(path + "->" + node.right.val);
            }

            // 左孩子后入栈（先处理）
            if (node.left != null) {
                stack.push(node.left);
                stack.push(path + "->" + node.left.val);
            }
        }
        return result;
    }
}
```

**要点：** 右孩子先压栈，左孩子后压栈，这样出栈时左孩子先被处理，符合前序遍历顺序。

## 回溯本质总结

```
递归进入 → path 加节点
    ↓ 探索这条路的所有可能
递归返回 → path 删节点（回溯，还原现场）
    ↓ 去探索其他分支
```

回溯 = 在递归返回后，**撤销**这一步对共享状态（`path`）的修改，让其他分支看到的是"干净的"状态。

**口诀：递归和回溯永远成对，必须在同一个花括号内。**
