# 349. Intersection of Two Arrays

> [LeetCode 349. Intersection of Two Arrays](https://leetcode.com/problems/intersection-of-two-arrays/)

给定两个整数数组 `nums1` 和 `nums2`,返回它们的交集。结果中每个元素必须是**唯一**的,可以按任意顺序返回。

**Example:**

```
Input: nums1 = [4,9,5], nums2 = [9,4,9,8,4]
Output: [9,4]
```

---

## 方法一:计数数组(标记来源法)

### 思路

利用题目约束 `0 <= nums[i] <= 1000`,用一个固定大小的数组做"哈希表"。

**关键点:不能简单地对两个数组都 `++` 计数**,因为这样分不清某个数的出现次数是来自同一个数组还是两个数组。

**反例:** `nums1 = [1,1,1]`, `nums2 = [2]`

- 如果两个数组都 `res[x]++`,会得到 `res[1] = 3`
- 按 `res[x] >= 2` 判断交集,会错误地把 1 当成交集 ❌

### 解决:用一个数组的不同数值表示多种状态

- `res[x] == 0`:nums1 里没出现过
- `res[x] == 1`:nums1 里出现过,但还没加入结果
- `res[x] == 2`:已经加入结果(用于去重)

这种"**用一个数组的不同数值表示多种状态**"的技巧在很多题里都会用到(比如 DFS 标记访问状态:未访问 / 访问中 / 已访问)。

### 代码

```java
class Solution {
    public int[] intersection(int[] nums1, int[] nums2) {
        int[] res = new int[1001];   // 题目约束元素范围 0~1000
        int[] temp = new int[1001];  // 临时数组存结果
        int idx = 0;                 // 记录实际存了多少个

        // 第一步:标记 nums1 里出现过的元素
        for (int i = 0; i < nums1.length; i++) {
            res[nums1[i]] = 1;
        }

        // 第二步:遍历 nums2,如果在 nums1 里出现过就是交集
        for (int j = 0; j < nums2.length; j++) {
            if (res[nums2[j]] == 1) {
                temp[idx++] = nums2[j];
                res[nums2[j]] = 2;   // 标记为已加入,防止 nums2 里重复的数再次加入
            }
        }

        // 第三步:裁剪到正确大小
        int[] ans = new int[idx];
        for (int k = 0; k < idx; k++) {
            ans[k] = temp[k];
        }
        return ans;
    }
}
```

### 复杂度

- 时间:O(n + m)
- 空间:O(k),k 为元素取值范围

**缺点:** 依赖元素范围有限这个前提,不够通用。

## 方法二:HashSet(推荐,更通用)

### 思路

1. 把 `nums1` 所有元素放进 `set`
2. 遍历 `nums2`,如果元素在 `set` 中,加入 `resultSet`
3. 用第二个 `Set` 自然去重
4. 把 `Set<Integer>` 转成 `int[]` 返回

### 代码

```java
class Solution {
    public int[] intersection(int[] nums1, int[] nums2) {
        Set<Integer> set = new HashSet<>();
        Set<Integer> resultSet = new HashSet<>();

        // 第一步:nums1 放进 set
        for (int i = 0; i < nums1.length; i++) {
            set.add(nums1[i]);
        }

        // 第二步:遍历 nums2,收集交集(Set 自动去重)
        for (int j = 0; j < nums2.length; j++) {
            if (set.contains(nums2[j])) {
                resultSet.add(nums2[j]);
            }
        }

        // 第三步:Set<Integer> 转 int[]
        int[] ans = new int[resultSet.size()];
        int idx = 0;
        for (int n : resultSet) {
            ans[idx++] = n;
        }
        return ans;
    }
}
```

### 复杂度

- 时间:O(n + m)
- 空间:O(n + m)

### 为什么用两个 Set 而不是 Set + List

如果用 `List` 收集结果,需要手动去重:

```java
if (set.contains(num) && !list.contains(num)) {  // list.contains 是 O(n)
    list.add(num);
}
```

`list.contains()` 是 **O(n)**,整体会退化成 **O(n²)**。用第二个 `Set` 去重是 **O(1)**,更干净。

## 关键知识点

### 1. Java 数组声明语法

```java
int[] arr = new int[10];         // ✅ 指定大小
int[] arr = new int[]{1, 2, 3};  // ✅ 直接初始化
int[] arr = new int[];           // ❌ 编译错误,必须指定大小或初始化
```

### 2. 结果大小未知时怎么办

Java 数组创建后无法改变大小,所以"不知道结果多大"时有两种常见做法:

**做法 A:临时大数组 + 索引计数器 + 裁剪**

```java
int[] temp = new int[1001];
int idx = 0;
// ... temp[idx++] = value;
return Arrays.copyOfRange(temp, 0, idx);
// 或手动:
// int[] ans = new int[idx];
// for (int k = 0; k < idx; k++) ans[k] = temp[k];
```

**做法 B:用 `ArrayList` 或 `Set`,最后转 `int[]`**

```java
List<Integer> list = new ArrayList<>();
// ... list.add(value);
int[] ans = new int[list.size()];
for (int i = 0; i < list.size(); i++) ans[i] = list.get(i);
```

面试里两种都可以,做法 A 更快(避免装箱),做法 B 更简洁。

### 3. `List<Integer>` / `Set<Integer>` 转 `int[]` 的坑

**`list.toArray()` 不能直接得到 `int[]`:**

```java
Object[] arr = list.toArray();                    // ✅ 返回 Object[]
Integer[] arr = list.toArray(new Integer[0]);     // ✅ 返回 Integer[]
int[] arr = list.toArray(...);                    // ❌ 没有这种重载
```

**原因:** Java 泛型不支持基本类型(`List<int>` 不合法),而 `Integer[]` 和 `int[]` 在 JVM 里是完全不同的类型,不能直接转。

**三种正确写法:**

```java
// 写法 1:手动循环(最清晰,面试推荐)
int[] ans = new int[set.size()];
int idx = 0;
for (int n : set) {
    ans[idx++] = n;
}

// 写法 2:Stream(一行搞定)
return set.stream().mapToInt(Integer::intValue).toArray();

// 写法 3:List 的 Stream 写法
return list.stream().mapToInt(Integer::intValue).toArray();
```

### 4. 用数组模拟哈希表的技巧

当元素范围有限时(题目会明确约束),用数组代替 `HashMap`/`HashSet` 更快:

| 场景 | 用法 |
|---|---|
| 只记录是否出现 | `boolean[] seen` 或 `int[]` 置为 1 |
| 记录出现次数 | `int[] count`,`count[x]++` |
| 记录多种状态 | `int[]`,用不同数值表示不同状态(如 0/1/2) |

**注意:** 数组大小要根据题目给的范围设置,这题是 `0~1000`,所以开 `new int[1001]`。

---

## 总结

1. **计数数组法**:依赖元素范围,需要用**多状态标记**区分"在 nums1 中"和"已加入结果",否则会错判。
2. **HashSet 法**:更通用,用**两个 Set**(一个存 nums1,一个去重结果)比 Set + List 更高效。
3. **结果大小未知**时用"临时数组 + 索引"或"集合 + 手动转换"。
4. `List<Integer>` / `Set<Integer>` 转 `int[]` **必须手动拆箱**,不能直接 `toArray()`。
5. 用数组模拟哈希表是在元素范围已知时的常见优化技巧。
