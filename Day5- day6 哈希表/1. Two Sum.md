# 1. Two Sum

> [LeetCode 1. Two Sum](https://leetcode.com/problems/two-sum/)

给定一个整数数组 `nums` 和一个目标值 `target`，返回数组中和为 `target` 的两个元素的下标。题目保证**恰好有一个解**，且同一个元素不能使用两次。

## 解法：HashMap 存 值 → index

核心思路：用 HashMap 存 `nums[i] → i` 的映射，这样可以通过值快速查到对应的 index。

有两种写法：
1. **两次遍历**：先把所有元素放进 map，再遍历查找 complement
2. **一次遍历**：边遍历边查找，查不到再放入 map（更优）

### 写法一：两次遍历

```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        int[] res = new int[2];

        // edge case
        if (nums == null || nums.length < 2) return res;

        // 第一次遍历：建立 值 → index 的映射
        Map<Integer, Integer> map = new HashMap<>();
        for (int i = 0; i < nums.length; i++) {
            map.put(nums[i], i);  // 重复值会被后面的 index 覆盖，这是安全的
        }

        // 第二次遍历：查找 complement
        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i];
            if (map.containsKey(complement) && map.get(complement) != i) {
                res[0] = i;
                res[1] = map.get(complement);
                return res;
            }
        }

        return res;
    }
}
```

### 写法二：一次遍历（更优）

```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> map = new HashMap<>();
        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i];
            if (map.containsKey(complement)) {
                return new int[]{map.get(complement), i};
            }
            map.put(nums[i], i);  // 查不到再放入
        }
        return new int[]{};
    }
}
```

## 我踩过的坑 & 误区

### 误区 1：把 key 和 value 搞反了

一开始想的是 `key = index, value = nums[i]`，结果发现**没法通过值反查 index**，卡住了。

**教训**：HashMap 的 key/value 设计要从"查询需求"出发反推。Two Sum 需要"通过值查 index"，所以一定是 `值 → index`。

### 误区 2：担心重复值会丢答案，想用 count 代替 index

看到 nums 有重复值，第一反应是用 `值 → 出现次数` 来统计，但这样完全偏离了题目需求——我们要返回的是 **index**，不是次数。

**正确理解**：重复值只保留"最后一次出现的 index"就够了，原因有两个：
- 如果答案是两个**不同的值**配对，互不干扰
- 如果答案是两个**相同的值**配对（如 `[3, 3]`, target=6），map 里最后保存的是第二个 3 的 index，第二次遍历走到第一个 3 时，`map.get(3) != i` 成立，正好返回 `[0, 1]` ✅

**关键前提**：题目保证"**exactly one solution**"。这个假设让我们可以放心地"找到就返回"、"覆盖旧 index"，不用担心多解或漏解。如果题目改成"找所有配对"，这些技巧就都不能用了。

### 误区 3：边界检查写成 `target == 0`

```java
if (nums.length == 0 || target == 0) return res;  // ❌
```

`target` 完全可以合法地等于 0，比如 `nums=[-1, 1], target=0` → 答案是 `[0, 1]`。

**教训**：边界检查只应该排除"无效输入"，不能把合法输入误杀。正确写法：

```java
if (nums == null || nums.length < 2) return res;  // ✅
```

### 误区 4：忘记判断 "元素不能和自己配对"

```java
if (map.containsKey(target - nums[i])) {  // ❌ 少了 != i 判断
    res[0] = i;
    res[1] = map.get(target - nums[i]);
    return res;
}
```

用 `nums=[3,2,4], target=6` 会挂：
- `i=0`, complement=3, `map.get(3)=0`，查到的就是自己！
- 返回 `[0, 0]` ❌

**修复**：加上 `map.get(complement) != i` 判断。

```java
if (map.containsKey(complement) && map.get(complement) != i) {  // ✅
    ...
}
```

> 注意：**写法二（一次遍历）不需要这个判断**，因为查找发生在 put 之前——当前的 `nums[i]` 还没放进 map，不可能查到自己。

### 误区 5：写了冗余的 containsKey 判断

```java
for (int i = 0; i < nums.length; i++) {
    if (map.containsKey(nums[i])) {
        map.put(nums[i], map.get(nums[i]) + 1);  // 白做了
    }
    map.put(nums[i], i);  // 立刻被覆盖
}
```

前面的 `put` 会被后面的 `put` 立刻覆盖，完全无用。直接写一行 `map.put(nums[i], i)` 即可，HashMap 的 put 本身就是"有则覆盖，无则新增"。

### 误区 6：找到答案后没有立即 return

如果只是 `res[0] = i; res[1] = ...;` 不 return，循环会继续跑，后面的无效迭代可能把 res 覆盖掉，或者白白浪费时间。**找到就 return。**

## 易错 API 速查

| 错误写法 | 正确写法 | 说明 |
|---------|---------|------|
| `target.length() == 0` | `nums.length == 0` | `int` 没有 `length()` 方法 |
| `map.contains(key)` | `map.containsKey(key)` | HashMap 没有 `contains` 方法 |
| `map.getKey` | `map.get(key)` | 获取 value 用 `get(key)` |

## 复杂度

- **时间**：O(n) — 每个元素最多访问常数次，HashMap 的 get/put 平均 O(1)
- **空间**：O(n) — HashMap 最多存 n 个元素

## 总结

1. **key/value 设计要反推查询需求**：要"通过 X 查 Y"，就存 `X → Y`
2. **善用题目的约束条件**："exactly one solution" 允许很多偷懒技巧
3. **边界检查只排除无效输入**，不要误杀合法输入
4. **元素不能和自己配对**：两次遍历写法必须加 `!= i` 判断
5. **一次遍历写法更优**：天然避免自配对，且找到即返回
