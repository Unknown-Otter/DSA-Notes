# 15. 3Sum

> [LeetCode 15. 3Sum](https://leetcode.com/problems/3sum/)

给定整数数组 `nums`，返回所有满足 `nums[i] + nums[j] + nums[k] == 0` 且 `i`、`j`、`k` 互不相同的三元组。结果集不能包含重复的三元组。

**示例：**

```
Input:  nums = [-1,0,1,2,-1,-4]
Output: [[-1,-1,2],[-1,0,1]]
```

## 1. 思路：降维成 Two Sum

三数之和 `nums[i] + nums[j] + nums[k] == 0` 可以变形为：

```
nums[j] + nums[k] == -nums[i]
```

也就是说，**固定一个数 `nums[k]`，然后在剩下的元素里找两数之和等于 `-nums[k]`**,这就把 3Sum 降维成了 Two Sum 子问题。

- 外层:遍历每个 `nums[k]` 作为固定位
- 内层:调用 `twoSum`,找所有和为 `-nums[k]` 的数对
- 拼接:把 `nums[k]` 和每一对 `(a, b)` 组合成三元组

这个"降维"套路同样适用于 4Sum、kSum。


**这份代码的问题清单:**

1. **方法嵌套定义**:`twoSum` 写在了 `threeSum` 内部。Java 不允许方法嵌套,辅助方法必须和 `threeSum` 平级,放在 class 里。
2. **`List` 拼写错误**:写成了小写 `list<Integer>`。
3. **逻辑错误 `nums[k] - twoSum(...) == 0`**:`twoSum` 返回 `List<Integer>`,不能和 `int` 相减。
4. **条件里的 bug**:`!map.get(temp)!=i` 这里 `!` 是多余的,对 `Integer` 使用 `!` 会编译错误。
5. **`twoSum` 返回类型错误**:返回 `List<Integer>` 只能装一对,但同一个 `k` 固定后可能有**多对**配对。
6. **完全没有处理三元组去重**(最关键的问题,见下一节)。
7. **没有排序**:没有排序就无法做去重。

## 3. 关键误区:两种"重"的区别

> 这是我最初以为"我已经去重了"的原因,其实我只处理了一种,漏了另一种。

**A. 避免同一个元素被用两次(index 去重)**

比如 `nums = [1, 2, 3]`,`target = 2`,不能用 `1 + 1 = 2` 然后返回 `(1, 1)`——因为数组里只有**一个** `1`。

我原代码里的 `i != idx` 和 `map.get(temp) != i` 防的就是这个。**这个确实需要防,但它不是 3Sum 的去重难点。**

**B. 避免返回重复的三元组(value 去重)**

比如 `nums = [-1, -1, 0, 0, 1, 1]`:

- 下标 `(0, 2, 4)` → `[-1, 0, 1]`
- 下标 `(0, 3, 4)` → `[-1, 0, 1]`
- 下标 `(0, 2, 5)` → `[-1, 0, 1]`
- 下标 `(1, 3, 5)` → `[-1, 0, 1]`
- ……

这些三元组**下标完全不同,但值一样**。题目要求不能有重复三元组,所以只能保留一个 `[-1, 0, 1]`。

**我最初的代码完全没处理 B 类重复,这才是 3Sum 的真正考点。**

| 需要做的去重 | 我最初做了吗 |
|------------|------------|
| 同一个元素不能用两次(`i != idx`) | ✅ 做了 |
| 外层固定的数字重复时要跳过 | ❌ 没做 |
| 内层找到一对后,后面相同值要跳过 | ❌ 没做 |
| `twoSum` 能返回多对 | ❌ 返回类型不够 |

## 4. 第二个易错点:twoSum 遍历范围

修完前面的问题后,我又踩了一个坑:即使加了外层 k 去重,依然会产生重复输出。

**错误输出:**

```
Input:    [-1,0,1,2,-1,-4]
Output:   [[-1,0,1],[-1,-1,2],[0,-1,1],[1,-1,0],[2,-1,-1]]
Expected: [[-1,-1,2],[-1,0,1]]
```

**原因:** 我的 `twoSum` 从 `i = 0` 开始遍历**整个数组**,这意味着 `twoSum` 会往 `idx` 的**左边**也去找。追踪一下排序后的 `[-4,-1,-1,0,1,2]`:

- `k=1 (nums[k]=-1)`:twoSum 在整个数组找和为 1 → 找到 `(0,1)` 和 `(-1,2)` → 拼出 `[-1,0,1]`、`[-1,-1,2]` ✅
- `k=3 (nums[k]=0)`:twoSum 看到左边的 `-1` 和右边的 `1` → 找到 `(-1,1)` → 拼出 `[0,-1,1]` ❌ 重复了
- `k=4 (nums[k]=1)`:找到 `(-1,0)` → 拼出 `[1,-1,0]` ❌
- `k=5 (nums[k]=2)`:找到 `(-1,-1)` → 拼出 `[2,-1,-1]` ❌

**同一组 `{-1, 0, 1}` 被数了多次,只是"谁当 k、谁当 pair"的角色换了。**

**解决办法:让 `twoSum` 只往 `idx` 的右边找**

```java
for (int i = idx + 1; i < nums.length; i++) { ... }
```

这样每个三元组只会在**下标最小的那个元素当 k** 的时候被发现一次。这个原则对 4Sum、kSum 都适用。

## 5. 正确代码(HashMap 版本)

```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> res = new ArrayList<>();

        // 排序是去重的前提
        Arrays.sort(nums);

        for (int k = 0; k < nums.length; k++) {
            // 去重①:跳过重复的 k(和前一个比较,不是和后一个)
            if (k > 0 && nums[k] == nums[k - 1]) continue;

            // 找所有和为 -nums[k] 的数对
            List<List<Integer>> pairs = twoSum(-nums[k], nums, k);

            // 拼成三元组
            for (List<Integer> pair : pairs) {
                List<Integer> triplet = new ArrayList<>();
                triplet.add(nums[k]);
                triplet.add(pair.get(0));
                triplet.add(pair.get(1));
                res.add(triplet);
            }
        }

        return res;
    }

    // idx 是固定位的下标,twoSum 只在 idx 右边找
    private List<List<Integer>> twoSum(int target, int[] nums, int idx) {
        List<List<Integer>> list = new ArrayList<>();
        Map<Integer, Integer> map = new HashMap<>();

        // 关键:从 idx + 1 开始,避免三元组被重复发现
        for (int i = idx + 1; i < nums.length; i++) {
            int temp = target - nums[i];

            if (map.containsKey(temp)) {
                list.add(Arrays.asList(temp, nums[i]));

                // 去重②:跳过后面重复的 nums[i]
                // 比如 [-2,-2,0,0,2,2],找到 (0,2) 后,下一个 2 又会找到 (0,2)
                while (i + 1 < nums.length && nums[i] == nums[i + 1]) i++;
            }

            map.put(nums[i], i);
        }

        return list;
    }
}
```

**关键点:**

- **三处去重缺一不可**:排序、外层 k 去重、内层 nums[i] 去重
- **`twoSum` 只往右找**:`i = idx + 1` 是避免三元组被重复发现的关键
- **条件简化**:因为 `i` 从 `idx+1` 开始,原来的 `i != idx` 和 `map.get(temp) != i` 都不再需要

## 6. 推荐解法:排序 + 双指针

HashMap 版本能过,但面试官大概率会问"能不能优化空间?"。**业界标准解法是排序 + 双指针**,时间复杂度一样 O(n²),但空间 O(1),去重逻辑也更干净。

```java
class Solution {
    public List<List<Integer>> threeSum(int[] nums) {
        List<List<Integer>> res = new ArrayList<>();
        Arrays.sort(nums);

        for (int i = 0; i < nums.length - 2; i++) {
            // 剪枝:排序后 nums[i] > 0,后面不可能和为 0
            if (nums[i] > 0) break;

            // 去重①:跳过重复的 i
            if (i > 0 && nums[i] == nums[i - 1]) continue;

            int left = i + 1;
            int right = nums.length - 1;
            int target = -nums[i];

            while (left < right) {
                int sum = nums[left] + nums[right];

                if (sum == target) {
                    res.add(Arrays.asList(nums[i], nums[left], nums[right]));

                    // 去重②:跳过重复的 left
                    while (left < right && nums[left] == nums[left + 1]) left++;
                    // 去重③:跳过重复的 right
                    while (left < right && nums[right] == nums[right - 1]) right--;

                    // 找到一组后,两边同时移动
                    left++;
                    right--;
                } else if (sum < target) {
                    left++;  // 和太小,左指针右移
                } else {
                    right--; // 和太大,右指针左移
                }
            }
        }

        return res;
    }
}
```

**三个去重点详解** (以 `[-1,-1,-1,0,1,2]` 为例):

- **去重①:`if (i > 0 && nums[i] == nums[i - 1]) continue;`**
  当 `i=0` 固定 `-1` 时找到了所有解,`i=1` 时 `nums[1]` 还是 `-1`,跳过避免重复。**必须和前一个比较**,否则会漏掉 `[-1,-1,2]` 这种需要两个相同数字的情况。

- **去重②③:找到解后跳过重复的 left/right**
  找到 `[-1,-1,2]` 后,如果 `nums[left+1]` 还是 `-1`,下一轮又会产生一样的三元组,所以要跳过。

## 7. 复杂度分析

| 解法 | 时间 | 空间 |
|-----|------|------|
| HashMap | O(n²) | O(n) |
| 双指针 | O(n²) | O(1)(不算结果数组) |

排序本身 O(n log n),被 O(n²) 吸收。

## 8. 总结

1. **固定一个数 → 转化为 Two Sum**:降维思路,4Sum、kSum 都是这个套路。
2. **排序的三个作用**:去重、双指针可行、剪枝(`nums[i] > 0` 直接 break)。
3. **两种"重"要分清**:index 重复(同一个元素用两次) vs value 重复(不同下标组成相同三元组)。本题的真正考点是后者。
4. **去重要在三个地方做**:外层固定位、内层左边界(或 left 指针)、内层右边界(或 right 指针)。
5. **twoSum 的遍历范围必须限制在固定位的一侧**:否则同一组三元组会被不同角色的"固定位"重复发现。
6. **双指针 vs HashMap**:在需要**去重 + 返回值本身**的题目里,双指针远优于 HashMap;在 Two Sum 原题(返回 index、不要求去重)里,HashMap 更好因为不能排序。
7. **面试首选双指针**:去重干净、空间 O(1),面试官会觉得你刷过题、有经验。
