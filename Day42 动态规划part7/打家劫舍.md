
# 198. 打家劫舍 (House Robber)

> [LeetCode 198. 打家劫舍](https://leetcode.cn/problems/house-robber/)

你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。

给定一个代表每个房屋存放金额的非负整数数组，计算你 **不触动警报装置** 的情况下，一夜之内能够偷窃到的最高金额。

**示例：**

```
输入：[2,7,9,3,1]
输出：12
解释：偷窃 1 号房屋 (2) + 3 号房屋 (9) + 5 号房屋 (1) = 12
```

---

## 核心洞察：这就是回溯的"选 / 不选"模板

打家劫舍本质上和回溯里的"选 / 不选"是 **同一个决策模型**，只是 DP 把这个决策做了 **记忆化 + 自底向上** 的优化。

对每个房子 `i`，都面临同一个二选一：

- **选** `nums[i]`：那么 `i-1` 不能选，跳到 `i-2` 继续决策
- **不选** `nums[i]`：直接跳到 `i-1` 继续决策

这就是灵神回溯模板里的经典二叉决策树。

---

## 思路演进：回溯 → 记忆化 → 递推 DP → 空间优化

### Step 1：纯回溯（O(2ⁿ)）

```java
// dfs(i) 表示从下标 0 到 i 能偷到的最大金额
private int dfs(int i, int[] nums) {
    if (i < 0) return 0;
    // 不选 i：跳到 i-1
    int notRob = dfs(i - 1, nums);
    // 选 i：跳到 i-2，加上 nums[i]
    int rob = dfs(i - 2, nums) + nums[i];
    return Math.max(notRob, rob);
}

public int rob(int[] nums) {
    return dfs(nums.length - 1, nums);
}
```

> 这里 **不需要 `path` 记录路径**，因为只关心最大值，所以是带返回值的 dfs，而不是 void + path。

### Step 2：加记忆化（O(n)）

观察到 `dfs(i)` 只依赖 `i`，会被重复计算（比如 `dfs(3)` 既被 `dfs(5)` 调用也被 `dfs(4)` 调用），用数组缓存：

```java
private int[] memo;

private int dfs(int i, int[] nums) {
    if (i < 0) return 0;
    if (memo[i] != -1) return memo[i];
    int notRob = dfs(i - 1, nums);
    int rob = dfs(i - 2, nums) + nums[i];
    return memo[i] = Math.max(notRob, rob);
}
```

### Step 3：翻译成递推 DP（自底向上）

把递归的依赖关系倒过来写：

```java
public int rob(int[] nums) {
    int n = nums.length;
    int[] dp = new int[n + 2]; // 偏移 2 位，避免处理 i-2 < 0
    for (int i = 0; i < n; i++) {
        dp[i + 2] = Math.max(dp[i + 1], dp[i] + nums[i]);
    }
    return dp[n + 1];
}
```

转移方程 `dp[i+2] = max(dp[i+1], dp[i] + nums[i])` **完全对应** 回溯里的 `max(notRob, rob)`。

### Step 4：空间优化（O(1)）

只依赖前两项，用两个变量滚动：

```java
public int rob(int[] nums) {
    int prev2 = 0, prev1 = 0;
    for (int num : nums) {
        int cur = Math.max(prev1, prev2 + num);
        prev2 = prev1;
        prev1 = cur;
    }
    return prev1;
}
```

## 动规五部曲（代码随想录标准解法）

### 1. 确定 dp 数组及下标含义

**`dp[i]`：考虑下标 i（包括 i）以内的房屋，最多可以偷窃的金额。**

> 注意"考虑"和"一定偷"的区别——`dp[i]` 是考虑到第 i 间为止的最优解，并不强制偷第 i 间。

### 2. 确定递推公式

决定 `dp[i]` 的因素就是 **第 i 间偷还是不偷**：

- **偷第 i 间**：`dp[i-1]` 一定不考虑，结果为 `dp[i-2] + nums[i]`
- **不偷第 i 间**：考虑 `i-1` 间，结果为 `dp[i-1]`

取最大值：

```java
dp[i] = Math.max(dp[i - 2] + nums[i], dp[i - 1]);
```

### 3. dp 数组初始化

递推公式的基础是 `dp[0]` 和 `dp[1]`：

- `dp[0] = nums[0]`（只有一间房，必偷）
- `dp[1] = max(nums[0], nums[1])`（两间房挑大的偷）

```java
dp[0] = nums[0];
dp[1] = Math.max(nums[0], nums[1]);
```

### 4. 确定遍历顺序

`dp[i]` 依赖 `dp[i-2]` 和 `dp[i-1]`，从前往后遍历：

```java
for (int i = 2; i < nums.length; i++) {
    dp[i] = Math.max(dp[i - 2] + nums[i], dp[i - 1]);
}
```

### 5. 举例推导 dp 数组

以 `nums = [2, 7, 9, 3, 1]` 为例：

| i      | 0 | 1 | 2  | 3  | 4  |
|--------|---|---|----|----|----|
| nums   | 2 | 7 | 9  | 3  | 1  |
| dp     | 2 | 7 | 11 | 11 | 12 |

- `dp[2] = max(dp[0] + nums[2], dp[1]) = max(2+9, 7) = 11`
- `dp[3] = max(dp[1] + nums[3], dp[2]) = max(7+3, 11) = 11`
- `dp[4] = max(dp[2] + nums[4], dp[3]) = max(11+1, 11) = 12` ✅

## 完整代码（Java）

### 标准 DP 版本

```java
class Solution {
    public int rob(int[] nums) {
        if (nums == null || nums.length == 0) return 0;
        if (nums.length == 1) return nums[0];

        int[] dp = new int[nums.length];
        dp[0] = nums[0];
        dp[1] = Math.max(nums[0], nums[1]);

        for (int i = 2; i < nums.length; i++) {
            dp[i] = Math.max(dp[i - 1], dp[i - 2] + nums[i]);
        }

        return dp[nums.length - 1];
    }
}
```

**时间复杂度：** O(n)
**空间复杂度：** O(n)

### 滚动数组优化版本

```java
class Solution {
    public int rob(int[] nums) {
        if (nums.length == 1) return nums[0];

        int[] dp = new int[2];
        dp[0] = nums[0];
        dp[1] = Math.max(nums[0], nums[1]);
        int res = dp[1];

        for (int i = 2; i < nums.length; i++) {
            res = Math.max(dp[0] + nums[i], dp[1]);
            dp[0] = dp[1];
            dp[1] = res;
        }

        return res;
    }
}
```

**时间复杂度：** O(n)
**空间复杂度：** O(1)

## 回溯 vs DP：关键区别

回溯和 DP 的差异 **不在思考方式**，而在 **计算方式**：

| 维度       | 回溯                          | DP                              |
|------------|-------------------------------|---------------------------------|
| 关心的东西 | 枚举所有方案（常需 `path`）   | 只要最优值 / 方案数             |
| 时间复杂度 | O(2ⁿ)（决策树有 2ⁿ 个叶子）   | O(n)（子问题只有 n 个）         |
| 状态压缩   | 一般不行                      | 可记忆化、可滚动数组            |

**为什么 DP 能从 O(2ⁿ) 压到 O(n)？**
因为子问题数量只由 `i` 唯一决定，只有 n 个，而不是 2ⁿ 个。重复的子问题被缓存了。

---

## 判断技巧：以后看到"选 / 不选"类问题怎么走？

- **要枚举所有具体方案**（子集、组合、排列） → **回溯**
- **只要最优值或方案数** → **DP**（先按回溯思路写出递归，再翻译成递推）

灵神视频里反复强调的 **"当前操作？子问题？下一个子问题？"**，本质就是在帮你写出回溯函数，这个回溯函数稍加改造（去掉 `path`、加记忆化、翻译成递推）就是 DP 的转移方程。

## 总结

1. **打家劫舍 = 回溯"选/不选"模板 + 记忆化**，是 DP 入门的经典题。
2. **状态定义**：`dp[i]` 表示考虑前 i 间房能偷到的最大金额（不强制偷第 i 间）。
3. **转移方程**：`dp[i] = max(dp[i-1], dp[i-2] + nums[i])`
4. **初始化**：`dp[0] = nums[0]`，`dp[1] = max(nums[0], nums[1])`
5. **空间可优化到 O(1)**（只依赖前两项，滚动变量）。
